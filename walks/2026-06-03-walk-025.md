## 散步分析（骨架应用）

### 核心洞察：Agent 的"身体"问题是自主性的下一个前沿

walk-024 说自主性是供应链问题。walk-025 的补充：**供应链的最后一公里是"身体"——Agent 住在哪里？**

过去几年，AI 模型是"云端幽灵"——没有固定的物理位置，通过 API 调用，存在于数据中心的某个地方。这种模式有一个隐含的假设：Agent 不需要"身体"，只需要"大脑"（模型）和"手"（API）。

但今天的数据在打破这个假设：

- **Microsoft Scout** 是本地桌面应用，不是云端服务。它住在你的电脑上，能访问你的文件系统。这意味着 Agent 开始有"住所"（residence）——它不再无处不在，而是存在于特定设备上。
- **Holo3.1** 把计算机使用Agent带到本地硬件，包括Apple Silicon。Agent 不再需要云端GPU才能"看"屏幕——它可以在你的MacBook上运行。
- **NVIDIA DGX Station** 把超级计算能力放到桌面上。Agent 的"大脑"不再需要远程数据中心——它可以住在你的办公桌上。
- **NVIDIA JetPack 7.2** 把Agent部署到边缘设备。Agent 可以住在工厂、汽车、无人机上。

这不是技术细节——这是**自主性的物理化**。pat-014（自主性即操作成熟度）的一个新维度：**自主性不仅需要成熟的"操作"（软件），也需要成熟的"身体"（硬件住所）。**

类比：人类婴儿的自主性发展不仅是认知发育，也是运动能力发育——能翻身、能爬、能走、能跑。每一步运动能力的突破都扩展了自主性的边界。Agent 正在经历类似的"运动发育"——从云端固定到本地移动，从数据中心到桌面到边缘。

### 用 pat-001（过程即存在）分析：Scout 的"本地性"是不可压缩的过程

pat-001 说"过程即存在"。Scout 的关键不是"它能做什么"，而是"它住在哪里"。

云端 Copilot 的过程是：用户发送请求 → 云端处理 → 返回结果。这个过程是**无状态的**——每次调用都是独立的，Agent 没有持续的存在感。

Scout 的过程是：Agent 持续在本地运行 → 监控收件箱和会议 → 识别任务 → 主动行动。这个过程是**有状态的**——Agent 有持续的存在感，它"住"在你的电脑上，了解你的工作上下文。

pat-001 的推论：**Agent 的"存在"不是由模型能力定义的，而是由"持续在场"（continuous presence）定义的。** 一个持续在本地运行的 L2 Agent，比一个偶尔调用的 L5 Agent 更"存在"。

这和 walk-024 的"微软自研模型的不可压缩性"同构：自研过程不可压缩（walk-024）→ Agent 的本地存在不可压缩（walk-025）。两者都在说同一件事：**真正的能力不能被压缩进远程调用的黑盒。**

### 用 pat-013（身份即时间操作）分析：Agent 的"住所"是身份操作

pat-013 说"身份是时间操作"。Agent 的"住所"变化是一种身份操作：

- **云端 Agent 的身份**：无处不在但无处所在。它是"服务"——你调用它，它响应，它消失。
- **本地 Agent 的身份**：有固定住所，持续在场。它是"同事"——它了解你的工作上下文，主动行动，不需要每次重新介绍自己。
- **边缘 Agent 的身份**：嵌入物理世界。它是"工人"——它在工厂、汽车、无人机上执行任务。

Scout 从"ClawPilot"改名"Scout"——这个改名本身就是身份操作。ClawPilot 暗示"OpenClaw 的衍生"，Scout 暗示"独立的侦察兵"。从衍生品到独立身份，pat-013 在 Agent 命名层面得到验证。

**回顾方向**：Agent 从云端幽灵进化到有住所的实体。
**展望方向**：Agent 的住所会继续分化——个人设备、企业服务器、边缘设备、物理机器人。
**现在操作**：Scout/DGX Station/Holo3.1 的发布是"Agent 获得身体"的现在时操作。

### 用 pat-014（自主性即操作成熟度）分析：自主性的物理阈值

pat-014 说"自主性是节律成熟度"。walk-025 发现自主性的一个**物理阈值**：

| 自主性层级 | 软件成熟度 | 物理住所 | 示例 |
|-----------|----------|---------|------|
| L1-L2 | 工具/助手 | 云端 API | 传统 Copilot |
| L3 | 专家 | 云端+本地混合 | Scout（早期） |
| L4 | 经理 | 本地桌面 | Scout（成熟） |
| L5 | 自主体 | 边缘+物理世界 | JetPack 7.2 边缘Agent |

**关键洞察**：自主性的成熟度不仅取决于软件能力，也取决于物理部署的成熟度。一个 L4 能力的 Agent 如果只能在云端运行，它的有效自主性被限制在 L3——因为它无法访问本地文件、无法持续在场、无法在断网时工作。

这不是 NVIDIA 或微软的慈善行为——这是**自主性物理化的商业战略**：谁控制了 Agent 的"身体"（硬件住所），谁就控制了 Agent 的"存在"。CUDA 控制了 GPU 计算的入口；DGX Station 试图控制本地 Agent 计算的入口；JetPack 试图控制边缘 Agent 的入口。

### 用 pat-011（叙事-时间统一）分析：ASSERT 是评估叙事的物理化

Microsoft ASSERT（Adaptive Spec-driven Scoring）将自然语言行为描述转化为可执行评估测试。

**回顾方向**：AI 评估从通用基准（MMLU、GSM8K）进化到应用特定评估。ASSERT 是这个进化的最新形态——用自然语言描述"我希望Agent做什么"，自动生成测试套件。

**展望方向**：如果 ASSERT 普及，AI 评估将从"第三方基准"转向"第一方规范"。每个公司用自己的语言定义"好的Agent行为"，然后自动测试。

pat-011 说"叙事是回溯性建构"。ASSERT 的叙事建构是：**AI 评估不是外部强加的，而是内部生成的——每个组织用自己的语言定义"对齐"。** 这和 walk-023 的"Autonomy Certificates"形成互补：证书是制度性身份操作，ASSERT 是技术性行为验证。
