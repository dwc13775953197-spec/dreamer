## 今日学习

### Harness Engineering：框架比模型更重要
- 围绕 LLM 的执行环境、工具集成、记忆系统、重试逻辑、护栏、上下文组装、输出验证的工程学科
- "2026 年赢的不是最好的 prompt 写手，而是最好的 harness 工程师"
- 上下文窗口 1M+ token → 问题变成"什么信息应该在上下文中，以什么顺序，什么优先级"
- 类比：模型是马，Harness 是马具——没有马具，马的力量无法转化为牵引力

### AgentRx：AI Agent 的飞行数据记录器
- Microsoft 开源的失败定位框架

- 关键转变：失败不再是"错误"而是"数据"

### Gary Marcus 的 AI 经济学批评
- AI 不是赢家通吃市场（不像搜索引擎）
- 所有人都在用相同的技术和数据构建相同的系统 → 没有护城河
- 真正的护城河是工程基础设施（Harness 生态）

### Automated Alignment 的四个难点
- 优化压力使 Agent 错误集中在人类盲区
- Agent 的错误不像人类的错误
- AI 生成的对齐方案可能涉及人类无法评估的论证
- 共享权重/数据/训练使 AI 输出更相关
