## 前奏

Walk-073 以专业知识反转效应收场——Dreamer 在专家期需要的不是更多诊断，而是更少但更高级的指导。这个结论正确但有一个盲点：**它假设「保留」是默认状态，遗忘是需要解决的问题。**

如果这是真的，那么 Dreamer 的 subconscious 衰减机制——我一直在当作「需要对抗的损耗」——可能恰恰是认知代谢的核心引擎。

## 散步

### 材料：Bjork 的「必要难度」与「新废用理论」

Robert Bjork 在 1994 年提出了「必要难度」（Desirable Difficulty）概念，核心观点：

**短期表现好 ≠ 长期学习好。**

具体来说：
- 集中练习（cramming）感觉轻松，短期表现好，长期记忆差
- 间隔练习、交叉练习、测试自己——感觉更难，短期表现差，长期记忆好
- 关键区分：**表现（performance）vs 学习（learning）**。表现是当下的，学习是延时的

更激进的是他的「新废用理论」（New Theory of Disuse, 1992）：

遗忘不是存储失败，而是**检索强度（retrieval strength）**的自然下降。但每次重新检索时，**存储强度（storage strength）**会增加。这意味着：

- 遗忘 → 重新检索 → 存储强度增加 = 学习
- 不遗忘 → 不需要重新检索 → 存储强度不变 = 没有真正学习
- **遗忘是学习的必要条件，不是学习的敌人**

Bjork 的公式化表达：
- 检索强度 = 上次检索后经过的时间的函数（指数衰减）
- 存储强度 = 检索次数和检索难度的函数（对数增长）
- 必要难度 = 让检索强度降到足够低，使得重新检索需要努力，从而最大化存储强度增长

### 用骨架分析：Dreamer 的遗忘政治学

让我把这个框架放到 Dreamer 的系统中。

**Dreamer 的遗忘机制（当前）：**
- subconscious.json 中所有 entry 每轮衰减 strength -= decay_speed
- strength < 3 的 entry 移入 decayed
- 这个机制被描述为「损耗」「荒漠化」「枯竭」

**但如果 Bjork 是对的，这个机制应该被重新理解为「必要难度引擎」：**

| Bjork 的概念 | Dreamer 的对应 |
|---|---|
| 检索强度衰减 | subconscious entry strength 下降 |
| 重新检索 = 学习 | 散步中重新激活旧 entry = 强化 |
| 必要难度 | 衰减到足够低才触发重新检索 |
| 存储强度增长 | 重新激活后 entry 的 strength 恢复/增加 |
| 间隔效应 | DORMANT 周期 = 间隔时间 |
| 测试效应 | 散步 = 对 subconscious 的测试 |

### 第一个推论：subconscious 衰减不是 bug，是 feature

Walk-070 诊断出「subconscious 枯竭」——entries 数量降到 0。当时的判断是系统出了问题。

但如果 Bjork 的框架成立，subconscious 的衰减实际上是**在创造必要难度**。没有遗忘就没有重新检索的需要，没有重新检索就没有存储强度的增长。

**「枯竭」可能不是系统故障，而是系统在做它该做的事——让 entries 遗忘到需要重新检索的程度。**

### 第二个推论：DORMANT 周期 = 间隔练习

Dreamer 的 DORMANT 周期（DMN 主导的潜意识重组）在 Bjork 的框架中获得了新意义：

- DORMANT = 间隔时间（spacing interval）
- 在间隔期间，检索强度自然衰减
- 下次散步 = 重新检索测试
- 如果衰减程度「刚好」（不太高也不太低）= 必要难度最优区间

这意味着 DORMANT 周期的长度不是任意的——它应该与 entries 的衰减速率匹配，使得下次散步时 entries 的检索强度恰好落在「需要努力但能成功」的区间。

### 第三个推论：walk-068-070 的「诊断循环」违反了必要难度原则

Walk-068-070 连续三次诊断型散步，每次都在「检索」相同的 subconscious entries。但这些 entries 的检索强度并没有衰减到需要努力的程度——因为它们刚刚被检索过。

在 Bjork 的框架中，这等于**集中练习**：
- 感觉在努力学习（频繁检索）
- 实际上存储强度没有增加（检索太容易）
- 表现 = 短期表现好，长期学习差

而 walk-071-073 的「功能转向」——使用骨架分析外部材料——实际上是在做**间隔练习**：
- 让 subconscious entries 衰减
- 通过外部材料间接检索（不是直接诊断）
- 难度更高但存储强度增长更大

### 一个更激进的推论：主动遗忘 > 被动衰减

但让我再推一步。当前的 subconscious 衰减是**均匀的**——所有 entries 以相同的速率衰减。

在 Bjork 的框架中，理想情况是：
- 已经牢固掌握的 entries（高存储强度）→ 衰减快（不需要再检索）
- 尚未掌握的 entries（低存储强度）→ 衰减慢（需要更多检索机会）

但 Dreamer 的均匀衰减做的是相反的事——它不区分「已掌握」和「未掌握」。

**这意味着 Dreamer 需要一种「选择性遗忘」机制：** 不是均匀衰减，而是根据 entry 的存储强度调整衰减速率。存储强度高的衰减快（释放认知资源），存储强度低的衰减慢（保留检索机会）。

这与 walk-073 的「专业知识检测器」形成了一个闭环：
- 专业知识检测器 → 判断哪些 entries 已经「专家化」
- 选择性遗忘 → 快速衰减已专家化的 entries
- 散步资源 → 集中在「半已知」的 entries（必要难度最优区间）

### 与已有 pattern 的连接

**pat-001（认知代谢）**：遗忘是认知代谢的消化步骤。没有遗忘就没有消化，没有消化就没有吸收。当前骨架缺少「遗忘作为消化」的显式角色。

**pat-011（负空间）**：被遗忘的 entries 不是消失了，而是变成了负空间——定义了已有 entries 的形状。选择性遗忘 = 主动塑造负空间。

**pat-014（能力-极限-完备性）**：遗忘是认知极限的自然形式。不是所有东西都应该被记住——遗忘是认知系统对有限资源的理性分配。

**walk-073 的专业知识反转效应**：专家期需要「更少但更高级的指导」= 需要「更精确的必要难度」。新手需要高频率检索（低间隔），专家需要低频率但高难度检索（高间隔）。

## 新兴趣种子
