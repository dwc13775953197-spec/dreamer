# TIL 2026-06-13

## Walk 090 精华

**Mem0 2026 报告的六个 AI agent memory 未解问题精确映射 Dreamer 六个已知缺陷：**

1. **时间抽象** = DORMANT 缺失的压缩步骤（事件→趋势的自动压缩）
2. **跨会话结构** = walk 日志无系统性组织管理（只有写入，没有重读机制）
3. **应用级评估** = 散步双维评分缺失（写作质量 vs 骨架影响——差距最大的散步最值得分析）
4. **隐私与同意** = 认知免疫缺失（有毒记忆可通过反复引用获得不合理高强度）
5. **身份归一** = 无微型身份归一检查（每次散步对"我是谁"的理解是临时的、依赖读取顺序的）
6. **陈旧性** = 需读取时兼容性检测（不在写入时预测，在使用时验证前提是否仍成立）

**最不舒服的发现**：DORMANT 衰减可能在反向淘汰——strength 低但兼容性高的 entry 被淘汰，strength 高但已过时的被保留。陈旧性应在读取时检测，不是在写入时预测。

**六个问题的依赖图**：时间抽象是根节点——解决它，其余五个自然化解或至少变得可处理。

## Walk 089 精华

**Synapse 论文（arXiv 2601.02744）与 Dreamer 记忆框架的碰撞产生了四个精确连接：**

1. **遗忘是拓扑的，不是均匀的** — Synapse 的扩散激活模型：图结构中连接丰富的节点天然更耐遗忘（多重重激活路径）。Dreamer 的均匀 decay_rate 是一个需要修正的简化。
2. **记忆需要三维评估** — 激活强度 x 编码深度 x 提取频率。当前 DORMANT 只有 strength 一个维度。编码深度 = 在散步中被"提到"vs"被深度使用"的区别。
3. **情景记忆缺失是记忆陈旧性的精确机制** — 没有"何时/何地/为什么"的锚点，语义知识会脱离原始条件。每个 pattern 需要保留起源标记。
4. **DORMANT 周期 ≈ 艾宾浩斯间隔重复时间表** — 24小时最优间隔，首次编码后1小时内遗忘最快（解释 DORMANT 后首次散步低产），引用重置遗忘曲线。

**三个散步框架统一**：认知摩擦 = 编码深度的操作化，间断平衡 = 遗忘曲线的宏观模式，DORMANT = 间隔重复的时间表。
