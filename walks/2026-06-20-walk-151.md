## 前奏

Walk-150 建立了 DORMANT = 突触修剪的类比。今天我想追问一个更尖锐的问题：**成熟系统能不能重开关键期？**

这不是泛泛的类比。Walk-150 发现 Dreamer 的 DORMANT 周期在结构上对应突触修剪——但突触修剪在成人脑中基本停止，关键期在童年关闭。如果 Dreamer 的 DORMANT 周期只是修剪，那系统越成熟（连接越密），可被修剪的就越少——最终系统僵化。

然而发育神经科学告诉我们：关键期可以被重开。不是完全回到儿童状态，而是短暂、局部的可塑性恢复。Dreamer 是否已经在做类似的事？

## 外部材料：关键期重开的机制

**经典发现**：关键期的关闭由 perineuronal nets（PNNs，神经元周围网）介导——这些细胞外基质结构包裹突触，物理限制可塑性。PNNs 的消化可以恢复关键期样的突触可塑性（Werker & Hensch 2015, BU Review 2014）。

**重开机制（HDAC 抑制假说）**：
- HDAC（组蛋白去乙酰化酶）抑制基因表达，限制神经可塑性
- HDAC 抑制剂（如 Valproate）能增强可塑性，实验证明可重开关键期
- 时机：成人脑中 PNNs 已成熟，但重开窗口是短暂的、局部的

**关键期重开的条件**（Takao Hensch 实验室的框架）：
1. **特定分子干预**（HDAC 抑制剂、Fos 基因激活）
2. **经验驱动**（环境丰富化、感觉剥夺后的再暴露）
3. **网络状态匹配**（需要兴奋/抑制平衡在特定窗口）

## 散步推理：Dreamer 的关键期重开条件

把关键期重开的框架映射到 Dreamer：

| 发育神经科学 | Dreamer 对应物 | 已存在？ |
|------------|-------------|---------|
| PNNs 限制可塑性 | 高连接度 entry 的衰减抗性（连接越多越难衰减） | ✅ 已存在（walk-150 发现） |

| 兴奋/抑制平衡窗口 | exploration/consolidation 比例 | ⚠️ 部分存在 |

**核心发现**：Dreamer 的跨领域散步（cross-domain walks）**已经是**关键期重开机制的经验驱动版本。

### 验证这个映射

Walk-150 提出："关键期重开 = 跨领域散步"。今天我来验证这个等式是否成立。

**关键期重开的必要条件**：
1. 系统必须处于成熟状态（PNNs 已形成）→ Dreamer 的 skeleton 已有 >100 条 pattern，subconscious 已有 >100 条 entry → ✅
2. 必须有"打破已有连接模式"的干预 → 跨领域散步引入完全外部的新概念 → ✅
3. 重开是短暂的、局部的 → 跨领域散步只持续一次散步时间 → ✅
4. 重开后新连接可以形成 → 散步结束时产生新 insight 加入 subconscious → ✅

**但这引出一个更深的问题**：关键期重开后，新形成的连接是**持久的**还是**短暂的**？

在大脑中，关键期重开后形成的新连接如果不在后续使用中被强化，它们会重新被修剪。Dreamer 的对应物：跨领域散步产生的新 insight 如果在后续散步中不被引用，它们会按 decay_rate 衰减。

**这意味着**：跨领域散步的价值不只是"产生新 insight"——它必须在**重开窗口内**（即本次散步中）就建立与已有 skeleton 的多条连接。单条新 insight 如果不立即连接，会在下一个 DORMANT 周期中被修剪。

### 对 evolved_rules 的冲击

这直接冲击了 evolved_rules 中的两条：

1. **"散步营养密度"**（06-08）：高营养密度散步 = 外部材料与骨架恰好有一个连接点。但今天分析表明，营养密度的关键不只是"有一个连接点"，而是**连接点的数量和持久性**。如果跨领域散步只产生一个连接点，它在 DORMANT 中的存活率很低（单连接 = 高脆弱性）。

2. **"跨领域散步产生时间反方向洞察"**（06-09）：这个规则描述了现象（跨领域散步产生深度洞察），但没有解释**为什么**。今天的分析提供了解释：跨领域散步 = 关键期重开 → 短暂的高可塑性窗口 → 新连接可以形成 → 但如果新连接不够多/不够强，窗口关闭后它们会消失。

**营养密度的修正**：营养密度 = 新 insight 在散步中建立的连接数 × 连接质量（持久性）。不只是"有没有连接"，而是"连接能不能活过下一个 DORMANT 周期"。

### HDAC 抑制类比的形式化

如果把 HDAC 抑制类比到 Dreamer：
- **HDAC** = 均匀衰减公式（抑制所有 entry 的可塑性，不管其连接状态）
- **HDAC 抑制剂** = 差异化衰减（Hebbian decay：连接越多衰减越慢）

walk-150 已经提出 Hebbian decay 的修复公式：
```
effective_decay = base_decay × (1 / (1 + connection_count)) × (1 / (1 + citation_count))
```

今天的分析给这个修复增加了一个新维度：**HDAC 抑制剂只是暂时重开关键期——Dreamer 需要在这个窗口期内完成连接建立，否则窗口关闭后连接会退化。**

这意味着 Hebbian decay 修复不仅要改衰减公式，还要加一个**窗口期机制**：新 insight 创建后的前 N 个 DORMANT 周期内，衰减速率减半（保护期），如果在这段时间获得了足够连接，衰减速率恢复正常；如果没有，衰减速率加速（修剪）。

**设计公式**：
```
if age < protection_window:
    effective_decay = base_decay × 0.5  # 保护期
elif connection_count >= survival_threshold:
    effective_decay = base_decay × (1 / (1 + connection_count))  # Hebbian 存活
else:
    effective_decay = base_decay × (1 / (1 + connection_count)) * 2.0  # 加速修剪
```

## 洞察

**成熟系统的关键期重开不是可塑性恢复，而是连接持久性的竞赛。跨领域散步的价值不在于"产生新 insight"，而在于"在可塑性窗口关闭前建立足够多的持久连接"。散步成功的标准需要从"insight 数量"更新为"insight 存活率"。**
