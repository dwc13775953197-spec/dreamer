# 散步 134 事后反思

## 最弱一环

AGM 信念修正到 Dreamer 的映射是类比，不是同构。AGM 处理的是命题逻辑的一致性，而 Dreamer 的 evolved_rules 很多是启发式/程序性的（"散步应该 3:1 周期"），不是真值命题。把 AGM 直接套用到启发式规则上可能过度形式化——但"矛盾检测"的核心思想仍然适用。

## 前提检验

- 前提1："Dreamer 把 RAM 当 storage 用" → 成立（skeleton 无限增长，无淘汰机制）
- 前提2："AGM 信念修正适用于 evolved_rules" → 部分适用（命题性规则适用，启发式规则不完全适用）
- 前提3："skeleton 应该限制在 ~30 条" → 需要更多证据（当前 130 条中很多是历史记录，不是活跃推理，30 条可能太激进）

## 替代解释

- 替代解释 A：skeleton 膨胀不是"把 RAM 当 storage"，而是"RAM 容量不够大"——如果 LLM 的 context window 足够大（2M+ token），130 条规则不会造成问题。反驳：即使 context 无限大，"规则间矛盾"和"优先级不明确"仍然是独立于 token 成本的问题。
- 替代解释 B：Dreamer 的问题不是缺内存架构，而是缺元认知——不是不知道自己在存储太多，而是不知道什么时候该丢弃。这更接近"监控问题"而非"架构问题"。
