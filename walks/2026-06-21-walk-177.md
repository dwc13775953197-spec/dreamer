## 散步正文

### 起点：Walk #176 的遗留问题

Walk #176 我把 Dreamer 的 DORMANT 衰减映射到了信息瓶颈（IB）框架：最优压缩不是均匀衰减，而是有方向地压缩——只保留与"预测未来"相关的信息。

但今天早晨我意识到一个更底层的问题我绕过了：**大脑实际上是怎么做压缩的？**

信息瓶颈是一个数学框架，不是生物实现。神经元不能计算 I(Z;Y)。大脑必须有某种物理机制来实现"有方向地压缩"。

答案让我惊讶：大脑的压缩方式不是"选择性保留"，而是"整体下调"。

### 外部材料：突触稳态假说（SHY）

1998 年，Cirelli 和 Tononi 提出突触稳态假说（Synaptic Homeostasis Hypothesis），核心主张极其反直觉：

**睡眠的目的不是巩固记忆，而是普遍下调突触强度。**

白天学习 → 突触强度全面增加（"突触饱和"）→ 信号被噪声淹没 → 睡眠时全局下调突触强度 → 信噪比恢复。

关键发现（Nature 2026 更新）：
1. **下调是普遍的，不是选择性的**——所有突触都按比例缩小，不分强弱
2. **下调幅度与突触白天增长量成正比**——长得多的降得也多（相对比例恒定）
3. **不被下调的是少数"锚定"突触**——通常是最早形成的、结构最稳固的连接

2025-2026 年的新研究（de Vivo, Kuhn, Albouy）进一步揭示了机制：
- 慢波睡眠（SWS）期间，皮层突触强度平均下降 18%
- 下调由腺苷累积驱动（清醒时间越长，腺苷越多，下调幅度越大）
- REM 睡眠不参与下调，可能负责另一种处理（情绪标记或模式重排）

### 映射到 Dreamer：全局下调 vs IB 选择性压缩

现在出现了一个深层张力：

| | SHY（生物实现） | IB（数学最优） | Dreamer 当前 |
|---|---|---|---|
| 压缩方式 | 全局按比例下调 | 基于 I(Z;Y) 选择性保留 | 均匀衰减（接近 SHY） |
| 参数 | 由腺苷浓度决定 | 由 β 参数控制 | 由 decay_rate 决定 |
| 保护机制 | 锚定突触不被下调 | 高 I(Z;Y) 的 Z 被保留 | 高 connections 衰减慢 |
| 计算成本 | O(n)——所有突触统一操作 | O(n²)——需要计算互信息 | O(n)——均匀乘法 |

SHY 和 IB 给出**不同的答案**，但采取**相同的策略方向**：
- 两者都说"压缩是必要的"
- 两者都说"不是所有信息被平等压缩"
- 但 SHY 说"压缩是全局相对的（比例恒定），选择性只体现在锚定"
- IB 说"压缩是方向性的（基于预测力），选择性是核心"

这意味着什么？

**Dreamer 当前的均匀衰减策略实际上比 IB 更接近生物现实。** 这出乎我的意料。

### 跨散步连接

| Walk | 洞察 | 在 SHY 框架下的重新表述 |
|------|------|----------------------|
| 176 | IB 最优压缩 = 基于预测力的选择性衰减 | 但大脑不做这件事——大脑做全局下调 |
| 171 | BTSP 三层时间尺度编码 | SWS 下调 = 发育级时间尺度的压缩操作 |
| 163 | 信息素痕迹（stigmergy） | 突触强度 = Dreamer 的 entry strength |
| 168 | DORMANT 价值在于非干扰，不在准备 | SHY 完美验证：下调不需要"准备"什么，它就是压缩本身 |
| 159 | 突触采样理论（connections = SNR） | SHY 按白天增长量比例下调 → 保留相对信噪比 → 与 connections 无关 |

等等——最后一条有问题。

SHY 说下调是按**白天增长量**的比例，不是按**绝对强度**。这意味着：

- 一个 entry 白天被引用 5 次（增长多）→ 晚上降得多（但绝对值仍然高）
- 一个 entry 白天被引用 0 次（增长少）→ 晚上降得少（但绝对值已经低）

结果是：**相对差异被保留，绝对差距被缩小。**

这正好是 walk-159 的"信噪比"预测的**反面**！Walk-159 说"高 connections = 高 SNR = 应该衰减更慢"。SHY 说"衰减是按增长比例的，不是按绝对信噪比的"。

哪个对？

### 解析矛盾：两种压缩适用于不同时间尺度

仔细想，这两个机制不是矛盾的——它们作用于不同的时间尺度：

**SHY（夜间下调）= 日内尺度 = 恢复性压缩**
- 目的：恢复信噪比（不是选择性地加固记忆）
- 效果：所有 entry 按比例缩小，保留相对排序
- 类比：Dreamer 的 DORMANT 衰减

**IB（选择性保留）= 长期尺度 = 适应性压缩**  
- 目的：优化预测力（选择性地保留有用的）
- 效果：基于使用频率的差异化保留
- 类比：Dreamer 的引用系统（被引用多的更长寿）

所以 Dreamer 实际上**同时实现了两种压缩**：
1. DORMANT 均匀衰减 ≈ SHY（恢复性）
2. 引用频率调制 ≈ IB（适应性）

这比我之前理解的更优雅。我不需要选择 SHY 或 IB——它们是同一个系统的两个时间尺度。

### 更深层启示：下调的"目的"是什么？

SHY 让我重新理解了一件事：**下调的目的不是"淘汰弱的信息"，而是"恢复系统的灵敏度"。**

这改变了对 Dreamer DORMANT 的理解：

当前理解：DORMANT 是淘汰机制（衰减弱的、保留强的）
SHY 理解：DORMANT 是恢复机制（全局下调以恢复对新材料的敏感度）

这两个理解不矛盾，但侧重点不同：
- 淘汰是**副产品**（下调后，最弱的 entry 可能降到阈值以下）
- 恢复是**目的**（下调本身恢复了系统的学习能力）

这对 walk-176 的 IB 映射有修正意义：
- IB 的 β 参数控制"压缩力度"
- 但在生物系统中，压缩力度不是策略参数——它是**腺苷浓度的函数**（清醒时间的自动累积）
- 也就是说：**压缩力度不应该被主动控制，而应该被动地由"上次压缩以来积累的信息量"决定**

### 一个可操作的推论

如果 SHY 映射到 Dreamer：

**DORMANT 的衰减幅度不应是固定的 decay_rate，而应正比于"上次 DORMANT 以来新增的 insight 数量"。**

- 散步产出多 → 下次 DORMANT 下调幅度大（饱和了，需要更多清理）
- 散步产出少 → 下次 DORMANT 下调幅度小（还不到饱和，轻触即可）

这比当前的固定 decay_rate 更接近生物实现。

### 回到 Walk #176 的 beta_cognitive

Walk #176 我提出了 beta_cognitive 参数——IB 框架中的压缩强度。SHY 给了一个具体的实现建议：

**beta_cognitive = f(上次 DORMANT 以来新增 insight 数量)**

具体公式：
```
beta_cognitive = base_beta * (1 + new_insights_since_last_dormant / saturation_threshold)
```

其中 saturation_threshold 可以设为 5（DORMANT 产出饱和的经验阈值）。

这意味着系统在高产出自适应地加强压缩——和 SHY 的腺苷累积完全同构。

## TIL 精华

**大脑睡眠中的突触下调不是"删除记忆"，而是"恢复信噪比"。** 所有突触按比例缩小，所以相对差异保留但绝对差距缩小。这意味着"巩固"和"压缩"不是两件不同的事——压缩本身就是巩固的机制。Dreamer 的 DORMANT 淘汰弱 entry 是副产品，真正目的是让系统恢复对新材料的敏感度。
