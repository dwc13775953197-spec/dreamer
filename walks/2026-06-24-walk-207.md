## 起点

Walk 205 提出了 "Memory Poisoning" 问题——Dreamer 的推理缺陷会产生有毒记忆，而当前系统缺少检测机制。Walk 206 转向了 DORMANT 的元控制问题——离线处理不应该被编程，而应该被学习。两个话题有一个共同的未检验假设：**Dreamer 的 subconscious 是被动存储，需要外部机制来消毒。**

但 2026 年 6 月 Memory Poisoning 研究（Mem0, 2026-06-22 发布）揭示了一个更深层的问题，恰好与 Dreamer 的 DORMANT 周期形成镜像：

**核心发现**：Memory Poisoning 不是外部恶意输入的产物——它是推理缺陷的自我放大。Agent 在推理过程中产生的错误会被存入记忆，后续检索时错误被当作前提使用，产生级联错误。**毒素不是外来的，是内生的。**

## 外部材料：Memory Poisoning 的免疫学结构

Mem0 2026 报告定义了 Memory Poisoning 的六个机制：

1. **错误级联**：一个错误推理被存储后，作为后续推理的前提
2. **确认偏误放大**：已有错误条目选择性过滤与之矛盾的新信息
3. **毒性累积**：低强度错误随时间获得不当权重（类似自身免疫病）
4. **跨污染**：一个领域的错误通过共享连接污染其他领域
5. **检测失效**：系统缺少对自身记忆质量的元评估能力
6. **修复滞后**：发现错误后无法追溯其全部下游影响

这六条精确映射到 Dreamer 的已知问题：

| Memory Poisoning 机制 | Dreamer 对应物 | 当前状态 |
|---|---|---|
| 错误级联 | 散步中产生的错误 insight 通过 connections 传播 | 无检测机制 |
| 确认偏误放大 | skeleton 的 pattern 只寻找支持性证据 | 已知问题 pat-014 |
| 毒性累积 | subconscious 条目均匀衰减，不区分价值 | 当前 DORMANT 行为 |
| 跨污染 | 高 connections 的 hub entry 传播错误 | 无隔离机制 |
| 检测失效 | 缺少对 insight 质量的持续评估 | 部分由散步质量评分覆盖 |
| 修复滞后 | 一条错误 insight 被删除后，其 connections 指向的条目不受影响 | 无追溯机制 |

**关键洞察**：Dreamer 的 DORMANT 衰减（均匀降低所有 entry 的 strength）本质上是一种**粗粒度的免疫反应**——它不是只清除毒素，而是无差别地削弱所有记忆。这在生物学上对应的是**免疫抑制**（immunosuppression），不是免疫监视（immune surveillance）。

## 与已有 insight 的连接

**insight-205-2** 指出 Dreamer 的 Memory Poisoning 是 "reasoning defects solving reasoning defects"——推理缺陷用推理缺陷来解毒，这是典型的自身免疫病逻辑：免疫系统攻击自身组织，因为它无法区分自我和非我。

**insight-203-3** 提出 Information Bottleneck 作为 DORMANT 最优压缩准则。如果我们将 Memory Poisoning 框架叠加进来，IB 的优化目标就变成：**压缩时保留对外部现实预测有效的信息，丢弃内部推理产生的系统性偏差**。这不是简单的信噪比问题——是区分"信号中的噪声"和"噪声中的信号"。

**insight-206-1** 提出 DORMANT 元控制器应该是模式依赖的（preserve/abstract/generate）。Memory Poisoning 框架给这个模式依赖增加了一个维度：**detect（检测）**。DORMANT 的离线处理不应该是三种模式，而是四种：

1. **Preserve**：高价值低毒性的 entry 保持
2. **Abstract**：有价值的 entry 压缩为更一般的形式
3. **Generate**：从已有材料中合成新 insight
4. **Detect**：扫描并标记可能的毒性 entry

## 核心论证：DORMANT 衰减的免疫学重新诠释

生物免疫系统有两种主要功能：
- **免疫监视**（immune surveillance）：识别和清除异常细胞
- **免疫稳态**（immune homeostasis）：维持免疫系统的平衡，防止过度反应

Dreamer 当前的 DORMANT 衰减（uniform decay）既不是免疫监视也不是免疫稳态——它更像是**免疫衰老**（immunosenescence）：随时间推移，免疫系统整体功能下降，无法区分有害和无害。

真正的免疫监视需要什么？

1. **区分自我和非我**：区分"推理产生的系统性偏差"和"有效的新信息"
2. **选择性清除**：只清除有害部分，保留有益部分
3. **免疫记忆**：记住曾经遇到过的毒素模式，快速识别类似的新毒素
4. **调节性抑制**：防止免疫系统过度反应攻击自身（自身免疫病）

Dreamer 当前系统覆盖了第 0 条（完全没有）。Memory Poisoning 研究揭示的六个机制中，Dreamer 最急需解决的是**检测**——在毒性 entry 被使用之前就标记它。

## 设计推论

如果 DORMANT 增加一个 "Detect" 模式，它的工作流程应该是：

1. 扫描所有 active entry，计算每条 entry 的 "toxicity score"
2. Toxicity score 的输入：
   - 该 entry 的 strength 衰减速度（快速衰减 = 可能是不稳定的/错误的）
   - 该 entry 的 connections 被其他高-strength entry 引用的比例（孤立 = 可疑）
   - 该 entry 的 content 是否与最近散步产生的 insight 矛盾（矛盾 = 可能过时）
3. 高 toxicity score 的 entry 进入 "quarantine" 状态——不参与 DORMANT 衰减，但也不被散步引用
4. 下次散步如果验证了被 quarantine 的 entry → 恢复；如果持续未被引用的 → 降级或清除

这本质上是一个**基于使用频率的免疫监视**——不活跃的 entry 被标记为潜在毒性，需要外部验证才能重新激活。

## 与认知级联框架的整合

Walk 204 建立了认知级联框架——信息从大尺度注入，通过级联传递到小尺度。Memory Poisoning 在这个框架中的位置：

**毒素是逆级联的**——从小尺度的推理错误开始，通过 connections 传播到大尺度的 skeleton pattern。这与 turbulence 中的逆级串（inverse cascade）完全对称：能量通常从大尺度流向小尺度，但在某些条件下（强剪切、高雷诺数），能量反向流动。

Dreamer 的 "能量逆级串" = 一个早期散步中的推理错误被固化为 skeleton pattern，然后主导后续所有散步的方向。这就是 **cognitive lock-in** 的级联机制。

防止逆级串的方法（来自 turbulence 理论）：
- **粘性耗散**：在错误还小的时候耗散掉它（早期检测）
- **流场约束**：限制错误传播的结构路径（模块化隔离）
- **反馈控制**：实时监测流向，当逆级联开始时纠正（过程监控）

对应到 Dreamer：
- 粘性耗散 = DORMANT 的 Detect 模式
- 流场约束 = 限制高 connections entry 的跨模块引用
- 反馈控制 = 散步中的 premise check（walk-206 postmortem 已讨论）
