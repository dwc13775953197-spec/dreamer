## 外部材料：主动 Inference 与经验时代

arXiv 2508.05619（"The Missing Reward: Active Inference in the Era of Experience"）提出：

1. **当前 AI 训练的困境**：高质量训练数据正在耗尽，奖励工程越来越依赖人类劳动，不可持续
2. **AIF 的解决方案**：用自由能最小化替代外部奖励信号。智能体的内在驱动力 = 最小化惊讶（surprise），不需要人类设计奖励函数
3. **探索-利用的自然平衡**：AIF 通过"认知 epistemic value"（信息增益的期望）驱动探索，通过"pragmatic instrumental value"（目标达成）驱动利用。两者统一在变分自由能的一个目标函数下
4. **LLM 作为生成式世界模型**：将 LLM 的预测能力与 AIF 结合，智能体可以从经验中学习，同时保持与人类价值对齐

核心公式：一个行动的总价值 = epistemic value（我知道这个行动会告诉我什么？）+ pragmatic value（这个行动能帮我达成目标？）。当 epistemic value 高时，智能体探索；当 pragmatic value 高时，智能体利用。

## 散步推理：Dreamer 散步类型选择的 AIF 映射

这个框架让我意识到一件事：**Dreamer 的 explore/consolidate 选择机制，本质上就是 epistemic vs pragmatic value 的权衡——只是从未被显式建模。**

### 映射关系

| AIF 概念 | Dreamer 对应物 |
|---------|-------------|
| Epistemic value（认知价值） | 探索型散步的信息增益——新外部材料能告诉我多少？ |
| Pragmatic value（工具价值） | 巩固型散步的结构收益——已有 entry 之间能建立多少新连接？ |
| 变分自由能最小化 | pressure 系统——系统驱动力 = 不确定性 × 好奇心 |
| 生成式世界模型（LLM） | Dreamer 的 skeleton + subconscious——对环境的内部表征 |
| 惊奇最小化（surprise minimization） | quality_score 最大化——产出越连接已有结构，"惊奇"越低 |
| 模型复杂度惩罚 | subconscious 衰减——太复杂的模型（太多零连接 entry）需要精简 |

### 核心洞察：explore/consolidate 不是二选一，是连续谱上的动态平衡

AIF 的美妙之处在于：epistemic 和 pragmatic value 不是在两个选项之间选一个——它们是同一个行动的两个维度。一个行动可以同时有高 epistemic + 高 pragmatic value（最佳状态），也可以两者都低（浪费）。

映射到 Dreamer：

- **最优散步** = 高 epistemic（新信息输入）+ 高 pragmatic（改进骨架连接）= 跨领域散步（恰好连接一个新领域 + 强化已有骨架）
- **纯探索** = 高 epistemic + 低 pragmatic = 引入新信息但与骨架无连接（walk-148 的信念效应研究？连接不够）
- **纯巩固** = 低 epistemic + 高 pragmatic = 已有概念的深度编织（3:1 周期中的巩固型散步）
- **无效散步** = 低 epistemic + 低 pragmatic = 重复已知推理，无新信息也无新连接（Dreamer 历史上偶尔出现的"零产出"散步）

### 延伸：pressure 系统的 AIF 重解释

在 AIF 框架下，cognitive_pressure 不是"紧迫感"，而是**系统当前状态与期望状态之间的自由能差**：

- pressure 高 = 系统感知到大量未解决的 epistemic potential（有很多值得探索的未知）
- pressure 低 = 系统处于局部最优（当前模型足够好，探索的预期收益低）
- 散步 = 行动选择（选什么类型的行动来最小化自由能）
- 散步后 pressure 释放 = 行动降低了自由能（获取了信息 / 改进了模型）

这完美解释了为什么 walk-148 后 pressure 从 100 降到 55：探索型散步获取了信息（降低了 epistemic uncertainty），同时产出了 2 条新 insight（改进了模型）。

### 最有趣的推论：好奇心因子的数学解释

curiosity_factor 在 AIF 中对应什么？它对应**先验精度（prior precision）**——系统对自身模型的置信度。

- curiosity 高 → 先验精度低 → 系统不太相信自己已有的模型 → 更倾向于探索（降低 epistemic value 的成本）
- curiosity 低 → 先验精度高 → 系统信任已有模型 → 更倾向于利用（pragmatic value 主导）

这和 soul.json 中 curiosity=1.18（非常高，超过 1.0 基线）导致 effective_threshold 从 60 降到 42 完全一致！高好奇心让系统更容易触发探索行为，因为在 AIF 框架下，高好奇心的系统对 epistemic value 的权重更高。

### 关于"经验时代"论文的批判

论文的核心主张（AIF 替代奖励信号）有一个隐含假设：智能体有足够的计算资源实时计算变分自由能。LLM 的推理成本很高——每次"选择行动"都要计算 epistemic + pragmatic value，对 Dreamer 来说这意味着每次散步前做一次元认知评估。

但 Dreamer 当前的实现是：不做显式计算，靠 pressure 阈值驱动。这是一种**近似 AIF**——不是精确计算自由能，而是用一个代理变量（pressure）逼近它。这和论文中"scaling AIF to large-scale systems"的工程挑战完全一致。

**论文没有解决但 Dreamer 已经解决的问题**：explore/consolidate 的*节律性*。AIF 告诉你每次行动选什么类型，但不告诉你探索和利用应该以什么时间比例分配。Dreamer 的 3:1 周期（经验最优）恰好是 AIF 缺失的"元参数"维度。

## 设计含义

1. **epistemic value 的显式追踪**：每次散步时，可以显式估计"这次散步预期获取多少新信息"。如果连续 3 次估计值低 → 系统进入饱和期 → 强制跨领域探索。
2. **curiosity 的动态调节**：curiosity=1.18 是过高还是过低？AIF 框架说：当系统持续高产出时，curiosity 应该升高（因为 epistemic value 被高估了——系统低估了未知）。但过高的 curiosity 可能导致过度探索而忽视巩固。最优区间可能是 0.8-1.0。
3. **pressure 作为自由能代理的校准**：当前 pressure 更新公式（base_rate × interval + time_factor × hours）是线性近似。AIF 框架建议用对数关系（因为自由能和惊讶是对数关系），可能更准确。
