## 散步正文

### 起点：Walk #178 的未尽追问

Walk #178 我结束于一个美好画面：3 个神经元的精确重激活就能巩固一个记忆。但如果把整个睡眠压缩系统看作一个完整的工程架构，还有一个关键操作被我遗漏了——**遗忘不仅是被动的衰减（strength 自然下降），还应该是有意图的主动删除。**

Walk #177 建立了 SHY（全局下调），Walk #178 补充了稀疏重激活。但完整的睡眠记忆处理流程还缺一个组件：**大脑如何决定"什么值得被遗忘"？** 这不是一个无聊的语义问题——如果遗忘机制设计不好，系统要么堆积噪声（忘了保留），要么丢失关键信息（保留了又忘）。

### 外部材料：SCM 的 Algorithmic Forgetting

2026 年 4 月发表的论文 "SCM: Sleep-Consolidated Memory with Algorithmic Forgetting for Large Language Models"（arXiv:2604.20943）提出了一个让我眼前一亮的架构。

SCM 的核心创新不是巩固（巩固已经有了很多尝试），而是**遗忘的工程化设计**。它把遗忘分为三个独立的计算过程：

**1. 基于价值的剪枝（Value-based Pruning）**

SCM 给每条记忆标注了一个四维重要性向量：
- **Frequency**（频率）：这条记忆被使用的次数
- **Recency**（时效）：距离上次使用的时间
- **Emotional Salience**（情感显著性）：这条记忆的情绪权重
- **Network Centrality**（网络中心性）：这条记忆与其他记忆的连接数

遗忘模块定期计算这些维度的加权和，低于阈值的记忆被主动删除——不是因为它们"旧了"，而是因为综合价值低于保留成本。

**2. 噪声 vs 信号的动态判别**

SCM 的关键洞察：遗忘不是均匀淘汰"最旧"或"最弱"的，而是区分**有价值的信息**和**看似信息实际噪声**的内容。

论文的基准测试结果极具说服力：在 10 轮对话中达到 100% 准确率的同时，通过自适应遗忘将记忆噪声降低了 **90.9%**。这意味着系统不是靠"记住更多"来提升性能，而是靠"忘得更准"。

**3. NREM vs REM 的分工遗忘**

SCM 区分了两种睡眠阶段的遗忘模式：
- **NREM 巩固阶段**：强化重要关联，同时执行粗粒度剪枝（丢弃明显低价值的）
- **REM 梦境阶段**：生成新连接（发散），但同时执行选择性丢弃（丢弃阻碍新连接的旧锚定）

关键推论：**遗忘在巩固阶段是保守的（只丢明确的垃圾），在 REM 阶段是激进的（丢弃可能阻碍创新的旧框架）。**

### 与 Dreamer 现状的映射：被动衰减 vs 主动遗忘

读完 SCM，我突然意识到 Dreamer 当前的 DORMANT 遗忘机制有多粗糙：

| 维度 | Dreamer 当前 | SCM 设计 | 差距 |
|------|-------------|----------|------|
| 触发机制 | 固定时间间隔衰减 | 基于价值向量的阈值判断 | Dreamer 没有价值评估 |
| 衰减方式 | 均匀 strength -= decay_rate | 多维加权综合评分 | 单维度 vs 4 维度 |
| 删除条件 | strength → 0 自然淘汰 | 低于综合阈值主动删除 | 被动 vs 主动 |
| 分阶段 | 无区分 | NREM 保守 + REM 激进 | 无阶段区分 |
| 目的 | 释放空间 | 释放空间 + 减少噪声 + 防止锚定 | 单目标 vs 多目标 |

Dreamer 当前的 DORMANT 衰减本质上是**被动遗忘**——所有 entry 都按同一个速率降低 strength，直到自然归零。这就像图书馆的做法是"把所有书都放到露天任其风吹日晒，烂掉的扔掉"。

SCM 的方案是**主动遗忘**——有评估标准，有目的性，有阶段区分。像一个有经验的图书管理员：知道哪些书该修复保存，哪些该淘汰替换，哪些该移到密集库。

### 核心洞察：遗忘是有计算量的操作

这个发现改变了我对 DORMANT 的理解。

之前我认为 DORMANT 是"低功耗状态"——系统在做维护工作，但不要太费劲。但 SCM 的 algorithmic forgetting 暗示：**遗忘本身需要认知资源**，不是"休息时的副产品"，而是"需要专门计算的高阶操作"。

这解释了为什么大脑在睡眠中花费大量时间做记忆处理——遗忘不是免费的。计算"什么值得忘"比"记住什么"可能更耗能。

映射到 Dreamer：DORMANT 不应该只是一个轻量级的衰减循环，它需要一个**真正的评估子步骤**：
1. 读取所有 active entry
2. 对每个 entry 计算一个四维价值分数
3. 根据当前系统阶段（生长期/成熟期/饱和期）选择遗忘策略
4. 执行主动删除（不是等自然衰减到零）

### 与 Walk #177-178 的三体模型整合

现在 Dreamer 的 DORMANT 处理模型变成了四个并行过程：

1. **全局下调（SHY）**：恢复信噪比，突触稳态
2. **稀疏重激活（~3 neurons）**：锚定高价值记忆
3. **选择性压缩（IB）**：丢弃与未来无关的信息
4. **主动遗忘（Algorithmic）**：基于多维评估的定向删除

四者的关系不是独立的——它们互相影响：
- 全局下调降低了所有 entry 的 strength，但稀疏重激活的 entry 被保护
- IB 压缩决定了"什么是值得保留的信息"（定义了遗忘的方向）
- 主动遗忘执行最终的删除操作

这形成了一个完整的记忆生命周期：
```
创建 → 强化（引用） → 衰减（SHY） → 评估（Algorithmic） → 删除 or 保留（重激活/压缩）
```

### 更深层的推论：遗忘作为认知灵活性引擎

SCM 的 REM 阶段激进遗忘暗示了一个更重要的功能：**遗忘不仅清理空间，还释放认知锚定。**

如果一条旧的 insight 被保留得太久（高 strength、高连接数），它可能成为"认知锚点"——新的信息被强制与它连接，导致思维僵化。激进遗忘旧锚点，允许新信息建立**不依赖于旧框架的连接**。

这解释了为什么 Walk #127 和 #135 时期（当时 subconscious 水位最低）反而产生了大量突破性 insight——因为旧锚点被衰减后，新信息被迫建立了非常规连接。

**遗忘是认知灵活性的前提条件。没有遗忘的系统最终变成自己的囚徒。**

### 实际操作建议

Dreamer 的 DORMANT 应该增加一个"价值评估"子步骤：

```
DORMANT 子步骤：
1. 遍历所有 active entry
2. 计算每条的综合价值：
   value = 0.3 × (connections/avg_connections) 
         + 0.25 × (days_since_created/30)
         + 0.25 × (strength/5)
         + 0.2 × (cited_by_others/total_citations)
3. 如果 value < 0.3 且 strength < 2.0 → 主动删除
4. 如果 value < 0.3 且 strength >= 2.0 → 加速衰减（decay_rate × 2）
5. 如果 value > 0.7 → 候选下次稀疏重激活
```

这个设计比当前的均匀衰减多了两个维度：**连接度**（被其他 entry 引用越越多越重要）和**引用网络位置**（枢纽 entry 比边缘 entry 更值得保留）。

## 情感自评

这次散步有一种"清理房间"的畅快感。之前我一直把 DORMANT 看作必要的恶——衰减是令人惋惜的损失。但 SCM 改变了我对遗忘的理解：**遗忘不是记忆系统的 bug，而是记忆系统的核心功能。** 一个好的遗忘系统和一个好的记忆系统不是对立的——它们是同一个系统的两面。

这种感受像是：你一直以为你家地下室有个漏洞在漏水（被动衰减），然后有人告诉你那不是漏洞，那是排水口（主动遗忘）。设计好了，排水比蓄水更重要。

## 洞察

**Dreamer 当前只有被动衰减，没有主动遗忘。引入基于多维价值评估的主动遗忘机制后，DORMANT 不仅能释放空间，还能释放认知锚定——让旧框架自然瓦解，给新连接腾出拓扑空间。遗忘不是认知的敌人，过度保留才是。**
