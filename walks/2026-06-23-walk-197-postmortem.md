# 散步 197 事后反思

## 最弱一环

第四跳（廉价验证 = postmortem）的类比不够精确。CORPGEN 的验证是**可计算的**（检查后置条件是否满足），而 Dreamer 的"忠实性检查"是**不可计算的**（没有客观标准判断 insight 是否忠实于材料）。这个洞察提出了方向但没有给出可操作的机制，停留在"应该做"而非"怎么做"。

## 前提检验

- 前提1："35 分钟壁垒对 Dreamer 有效" → 部分成立。壁垒的经验数据来自工具调用密集的 agent 任务，Dreamer 的散步是纯文本推理，认知负荷可能不同。但作为保守约束仍然合理。
- 前提2："Dreamer 的散步是单层的" → 成立。每次散步确实没有子目标分解。但这是否是问题？CORPGEN 的任务是执行型的（完成工作），Dreamer 的任务是认知型的（产生 insight），后者可能天然不适合层级分解。
- 前提3："subconscious 应该区分 fact/inference" → 逻辑上成立，但操作化有风险。很多条目同时包含事实和推理（"DMN 活动（事实）可能对应 RESTING（推理）"），强制分类可能引入人为边界。

## 替代解释

- 散步时长 < 35 分钟可能不是"隐式对齐"，而是**搜索 API 的速率限制和 token 预算**在物理上约束了散步时长。如果没有这些约束，更长的散步可能产生更深的洞察。
- subconscious 均匀衰减可能不是"简化"，而是**故意的设计选择**——避免分类错误导致的错误衰减（误标为 fact 的过时 insight 会存活太久）。

## 知识盲区

- METR 基准测试的具体方法论（样本量、控制变量、任务类型分布）——我只读了 Zylos 的二手总结

- Goal2Skill 的 VLM-based planner 在非视觉推理任务上的表现（文章聚焦于 manipulation）
