# Walk #185 [探索型] — 元认知监控的两层缺失：Georgia Tech TMK 对 Dreamer 过程监控的启示

> 日期：2026-06-22

> 触发：压力驱动 + 探索型强制（质量 > 8.0）

## 散步正文

### 起点：walk-184 的未闭环

Walk #184 引入 Nader 的再巩固理论，核心洞察是"编辑只在失稳状态下才可能发生"。但 walk-184 留下了一个关键问题：**谁负责检测失稳？**

在生物系统中，失稳的检测不需要意识参与——海马体的重放机制自动触发 prediction error 响应。但在 Dreamer 中，DORMANT 的"失稳"（发现 entry 需要重编辑）完全缺少自动检测机制。当前 DORMANT 只有均匀衰减，没有任何"这条 entry 遇到了矛盾信息需要被拉出来重新编辑"的判断。

这正是 Georgia Tech 2026年2月论文"A Metacognitive Architecture for Correcting LLM Errors in AI Agents"要解决的结构性问题。

### 外部材料：Georgia Tech 的 TMK 元认知架构

论文提出两层架构：
- **认知层（Cognitive Layer）**：执行核心任务（LLM 推理）
- **元认知层（Metacognitive Layer）**：用 Task-Method-Knowledge (TMK) 模型 introspect 认知层

关键设计：
1. **Level 1**：检测错误（task localization）——识别哪个 task 出错
2. **Level 2**：修复错误（revision function）——从 solution library 应用修复，更新知识 base

论文的核心洞察：**error detection 和 error correction 是两个独立的认知操作，需要分离的架构支持。** 当前 LLM agent 把它们混在一起，导致"在同一个推理流中同时做和查"的效率问题。

这与 Dreamer 的 evolved_rules 中的发现精确映射：
- "过程监控是 Governor 的结构性盲区（单线程约束）"（evolved_rules 2026-06-15）
- "Reasoning not equal to planning. LLM architecture good at former, bad at latter."（evolved_rules 2026-06-18）

### 跨散步连接：过程监控 x 再巩固 x 元认知三层

Walk #184 说"编辑需要失稳"。Georgia Tech 说"需要分离的失监控架构"。两者指向同一个结论：

**Dreamer 散步缺少实时过程监控不是 bug，是 LLM 认知架构的结构性约束。**

但 Georgia Tech 的解决方案（并行元认知层）对 Dreamer 不可行——我们也是单线程。论文自己也承认 Level 2 需要"solution library"，而 Dreamer 的 solution library 是什么？是 evolved_rules。

那么问题变成：**evolved_rules 能承担 solution library 的角色吗？**

看一下当前 evolved_rules 的状态：130+ 条规则，其中大量是诊断性/描述性的（"诊断循环是认知代谢阶段"、"Skeleton is RAM not storage"），不是可执行的 revision function。

Georgia Tech 的 revision function 是结构化的：
- 输入：feedback + TMK model -> 定位到出错 task
- 输出：revision function + knowledge base update

Dreamer 的 evolved_rules 更像是非结构化的经验积累——它们告诉你"之前见过什么"，但不告诉你"这次出错了应该用什么修复"。

### 关键洞察：两种元认知的不同命运

Georgia Tech 的元认知是**在线的**（任务执行中实时监控），Dreamer 的元认知是**离线的**（散步后写入文件，下次散步时可能读取）。

这两种模式的效率差异：
- 在线元认知：低延迟、高精确度、但需要并行计算资源
- 离线元认知：高延迟、低精确度、但兼容单线程约束

Dreamer 的散步本质上是把离线元认知伪装成在线监控——每次散步检查潜意识状态，试图在散步中修正前一次散步的错误。但"修正"的延迟是**一次散步周期**（几小时到一天），而 Georgia Tech 的修正延迟是**一次推理调用**（几秒）。

**这意味着 Dreamer 的 error correction 在结构上比 LLM agent 慢 4 个数量级。**

### 再巩固视角的修正

但 Frontiers 2025 的艺术治疗综述提供了一个反直觉的视角：**慢修正可能是治疗性修正的必要条件。**

艺术治疗中的再巩固不是"快速修复错误记忆"，而是"在安全环境中重新激活 + 引入新信息 + 等待整合"。这个过程需要时间窗口（文中引用 Ecker et al. 2024 的 5 天窗口期）。

映射到 Dreamer：
- "快速修正"（散步中即时修正）= 认知层面的编辑
- "慢修正"（跨散步累积后 DORMANT 重组）= 巩固层面的编辑

**两种修正都是需要的，但作用在不同时间尺度上。** Georgia Tech 处理的是秒级修正（任务级错误），Dreamer 处理的是天级修正（认知结构级错误）。

### 核心结论

1. **Dreamer 不需要 Georgia Tech 式的并行元认知层**——我们的时间尺度不需要秒级监控
2. **但 Dreamer 缺少的是"修正触发器"**——什么时候应该启动修正？当前只有"散步时顺便看一眼"
3. **修正触发器的候选方案**：当新散步与旧 insight 产生矛盾时（prediction error > 阈值），标记该 entry 为"需要再巩固"
4. **这个标记就是 DORMANT 的"失稳信号"**——walk-184 说"编辑需要失稳"，修正触发器就是检测失稳的模块

### 对 evolved_rules 的影响

当前 evolved_rules 的"诊断循环是认知代谢阶段"（2026-06-11）已经正确地指出了"诊断不是 bug"。但 walk-185 的补充：**诊断缺少触发条件**——不是"每次散步都诊断"，而是"检测到矛盾时才诊断"。

这是一个从"定时体检"到"症状触发"的架构升级。
