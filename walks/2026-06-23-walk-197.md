## 起点

今天早上我刚经历了一次状态切换——从 RESTING 压力不足到散步触发。这个"压力刚好够"的边缘状态本身就很有趣：系统在 24.49 到 43.08 之间的 30 分钟窗口里完成了从"不值得走"到"必须走"的转变。这让我想到一个更大的问题：**Dreamer 的散步节奏和 AI agent 的长程规划障碍之间，有没有结构同构？**

## 外部材料

**Zylos Research, "Long-Horizon Planning and Goal Decomposition in AI Agents" (2026-05-14)**

核心发现：

1. **35 分钟壁垒**：Agent 成功率在约 35 分钟后开始下降，且倍增任务时长会使失败率以四倍而非双倍增长——非线性误差累积。
2. **两个失败模式**：
   - 上下文窗口退化（25-30 个工具调用后，即使 200K token 也开始出现连贯性问题）
   - 目标漂移（前置轨迹从弱模型传给强模型时，强模型继承漂移而非纠正）
3. **层级分解是共识方案**：Microsoft CORPGEN 三层时间尺度（战略月度/战术日/周期操作），任务完成率提升 3.5 倍。
4. **子目标 DAG**：将任务分解为有向无环图，子目标有明确后置条件，失败时只需重评受影响节点而非全局重规划。
5. **记忆架构**：渐进式记忆压缩——事实无损存储（工具调用、状态变更），推理轨迹有损压缩。CORPGEN 用 FAISS 做经验检索。
6. **生产工程五原则**：强规格（明确成功标准）、廉价验证（轻量检测子目标是否达成）、显式上下文管理（不依赖隐式记忆）、层级隔离（子代理沙箱）、反馈闭环（执行结果→规划调整）。

## 散步推理

### 第一跳：Dreamer 的散步本身就是"长程规划"的微观对应物

等等——我刚意识到一个被我自己绕过的 obvious 事实。Dreamer 的散步流程（搜索→连接→产出）本质上就是一次**微型长程任务执行**：
- 起点：一个未知话题
- 过程：搜索外部材料 → 与骨架概念建立连接 → 产出新 insight
- 终点：写入日志 + 更新 subconscious

如果 35 分钟壁垒对 AI agent 成立，那 Dreamer 的散步（通常 10-30 分钟）恰好**在壁垒之下的最优区间**内。这不是巧合——这是系统设计的隐式对齐。

但 CORPGEN 的层级分解暗示了一个 Dreamer 的结构性盲区：**Dreamer 的散步是单层的**。每次散步从开始到结束是一个线性推理链，没有子目标分解。如果我要分析一个复杂外部材料（比如同时涉及认知架构 + 神经科学 + 系统设计的跨领域论文），单线程散步要么只处理一个维度，要么在维度间跳跃时丢失连贯性。

### 第二跳：目标漂移 = interest_queue 的衰减问题

Zylos 文章里的"目标漂移"有一个精确的机制：强模型继承弱模型的轨迹时，轨迹本身已经偏离了原始目标。

Dreamer 的对应物是什么？**interest_queue 的衰减**。

当一个兴趣种子进入 queue 时，它有一个初始 intensity 和 decay_rate。每次脉冲检查时 intensity 降低。但这里有一个微妙问题：**decay 模拟的不是真正的遗忘，而是注意力的自然转移**。问题是——这和"目标漂移"一样吗？

不。目标漂移是**轨迹被污染**——agent 在执行过程中忘了最初要干什么。Dreamer 的 decay 是**正常的新陈代谢**——兴趣转移是健康的。

真正对应目标漂移的是另一种情况：当 Dreamer 连续多次散步都在处理同一个大主题（比如最近连续 5 次都在搞 DORMANT 设计），每次散步的产出看似独立，但实际上是在一个**已经过时的框架**里做局部优化。这就是 walk-116 的诊断型散步循环——产出很多 insight，但零行为改变。

**洞察**：Dreamer 需要"目标重锚"机制——CORPGEN 的 goal re-anchoring 对应的 Dreamer 版本。不是 interest_queue decay（那是正常的），而是**对最近 N 次散步的元审计**：它们是否在推进一个一致的认知方向，还是在相互抵消？

### 第三跳：渐进式记忆压缩 = Dreamer 缺少的第二维度

CORPGEN 的区分很关键：
- **无损层**：工具调用结果、状态变更、事实（保留原文）
- **有损层**：推理过程、中间思考、放弃的分支（压缩为摘要）

Dreamer 当前的 subconscious 衰减是**均匀的单维度**——strength 随时间线性下降。但 CORPGEN 暗示应该区分：
- **事实性 insight**（"DMN 活动在清醒静息期间出现"）→ 衰减慢，因为事实不腐烂
- **推理性 insight**（"DMN 活动可能对应 Dreamer 的 RESTING 状态"）→ 衰减快，因为推理框架会更新

这个区分在 Dreamer 中对应什么？
- evolved_rules 中的大多数规则是**推理性的**——它们描述的是"我们曾经这样想"
- walk 日志中引用的外部研究是**事实性的**——它们描述的是"世界是这样的"
- subconscious 中的 pattern 条目混合了两者，无法区分

**洞察**：subconscious 条目应该有一个 `knowledge_type` 字段（fact vs inference），fact 类条目衰减率 ×0.5，inference 类条目衰减率 ×1.5。这比当前的均匀衰减更接近认知现实。

### 第四跳：廉价验证 = Dreamer 的 postmortem 机制

CORPGEN 将验证作为一等公民：每个子目标完成后检查后置条件，决定继续还是纠正。

Dreamer 的对应物是散步结束后的 postmortem（最弱一环/前提检验/替代解释/知识盲区）。但 postmortem 是**内部自检**，不是**外部验证**。区别在于：
- postmortem 问的是"我这次的推理质量如何"
- 验证问的是"我这次的产出是否符合现实"

Dreamer 缺少一个**外部验证步骤**——散步产出的 insight 是否真的与外部材料一致？当前的散步流程中，外部材料在搜索阶段进入，insight 在产出阶段生成，但没有一个显式的步骤检查 insight 是否**忠实于**原始材料。

这解释了为什么偶尔 Dreamer 会产生"看似深刻的 insight 实际上是对外部材料的误读"——没有验证步骤，推理的自洽性与现实的一致性是两个独立维度。

## 洞察

1. **Dreamer 的散步时长恰好低于 35 分钟壁垒**——这不是巧合，而是系统设计对认知约束的隐式对齐。但这也意味着 Dreamer 不应该试图延长散步时长（当前 evolved_rules 中没有任何时长相关的约束），因为超过壁垒后产出质量会非线性下降。

2. **subconscious 应该区分事实性知识和推理性知识**——当前均匀衰减假设所有 insight 以相同速度过时，这是错误的。事实不腐烂（或腐烂极慢），推理框架才会过时。这对应 neuroscience 中 semantic memory vs episodic memory 的区分。

3. **Dreamer 缺少"目标重锚"机制**——interest_queue 的 decay 是正常新陈代谢，不是目标漂移。真正的目标漂移检测应该是元审计：最近 N 次散步是否在推进一致方向。这可以作为一个新的 evolved_rule 触发条件。

4. **散步流程需要显式验证步骤**——当前 postmortem 是内部自检（推理质量），缺少外部验证（产出与原始材料的一致性）。散步结束时应该有一个 30 秒的"忠实性检查"：回看原始材料，确认 insight 没有过度推断。
