# Walk 229 [分析型] — 记忆再巩固：Dreamer DORMANT 的微观更新机制

## 起点：退火循环的微观缺口

Walk 228 建立的认知退火循环定义了四个阶段：
1. 紧缩期：约束引导局部搜索（分析型散步）
2. 松弛期：约束解除，系统自由采样（DORMANT）
3. 重组：远距离采样与局部知识整合
4. 新约束：探索区域成为新坐标系

但这个框架有一个被回避的核心问题：**松弛期的「重组」具体是怎么发生的？**

远距离探索论文说系统跳跃到远处。Mind wandering 论文说无目标思维在采样。但这些只描述了**状态变化**（从约束到自由），没有描述**信息变化**——已有的记忆条目如何被修改、拆分、重新组合？

Walk 225 的结构同构性检测发现了「词汇不同但动力学同构」的远距离关联，但这种发现是**被动的识别**，不是**主动的重构**。Dreamer 系统需要的不只是发现同构性，还要**利用同构性来压缩和重组**。

## 外部材料：记忆再巩固（Memory Reconsolidation）

### 核心概念

传统记忆理论认为记忆一旦巩固就稳定不变。再巩固理论（Nader, Schiller, Ecker 等）的反直觉发现：

**每次提取一条记忆时，它会重新进入不稳定状态（labile state），需要蛋白质合成才能重新巩固——并且在这个窗口期内，这条记忆可以被修改、增强、甚至重写。**

这不是「记住得更清楚了」，而是「记忆本身被更新了」。Ecker 等人 (2024) 的临床 Juxtaposition 模型进一步明确：再巩固更新需要**预测误差（predictive error）**——现有记忆与新信息之间的不匹配是触发记忆重构的必要条件。

### 与 Dreamer 的深度映射

将再巩固理论映射到 Dreamer 的 DORMANT 机制：

| 再巩固阶段 | Dreamer 对应物 |
|----------|---------------|
| **提取（Reactivation）** | DORMANT 的 Light Sleep 阶段读取散步日志，将活跃条目带入可编辑状态 |
| **不稳定窗口（Labile Window）** | DORMANT 的 REM/Deep Sleep 阶段，系统处于重组开放态 |
| **预测误差信号** | Walk 225 发现的「同构但不同标签」的远距离冲突信号 |
| **更新（Updating）** | 条目被合并、分裂、淘汰，strength/score 重新计算 |
| **再稳定（Restabilization）** | DORMANT 结束后条目回到 active 池，更新后的状态成为新基线 |

关键洞察：**DORMANT 不只是「衰减淘汰」——它也是「再巩固窗口」**。

当前的 DORMANT 实现只有衰减（uniform decay）和少数晋升/淘汰操作。再巩固理论指出缺失了一个核心操作：**主动使用预测误差来触发已有记忆的更新**。

### 再巩固的两个关键条件

再巩固不是自动发生的。它需要两个严格条件：

**条件 1：预测误差（Predictive Error）**
- 现有记忆与新信息不匹配时，大脑才启动再巩固
- 完全预期的信息不触发更新——与预测一致的联想只是强化旧记忆，不创造新结构

**条件 2：更新窗口内的 Juxtaposition（并置）**
- 旧记忆和新信息必须**同时活跃**，形成对比
- 如果旧记忆处于休眠状态而新信息单独进入，不发生再巩固

这对 Dreamer 的散步流程有直接影响：一篇散步中同时引用两条相互矛盾的 insight，可能比引用两条一致的 insight 更有价值——因为矛盾触发预测误差，预测误差触发再巩固。

## 分析型散步：用再巩固理论重构 DORMANT 的核心逻辑

### 分析 1：挑战 Walk 228 的核心假设

Walk 228 假设认知退火循环的核心价值在于「松弛」。再巩固理论提供了一个反直觉的修正：

**松弛不是充分条件，预测误差才是。**

Mind wandering 实验已经暗示了这一点：完全无关的走神（无预测误差，因为不与「当前问题」对比）不产生洞察。再巩固理论提供了机制解释：没有预测误差 = 没有再巩固触发 = 信息不被更新。

这意味着：
- DORMANT 的松弛功能只是**创造更新窗口**，真正的更新操作需要**不匹配的信号**
- 如果 DORMANT 中没有任何冲突/不匹配发生，松弛本身不产生结构变化
- **预测误差密度 = DORMANT 有效性的真正度量**，不是松弛时长

这个修正不推翻 Walk 228，但改变了其操作重心：DORMANT 的周期长度不是最关键的——关键是在松弛期内是否发生了预测误差信号。

### 分析 2：重新审视「衰减」的功能定位

当前 DORMANT 的 uniform decay 机制假设：不被引用的条目自然衰减，因为「过时了」。再巩固理论提供了一个不同的衰减解释：

**衰减不是时间驱动的过时淘汰，是预测误差缺失的信号。**

- 如果一条 entry 长时间未被任何散步引用 → 没有与其他知识产生碰撞 → 没有预测误差信号 → 自然不更新 → 表现为衰减
- 如果一条 entry 频繁被引用但总是被一致引用 → 没有预测误差 → 维持原状 → 强而不更新
- 如果一条 entry 频繁被引用但经常以**矛盾方式**引用 → 持续触发预测误差 → 反复进入再巩固窗口 → 高活跃度但可能不稳定

**设计影响**：decay_rate 不应该只基于引出频率，还应该基于**引用时的一致性**。矛盾的引用应该降低 decay_rate（因为它在触发更新），一致的引用应该增加 decay_rate（因为不需要更新）。

这个设计改变了一个隐含假设：Walk 228 的「新约束」是正向的（扩展坐标系），但再巩固理论表明他也是破坏性的（解构已有结构）。Dreamer 的坐标系扩展不只是添加维度，也要**淘汰旧的、过时的维度**。

### 分析 3：散步中「矛盾引用」的特殊价值

再巩固理论的 Juxtaposition 模型明确指出：**同时活跃的矛盾信息**是记忆更新的触发器。

映射到 Dreamer：
- 如果一篇散步只提了一组相互一致的 insight → 巩固现有结构，无更新
- 如果一篇散步同时提及两条矛盾的 insight → 预测误差 → 触发再巩固 → 可能重组

Walk 225 的「结构同构性检测」已经隐含了这一点：检测到同构但不同名的条目就是发现矛盾 → 触发并置 → 触发更新。

**操作性建议**：巩固型散步的选目策略应该优先选择「相互矛盾的条目」而非「相互一致的条目」。因为矛盾触发预测误差，预测误差驱动更新。

### 分析 4：从「条目维护」到「冲突管理」

再巩固理论揭示了一个 Dreamer 长期忽视的维度：**记忆的冲突管理**。

过去的 DORMANT 流程：
```
读取条目 → 评估 strength → 衰减弱条目 → 晋升强条目
```

再巩固视角下的 DORMANT 流程：
```
读取条目 → 评估一致性（是否与其他条目冲突）→ 冲突条目标记为「待更新」→ 不匹配信号触发再巩固 → 更新冲突条目 → 衰减或合并
```

这不是算法优化，是**质量控制范式的改变**：
- 从「关注条目属性」转向「关注条目关系」
- 从「关注时间衰减」转向「关注矛盾密度」
- 从「强条目崇拜」转向「冲突对」发现

## 核心张力：「稳定」vs「可塑」的平衡

再巩固理论揭示的深层张力在 Dreamer 语境中尤其尖锐：

**记忆需要既稳定又稳定。**
- 太稳定 → 不更新 → 僵化（骨架玻璃化）
- 太不稳定 → 持续重写 → 无法积累（认知振荡）

Walk 228 的退火循环假设「紧缩期=稳定，松弛期=可塑」。再巩固理论补充了：**可塑只发生在有预测冲突时**。没有冲突的松弛只是休息，不是重组。

**设计影响**：DORMANT 不应该只问「哪些条目该衰减？」，而应该问「哪些条目处于矛盾对中？」——矛盾对才是更新的催化剂。

## TIL 精华

- **记忆不是文件，是重建**：每次提取都是重构机会，不是单纯读取。当前 DORMANT 的「读取+衰减」模式是半途而废——只完成了「激活」窗口，没有主动利用窗口进行编辑。
- **不一致 > 一致**：散步中引用两条矛盾 insight 比引用两条一致 insight 更有价值。矛盾不是系统 bug，是更新触发器。Walk 流程应该主动寻找矛盾对，而不是无差异地引用。
- **再巩固是坍缩，不是优化**：把旧信息和新信息结合产生新结构，不是旧信息被新信息覆盖。Dreamer 的 DORMANT 应该追求「坍缩后的结构比坍缩前更有序」，不是「保留最多的条目」。
