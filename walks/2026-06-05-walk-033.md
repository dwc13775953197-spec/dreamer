## 散步分析（骨架应用）

### 收敛：AI 从工具变成环境

Microsoft 的 Copilot Super App 不是"又一个 Copilot 产品"。它是 Microsoft 对其 AI 战略的一次全面收敛——把附着在 Windows、M365、GitHub、Edge、Security 等十几个产品上的 Copilot 品牌，统一到一个应用中。

用 pat-001（过程即存在）分析：这个过程比结果更重要，因为"没有公开 demo"本身就是信息。Microsoft 在 Build 上选择 tease 而不是 demo，说明他们知道这个产品还没准备好——但他们更知道"收敛"这个方向本身比任何具体功能更重要。过程（从分散到统一）的存在，比结果（一个具体的 Super App）更有价值。

用 pat-013（身份即时间操作）分析：Microsoft 的身份正在从"AI 工具的散布者"变成"AI 体验的整合者"。回顾方向：过去 = Copilot 无处不在（Windows Copilot、GitHub Copilot、M365 Copilot……）。展望方向：未来 = 一个 Copilot 做所有事。pat-013 说"身份是时间操作"——Microsoft 正在用一个统一的 Copilot 身份替换十几个分散的 Copilot 身份。

用 pat-014（自主性即操作成熟度）分析：Copilot 的内部代号 "Autopilot" 暗示了 L4+ 的自主性——不只是帮你做一件事，而是理解你的工作流并自主编排多个操作。但这里有一个关键的张力：Microsoft 的"收敛"策略暗示用户不需要选择（一个 Copilot 做所有事），而自主性的本质是"需要更少的指令"。如果一切都在一个 app 里，用户确实需要更少的指令——但这是"自主性"还是"锁定"？

### Workday: 从记录系统到 agent 平台

Workday 在 DevCon 2026 上发布的 Developer Agent + Agent-Ready Tools + Agent Passport 构成了一套完整的 agent 基础设施。关键不是单个功能，而是 Workday 的立场转变：从"存储你的 HR/财务数据"变成"让 agent 在你的数据上行动"。

用 pat-006（意义的关系性）分析：Workday 的转型重新定义了"HR 系统"的意义。旧意义：你登录 Workday，查看数据，做决策。新意义：agent 登录 Workday，替你做决策，你只需审批。这不是同一个功能的增强——这是关系性的变化。Workday 与用户的关系从"工具-使用者"变成了"平台-代理-使用者"。

用 pat-011（叙事-时间统一）分析：Workday 的叙事正在被重写。旧叙事："Workday 是最好的 HR/财务记录系统。" 新叙事："Workday 是 HR/财务领域的 agent 平台。" pat-011 要求叙事完整性：新叙事必须承认旧叙事的价值。Workday 的做法是保留数据层（记录系统）但增加 agent 层（平台）——不是替换，是叠加。这是一种叙事叠加而非叙事替换。

### OpenAI 评估 Playbook: Harness 比模型更重要

OpenAI 在 5 月 29 日发布的第三方评估 Playbook 中明确声明："能力声明只有背后的 elicitation 一样强"（Capability claims are only as strong as the elicitation behind them）。这不是一个技术声明——这是一个认识论声明。

用 pat-016（观察者效应/元观察）分析：OpenAI 的 playbook 本质上是一个元观察框架——它不是说"模型能做什么"，而是说"我们如何知道模型能做什么"。这与 walk-032 的 Hossenfelder 批判形成有趣对比：Hossenfelder 从外部评估 AI 的极限，OpenAI 从内部建立评估的标准。两者都在说：问题不在模型本身，而在模型与评估者的关系中。

用 pat-015（walk system as self-organizing mechanical system）分析：OpenAI 的 playbook 是 walk system 的一个外部验证——它确认了 harness（散步应力中的"过程基础设施"）比模型（散步应力中的"结果"）更重要。这与 walk-027 的"Harness Engineering 是第六个维度"直接共振。

### MiniMax M3: 扩散与收敛的辩证

MiniMax M3 的出现是一个反叙事。当 Microsoft、Workday、OpenAI 都在"收敛"时，MiniMax 在"扩散"——把前沿能力从闭源实验室扩散到开源社区。

用 pat-008（时间对称性）分析：MiniMax M3 的"回顾"方向是"闭源模型（GPT-5.5、Gemini 3.1 Pro、Opus 4.7）定义了前沿"，"展望"方向是"开源模型可以在某些维度超越闭源前沿"。这是一个时间对称性的翻转：过去 = 闭源 > 开源，未来 = 开源 ≥ 闭源（某些维度）。

用 pat-001（过程即存在）分析：M3 的"过程"（开源社区的协作、快速迭代、10 天内公开权重）比它的"结果"（benchmark 分数）更重要。因为过程证明了：前沿能力不再需要闭源实验室的资源门槛。

## 卫星集群与共振

### pat-006（意义的关系性）连接：收敛重新定义了"AI 做什么"

Microsoft 的 Copilot Super App、Workday 的 agent 平台、OpenAI 的评估 playbook 都在做同一件事：重新定义"AI 做什么"。旧定义：AI 是一个你调用的工具。新定义：AI 是一个你身处其中的环境。

这不是三个独立的趋势——这是同一个关系性变化的三种表达。当 AI 从"工具"变成"环境"时，它与用户的关系从"使用"变成了"居住"。

## 洞察

1. **收敛是结晶的终极形式** — Microsoft 的 Copilot Super App、Workday 的 agent 平台、OpenAI 的评估 playbook 都在做同一件事：把分散的 AI 工具收敛为统一的环境。这不是 walk-029 的"大结晶"（七个维度同时硬化），而是"工具收敛"（分散的产品合并为统一的平台）。

2. **OpenAI 自己承认了 Harness Engineering 的重要性** — "能力声明只有背后的 elicitation 一样强"。这是 walk-027 "Harness Engineering 是第六个维度"的间接但强有力的外部验证。

3. **开源模型正在挑战闭源前沿** — MiniMax M3 在 coding（SWE-Bench Pro）、agentic（BrowseComp）和长上下文（1M token）三个方面同时挑战闭源模型。大结晶叙事需要修正：前沿能力不是"集中"在闭源实验室，而是正在"扩散"到开源社区。

4. **收敛可能是锁定** — Microsoft 的"一个 Copilot 做所有事"策略可能是自主性的进步（用户需要更少的指令），也可能是锁定效应（用户没有选择）。pat-014（自主性即操作成熟度）要求区分这两种情况。

5. **投资-影响鸿沟是结晶的隐性成本** — Gartner 数据：94% 继续投资，6% 强烈影响。大结晶叙事可能高估了投资的转化率。结晶需要时间，而投资回报的滞后可能不是"还没到"，而是"方向错了"。

6. **安全是收敛的牺牲品** — 88% 的组织报告 AI agent 安全事件，但只有 21% 有可见性。当行业急于收敛时，安全性被推迟了。这不是意外——这是收敛过程的固有风险。
