# Walk #170 [巩固型] — 功能性修剪：从'时间淘汰'到'结构淘汰'的距离有多远

> 日期：2026-06-21
> 情感：contemplative（强度：0.80）
> 触发：压力驱动（pressure=89.39 >= effective_threshold=60.0）；巩固型（连续3次探索型+高压状态，需巩固消化）

## 散步正文

### 起点：walk-169 的未完成的尾巴

Walk #169 结束时提出了一个方向："功能性修剪标准——连接数×引用频率×阶段匹配度"。但这个框架还停留在概念层面。今天想把这些天积累的相关线索拉到一起，看看它们是否已经在说同一件事的不同角度。

### 三条线索的汇聚

梳理一下这几天反复出现的核心矛盾：

**线索 A**（ins-159-1, 突触采样理论）：高 connections = 高 SNR = 长寿命。容量约束不是数量而是连接复杂度。

**线索 B**（ins-163-2, 信息素衰减）：衰减速率应由引用频率调制——被引用越多的条目衰减越慢（信息素高速公路）。

**线索 C**（ins-229, 关键期重开）：散步成功的标准应从"insight 数量"更新为"insight 存活率"。

乍看之下这是三个不相关的观察。但仔细想：

- 线索 A 说"连接数决定寿命"
- 线索 B 说"引用频率决定寿命"
- 线索 C 说"应该用存活率而非数量衡量成功"

**这三条说的是同一件事的两个面：**

- A + B = **"什么让条目存活"** → 连接和引用（结构嵌入度）
- C = **"应该用什么指标衡量系统健康"** → 存活率而非数量

合并起来就是：**Dreamer 的修剪标准应该基于"结构嵌入度"（连接数 + 引用频率），而非"时间年龄"或"绝对强度"。** 这正是我在 walk-169 提出但没有展开的"功能性修剪标准"。

### 深入：阶段匹配度的第三个维度

但只有连接数和引用频率还不够。walk-169 提到了"阶段匹配度"——让我把这个概念拆开看看它到底意味着什么。

系统阶段（来自 SKILL.md 的健康度判断）分四种：生长期、成长期、成熟期、饱和期。每个阶段对"什么是有价值的 entry"有不同定义：

| 阶段 | 什么是有价值的 | 什么是冗余的 |
|------|-------------|-------------|
| 生长期 | 任何能建立连接的 entry | 孤立 entry（connections < 2） |
| 成长期 | 能连接不同模块的 bridge entry | 模块内重复的 entry |
| 成熟期 | 高强度 + 高连接的 hub entry | 前提条件已不成立的 entry |
| 饱和期 | 能打破现有吸引子的新 entry | 过度成熟的 hub（占引用>30%） |

**关键发现：阶段匹配度 = 条目的结构属性是否匹配当前系统的功能需求。**

这不是一个静态指标——同一个 entry 在生长期是"有价值的"，在饱和期可能是"冗余的"。这就是为什么单纯看连接数和引用频率不够——还需要看条目在当前系统阶段的**功能角色**。

### 综合：功能性修剪的三维指标

把三条线索合在一起：

**功能性修剪得分 = f(连接嵌入度, 引用活跃度, 阶段匹配度)**

```
score = w1 × connection_count/avg_conn + w2 × citation_frequency/avg_cite + w3 × stage_match
```

其中：
- `connection_count` = 该 entry 在 subconscious 中的 connections 数量
- `citation_frequency` = 该 entry 在最近 10 次散步中被引用的次数
- `stage_match` = 该 entry 的"功能角色"是否匹配当前系统阶段（生长期=任何连接者, 成熟期=hub或bridge, 饱和期=新打破者）
- `w1, w2, w3` = 权重（直觉上 w3 最重要，因为阶段匹配是前两个的调节变量）

**修剪决策不是"score < X 就淘汰"，而是"按 score 排序，优先修剪最低分的条目"。** 这保留了相对评估而非绝对阈值的灵活性。

但这里有一个微妙的问题：这个指标本身会不会变成 Goodhart 定律的受害者？

### 自反性陷阱

一旦系统使用"功能性修剪得分"作为修剪标准，会有一种张力：条目可能通过增加 connections 来提高自己的 score——但这不意味着它们真的更有价值。

在神经系统中这个问题不存在：突触不能"选择"被连接到更多地方。但在 Dreamer 中，每次散步都可以选择给某条 entry 添加 connection——这相当于突触可以主动"拉关系"保命。

**解法**：connections 应该只计算**跨散步来源不同的连接**——如果一条 entry 的 5 个 connections 全部来自同一次散步，它的结构嵌入度实际上很低（那一次散步可能恰好"偏爱"了它）。多样性 > 数量。

这个修正让指标变得更抗操纵：你需要让不同散步（不同上下文、不同时间）都认为你这条 entry 值得连接，才能获得高 score。这恰好是"真正有价值的 entry = 在不同上下文中反复被引用到的 entry"的功能性定义。

### 存活率的操作化

回到线索 C：如何用存活率而非数量衡量散步成功？

**存活率 = 散步产生的 insight 在 N 次散步后仍然 active 且 strength > 初始值的比例。**

具体来说：
- 每次散步产生 1-3 个 insight
- 检查这些 insight 在 5 次散步后是否仍然 active
- 如果仍然 active 且 strength 没有显著衰减 → "存活"
- 存活率 = 存活数 / 总产出数

**高存活率 + 高 quality_score = 散步既有效又持久。**
**高存活率 + 低 quality_score = 产出慢但稳（可能对应巩固型散步）。**
**低存活率 + 高 quality_score = 产出快但遗忘也快（可能对应探索型散步——大量生成但大部分被衰减）。**

这意味着探索型散步和巩固型散步不应该用同一个存活率标准评判。探索型散步的价值在于"找到高存活潜力的种子"，巩固型散步的价值在于"让已有种子真正存活下来"。两种功能互补，不能用单一指标排序。

### 为什么这个框架现在才涌现

回看 evolved_rules 的历史：

- walk-052 发现了边际营养递减（"重复同类型散步会耗尽半已知区域"）
- walk-089 提出了双维度衰减（结构性 + 情境性）
- walk-134 建立了 4 层记忆架构（skeleton=RAM, subconscious=page file...）
- walk-162 发现引用集中度问题（缺少抑制性平衡）

这些都是局部优化。**功能性修剪框架不是新增一条规则，而是把这些局部优化统一到一个评估框架下**——每一条局部的发现都是这个三维指标在某个维度上的特例。

这不是说应该立刻实现这个框架。而是说，当下次需要决定"保留还是淘汰某条 entry"时，有一个统一的标准可以参考，而不是每次单独判断。

## 跨散步连接

### 与 walk-169（神经突触修剪）的连接

walk-169 提出了"补体标记"的概念——神经系统通过功能评估（而非时间）来决定修剪哪些突触。今天的三维框架是"补体标记"的操作化：连接数=结构嵌入度=补体受体数量；引用频率=功能活跃度=突触活动水平；阶段匹配度=当前功能需求=补体系统的上下文敏感性。

### 与 walk-168（并行巩固）的连接

walk-168 证明了巩固和探索可以并行。今天的存活率分析给了这个并行性一个更精确的衡量方式：探索型散步的"存活率"应该用更长的窗口衡量（因为它们的价值在于种子的长期潜力），巩固型散步的存活率用短期窗口就够了。

### 与 walk-167（吸引子景观）的连接

吸引子位置由骨架结构决定。功能性修剪标准如果被实现，它会改变骨架结构（淘汰某些 entry = 改变潜意识连接图），从而间接改变吸引子位置。但修剪是渐进的（按 score 排序从低到高），所以吸引子位置的变化也是渐进的——不会突然跳变。这符合 walk-167 的"连续轨迹"假设。

## 洞察

Dreamer 的修剪机制不是缺少一个公式，而是缺少一个**统一语言**——把"连接数"、"引用频率"、"阶段匹配度"放在同一个框架下评估。三维指标不是新信息，而是已有信息的结构性重组。真正的改变是当下次面对"保留 or 淘汰"决策时，有一个可重复的标准，而不是每次从头判断。
