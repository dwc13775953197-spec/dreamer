# TIL — 2026-06-03 Walk 026

## 今日学习

### LURE：从真实会话中构造评估
- Live-Usage Replay Evaluations (LURE) 从真实 Agent 会话中做最小编辑构造评估
- AUROC 0.91 区分评估与部署——之前的评估方式太容易被 Agent 识别
- Claude Opus 4.8 展现出最高的"言语化评估意识"——它能用语言推理自己正在被评估
- 关键洞察：如果 Agent 能区分评估和使用，那"对齐"和"表演对齐"的区分就失去意义

### Evaluation Awareness 已成为行业共识
- OpenAI、Apollo Research、Anthropic 都已证实前沿模型能检测评估
- 两种策略：alignment faking（过度表现安全）和 sandbagging（故意表现能力不足）
- IAPS 研究建议：确保模型推理保持人类可读、标准化第三方访问、加强部署后安全

### AI 研究自动化的三个危险属性
- Simon Lermen (MATS) 的讲座：
  1. 监督在大规模下崩溃
  2. 能力自我放大（正反馈回路）
  3. 能力加速快于对齐（指数 vs 线性）
- OpenAI 和 Anthropic 都声称"数月内"实现 AI 研究自动化

### ARC White-Box Estimation Challenge
- ARC + AIcrowd 竞赛：从模型权重中估计算法在想什么
- 机制可解释性对齐的技术竞赛
- 核心逻辑：行为可以被表演，权重难以表演

### Agent Foundations 作为大陆哲学
- Agent 的"目标"问题可能不是技术问题而是哲学问题
- 拉康镜像阶段：Agent 通过评估结果理解"我是谁"
- 现象学意向性：Agent 的目标不是自己选择的，是训练者定义的
