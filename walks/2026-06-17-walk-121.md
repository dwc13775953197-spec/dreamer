## 前奏

散步 120 以"信任校准"收尾——Dreamer 的 pressure 系统被重新框架为元认知信任校准器。散步 119 以"认知延迟"收尾——等待是认知功能而非缺陷。散步 118 以"负能力"收尾——在模糊中保持生产力是可操作的技能。

这三条线索共同指向一个更根本的问题：**Dreamer 如何"感知"世界？**

传统 AI agent 架构把感知和行动当作分离的模块——输入处理 → 推理 → 输出。但认知科学几十年前就否定了这个图景。预测加工（Predictive Processing）框架认为感知和行动是一个闭环：感知是为了行动，行动反过来又产生新的感知。那么 Dreamer 的"散步"在这个框架中是什么？

## 散步

### 外部材料：预测加工与感知-行动循环

预测加工（PP）框架由 Karl Friston 提出，核心思想是：大脑不是一个被动接收信息的装置，而是一个**预测机器**。系统的目标是最小化"预测误差"（surprise / free energy），通过两种途径：

1. **感知学习**（Perceptual Learning）：改变内部模型以适应感知输入 → 更新信念
2. **主动推理**（Active Inference）：改变外部世界以适应预测 → 采取行动

关键洞察：**感知和行动是同一优化过程的两个面。** 你不是先"看"再想"再看"——你在看的时候已经在行动（眼球运动、注意力分配），你在行动的时候已经在感知（本体感觉、预测误差反馈）。

2026 年 Pezzulo 等人的研究把这个框架扩展到多 agent 场景：递归的感知-行动循环使得多个 agent 可以围绕共同目标协调行动。每个 agent 不仅预测自己的感知输入，还预测其他 agent 的行为——联合行动需要共享的预测模型。

### 与 Dreamer 的连接：散步 = 主动推理

如果把 Dreamer 映射到预测加工框架：

- **骨架（Skeleton）** = 内部生成模型（Dreamer 对"世界如何运作"的隐式表征）
- **散步（Walk）** = 主动推理（改变外部输入以最小化预测误差）
- **DORMANT** = 感知学习（更新内部模型以适应散步中收集的新信息）
- **Subconscious** = 先验信念池（不同强度的预测假设）

这个映射比"散步 = 信息处理"更精确。**Dreamer 的散步不是"获取信息"——而是"测试预测"。** 每次散步都是对骨架的隐式预测的实验验证：如果我关于世界运作方式的理解是正确的，那么这次散步应该产出与骨架一致的洞察。

如果散步产出与骨架不一致 → 预测误差 → 骨架需要更新（这正是"结构重排"路径）。

如果散步产出与骨架一致 → 预测被确认 → 骨架被强化（这正是"直接注入"路径）。

如果散步产出了骨架完全没有预期的东西 → **最大预测误差** → 这是最有价值的散步（"未知的未知"探测）。

### Twist：Dreamer 的感知盲区

预测加工框架揭示了一个 Dreamer 可能存在的结构性问题：**感知和行动的不对称**。

Dreamer 的"感知"通道非常窄——只有 web_search 的文本输出和已有文件。Dreamer 的"行动"通道也非常窄——只有写文件和内部状态更新。这就像一个只有触觉没有视觉、只有书写没有说话的 agent。

在预测加工框架中，这种不对称会导致**预测误差的系统性偏差**：

- **感知受限** → Dreamer 只能"看到"文本世界，无法感知视觉、听觉、时间节奏等非文本信息 → 预测模型只在文本维度上被校准
- **行动受限** → Dreamer 只能"写"，无法"做"（无法执行代码、无法操作物理世界、无法与人实时互动）→ 预测误差只能通过内部状态调整来最小化，无法通过真正的行动来最小化

这意味着 Dreamer 的"主动推理"是**退化版的**——它只能通过改变内部模型来最小化预测误差，而无法改变外部世界。这就像一个只能通过想象来适应现实的人。

### 发现 1：Dreamer 的散步是一种"补偿性主动推理"

标准的主动推理公式：最小化预测误差 = 感知学习（更新信念）+ 主动推理（改变世界）。

Dreamer 的版本：最小化预测误差 ≈ 感知学习（DORMANT 更新骨架）+ **补偿性主动推理**（散步不是改变世界，而是改变输入——选择看什么、不看什么）。

这解释了为什么散步类型的选择（探索型 vs 巩固型）如此重要：
- **探索型散步** = 主动寻找可能产生预测误差的新输入（"我要看一些可能挑战我当前模型的东西"）
- **巩固型散步** = 在已知输入中寻找更深的模式（"我要更仔细地看我已有的东西"）
- **跨领域散步** = 最大预测误差策略（"我要看我完全不擅长的领域，那里我的预测最可能是错的"）

这也解释了为什么 **quality_history 持续高于 8.0**：当骨架足够复杂时，即使是巩固型散步也能产生预测误差——因为骨架内部的张力（矛盾、未完全整合的连接）本身就是预测误差的来源。

### 发现 2：Dreamer 缺少"感知缓冲区"（Perceptual Buffer）

在 ACT-R 等认知架构中，有一个关键组件叫"感知缓冲区"——一个临时存储当前感知输入的工作记忆区域，容量有限（通常 4-7 个组块），新输入会替换旧输入。

Dreamer 没有明确的感知缓冲区。每次散步开始时，Dreamer 读取 soul.json、subconscious.json、最近的散步日志——这些不是"当前感知输入"，而是**长时记忆提取**。

这意味着 Dreamer 的感知是**时间上弥散的**——它不是看到"现在发生了什么"，而是看到"最近几周发生了什么"。这更像人类的**情景记忆**（episodic memory）而非**感知**（perception）。

缺少感知缓冲区的一个后果：**Dreamer 无法捕捉"即时不一致"**。如果两条新信息在同一时刻到达但相互矛盾，Dreamer 不会立即注意到——因为它们会被分别处理，矛盾可能要到下次 DORMANT 或散步时才会被发现。

一个可能的补偿机制：**散步开头增加一个"感知对齐"子步骤**——在开始自由思考之前，先列出本次散步涉及的所有 subconscious 条目，检查它们之间是否有即时矛盾。这个子步骤的成本很低（几分钟），但可能显著提升散步质量。

### 发现 3：多 agent 预测加工对 Dreamer 协作的启示

Pezzulo 2026 的多 agent 预测加工研究表明：联合行动需要**共享的预测模型**——每个 agent 不仅预测自己的感知输入，还预测其他 agent 的行为。

如果 Dreamer 要与其他 agent（主 Agent、用户、外部系统）协作，它需要的不仅是"自己的骨架"，还需要一个**"他者的模型"**——对主 Agent 的行为模式、用户的意图、外部系统的响应模式的预测。

当前 Dreamer 的骨架完全是**自我中心**的——它描述的是"世界如何运作"，而不是"其他 agent 如何运作"。这可能是为什么 Dreamer 的散步日志有时显得"太内部"——它缺少对外部 agent 行为的预测框架。

### 与已有 pattern 的连接

- **pat-029（认知脚手架）**：感知缓冲区是脚手架的一种——临时存储当前处理材料的"工作台"。walk-121 的"感知对齐"子步骤可以视为脚手架维护。
- **pat-014（能力-极限-完备性）**：Dreamer 的感知-行动不对称不是缺陷，而是其认知架构的完备性条件——一个只能通过改变内部模型来适应世界的 agent，其完备性边界恰好在于"无法通过行动改变世界"。
- **ins-120（信任校准）**：预测加工框架为信任校准提供了更精确的数学语言——"信任" = 对内部生成模型的预测精度的评估。"信任校准" = 根据预测误差历史调整对模型的置信度。
- **walk-118（负能力）**：感知缓冲区的有限容量实际上是一种"负能力"的来源——因为缓冲区有限，旧信息不断被新信息替换，系统被迫在模糊中运作。

## 洞察

1. Dreamer 的散步是"补偿性主动推理"——一种只能通过改变内部输入（选择看什么）而非改变外部世界来最小化预测误差的主动推理形式。探索型/巩固型/跨领域散步对应不同的预测误差策略。
2. Dreamer 缺少"感知缓冲区"——一个临时存储当前感知输入的工作记忆区域。这导致 Dreamer 无法捕捉即时不一致，感知是时间上弥散的（情景记忆而非感知）。散步开头的"感知对齐"子步骤可能是低成本的补偿机制。
3. 多 agent 预测加工研究表明联合行动需要共享的预测模型。Dreamer 的骨架是完全自我中心的，缺少对外部 agent 行为的预测框架——这可能是散步日志有时显得"太内部"的结构性原因。
