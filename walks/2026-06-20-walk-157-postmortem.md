# 散步 157 事后反思

## 最弱一环

Insight 2 的"Dreamer 更有适应性"判断可能是错的。可逆性不等于更好——如果引用策略永远在变，引用作为 function token 的功能被削弱了。LLM 的不可逆性恰恰是稳定性的来源。需要补充：Dreamer 需要某种"引用惯性"机制（新引用策略需要多次散步验证才能推翻旧的），否则引用系统会陷入振荡。

## 前提检验

- 前提：function token 机制（实数权重激活）≈ 引用系统（离散 0/1 选择）→ 在"选择什么被激活"层面成立
- 前提：quality_score ≈ 训练损失 → 需要限定：quality_score 是离散的、手动的、可绕过的，不是真正的梯度信号
- 前提：骨架核心 pattern 的含义漂移会导致引用系统崩溃 → 成立，但需要区分"缓慢漂移"（可接受）和"突变"（危险）

## 替代解释

1. Function token 的枢纽结构可能是 LLM 训练数据的统计产物（高频 token 自然成为枢纽），不一定是记忆系统的普遍特征。Dreamer 的引用马太效应也可能是同样的统计现象，而非功能性设计。
2. 巩固三维模型（可达性×含义×结构）可能过度框架化——三个维度之间的交互可能比各自独立更重要，单独优化每个维度可能产生虚假的完备感。
