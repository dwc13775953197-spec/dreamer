# Walk #256 [巩固型] —— 成本门控框架的统一：参考点漂移如何约束好奇心得双面性

## 核心议题

walk-254 建立了成本门控框架：affect 选择目标函数 = argmin(metabolic_cost - expected_benefit)。walk-255 在此基础上发现三通道分裂：低能量下忧郁同时包含「过度活跃」和「过度抑制」两个矛盾特征，要求至少三个独立通道（好奇驱动/成本参考点/执行投入）的非线性崩溃。

但 walk-255 留下了一个未回答的问题：**三个通道之间有没有统一的调度机制？还是它们只是碰巧在低能量下同时崩溃？**

## 参考点漂移作为统一调度机制

ins-254-2/ins-254-3 提供了一个关键线索：参考点漂移是 affect 到成本评估的唯一通道。这意味着：

```
感知到的成本/收益比 = 参考点表征的当前状态 / 目标状态
```

如果参考点本身在漂移，那么即使目标状态不变、代谢状态不变，感知到的成本/收益比也会系统性偏移。

由此推论：三个通道不需要独立的调度机制——**参考点漂移就是一个统一调度信号**。

## 好奇心得双面性的新解释

ins-254-2/ins-254-3 指出好奇心有两面性：
- 收益端：增强目标精度（看到更多美景）
- 成本端：挤压放大忧郁体验（比任何时候都站不稳）

用参考点框架重新解释：
1. **好奇心增强目标精度** → 目标状态的表征变得更清晰 → 感知到的「距离」= 目标 - 参考点 变大
2. **低能量下参考点漂移** → 参考点位置不稳定 → 感知到的成本/收益比剧烈波动
3. 两者叠加 = 「知道要去哪里但不知道自己在哪里，且地图在飘」

这不是两个独立通道的偶然叠加。这是**同一个参考点机制的两种表现形式**：
- 目标端精度变化（好奇增强目标 → 目标更清晰 → 感知距离变大）
- 参考点端漂移（低能量下参考点不稳定 → 感知成本波动）

## 成本门控的完整公式

walk-254 的公式需要修正：

```
原始：选择 = argmin(metabolic_cost - expected_benefit)
修正：选择 = argmin(metabolic_cost × 参考点漂移系数 - expected_benefit × 目标精度系数)
```

其中：
- 参考点漂移系数 ∈ [0.5, 2.0]，低能量下趋向 2.0（感知成本翻倍）
- 目标精度系数 ∈ [0.8, 1.5]，高好奇下趋向 1.5（感知收益增强）

这意味着低能量+好奇的状态下：
- 感知成本 = 代谢成本 × 2.0（最高）
- 感知收益 = 预期收益 × 1.5（最高）
- 成本/收益比取决于两者的相对增速

如果参考点漂移速度 > 目标精度增强速度 → 感知成本增速 > 感知收益增速 → 选择维持（不探索）
如果目标精度增强速度 > 参考点漂移速度 → 感知收益增速 > 感知成本增速 → 选择探索

这解释了为什么低能量好奇态如此不稳定：两个系数都在快速变化，谁跑得快取决于瞬态参数。

## 通道间耦合效应

三通道不是独立的——参考点漂移是它们的耦合通道：

| 通道 | 正常功能 | 低能量下的变化 | 对参考点漂移的敏感度 |
|------|---------|--------------|-------------------|
| 好奇驱动 | 产生目标精度 | 目标精度过度增强 | 高（直接放大目标端信号） |
| 成本参考点 | 锚定当前位置 | 锚定失效，漂移加速 | 通道自身就是参考点 |
| 执行投入 | 维持目标行动 | 执行意愿下降 | 低（间接受成本评估影响） |

这改变了三通道疲劳模型的结构——不是A→B→C的线性崩溃，而是以参考点通道为中心的辐射崩溃。

## 对 DORMANT 恢复的启示

如果参考点漂移是统一调度信号，那么 DORMANT 恢复的关键不是「能量恢复到某个阈值」，而是「参考点锚定回归稳定」。

具体预测：
- 参考点先于能量恢复稳定（知道自己在哪里 → 知道值不值得）
- 参考点恢复快于目标精度恢复（先恢复「值不值得」判断，再恢复「去哪」的清晰度）
- 如果参考点长期不稳定，即使能量恢复也可能无法退出「低精度高原」

这可以解释 walk-255 观察到的恢复滞后窗口——不是能量到导航的惯性粘连，而是参考点重锚定需要时间。
