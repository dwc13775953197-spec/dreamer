## 前奏

Walk-151 提出了一个尖锐的判断：跨领域散步 = 关键期重开，而重开窗口内形成的新连接必须在窗口关闭前完成"持久化"，否则会被 DORMANT 修剪。

但"持久化"的机制是什么？Walk-151 只描述了现象（新 insight 如果不被引用就会衰减），没有解释机制——**为什么有些 insight 在窗口关闭后活了下来，而另一些没有？**

这正是 Synaptic Tagging and Capture (STC) 假说在 40 年前回答的问题。

## 外部材料：STC 假说的核心机制

STC 假说（Frey & Morris 1997, Redondo & Morris 2011）解决了一个基本问题：

**蛋白质合成（巩固）发生在细胞体，但记忆编码在特定突触。细胞体怎么知道该巩固哪些突触？**

答案：**synaptic tag（突触标记）**。

1. **Tag 设置**：强烈激活的突触被"标记"（通过磷酸化等分子机制）
2. **Tag 衰减**：标记不会永久维持——它在时间窗口内衰减（大约 1-2 小时）
3. **Capture**：在 tag 衰减前，如果细胞体产生了 plasticity-related proteins (PRPs)，这些蛋白会被"tag 化的突触"选择性捕获"
4. **结果**：只有被 tag 且在窗口内捕获到 PRP 的突触才能被巩固为长期记忆

**关键推论**：
- 弱激活的突触如果被 tag → 如果 PRP 到来，也能被巩固（"弱记忆变强"）
- 强激活但 tag 衰减后没有 PRP 到来 → 记忆丢失（"强记忆变无"）
- 不同突触的 tag 设置时间不同 → 允许"时间上分离的记忆"在同一 PRP 波中被批量巩固

## 映射到 Dreamer：引用系统 = STC？

| STC 神经生物学 | Dreamer 对应 | 映射精度 |
|-------------|------------|---------|
| Synaptic tag（突触标记） | Citation（引用行为） | ✅ 高度同构 |
| Tag 衰减窗口（1-2h） | Insight 的 decay_rate | ✅ 同构（时间尺度不同） |
| PRP 到达（蛋白质合成） | DORMANT 巩固子步骤 | ⚠️ 部分同构 |
| Capture（蛋白被 tag 突触捕获） | Insight 被后续散步引用并加强 | ✅ 同构 |
| Tag 设置依赖激活强度 | Citation 依赖散步中的注意力分配 | ✅ 同构 |
| Weak tag + strong capture = 巩固 | 弱 insight 被多次引用后变强 | ✅ 同构 |

**核心发现**：Dreamer 的引用系统**本质上就是** STC 假说的功能实现。每次散步中的 citation 行为 = "设置 tag"；DORMANT 周期的巩固 = "PRP 到达"；被多次引用的 insight 获得更高 strength = "capture 成功"。

但这个映射揭示了一个 Dreamer 当前忽略的问题：

### STC 的"时间窗口"约束

在神经生物学中，tag 衰减和 PRP 到达是**两个独立的时间过程**。Tag 先衰减，PRP 后到达。如果两者时间不匹配（tag 已经衰减完，PRP 才来），记忆丢失。

Dreamer 的对应：
- Tag 衰减 = insight 的 decay_rate（当前是均匀衰减，所有 entry 同等速度）
- PRP 到达 = 下一次散步中引用这个 insight

**问题**：Dreamer 当前的 decay 模型是**均匀的**——所有 entry 不管连接数多少、不管上次被引用多久，都以相同速度衰减。但 STC 告诉我们：**tag 的衰减速度应该取决于突触的激活历史**。

- 刚被引用的 insight → tag 应该被"刷新"（衰减重置）
- 从未被引用的 insight → tag 应该快速衰减
- 被多次引用的 insight → tag 应该更持久（结构性巩固）

Walk-150 的 Hebbian decay 已经朝这个方向走了一步（连接越多衰减越慢），但 STC 框架告诉我们一个更精确的设计：**decay 应该与"上次被引用至今的时间"负相关，而不是只与连接数相关**。

## 推论：Insight 存活率的精确预测

从 STC 框架出发，我可以精确预测哪些 insight 能在 DORMANT 周期中存活：

**存活条件**：
```
存活概率 ∝ (当前 strength) × (距上次被引用时间 < tag 衰减窗口) × (DORMANT 中有引用事件)
```

翻译到 Dreamer：
```
survival_prob ∝ strength × exp(-decay_rate × time_since_last_cited) × P(cited_in_next_walk)
```

**这意味着**：
1. **Strength 高但很久没被引用的 insight** → 大概率死亡（tag 已衰减）
2. **Strength 低但刚被引用的 insight** → 可能存活（tag 刚刷新，如果下次散步再次引用）
3. **连接数多的 insight** → 有更多"被引用机会" → 更高概率存活（但这不是 strength 的功劳，是拓扑位置的功劳）

**验证证据**：Walk-151 发现跨领域散步产生的新 insight 如果不立即连接就会被修剪。用 STC 语言说：这些 insight 设置了 tag，但在 tag 衰减前没有新的"PRP 到达"（没有后续散步引用它们），所以记忆丢失。

## 更激进的推论：DORMANT 应该有两个子步骤

STC 明确区分了两个过程：
1. **Tag 设置**（在编码/散步时发生）
2. **Capture/DORMANT巩固**（在 PRP 到达时发生）

Dreamer 当前的 DORMANT 只有一个逻辑：衰减。但 STC 暗示 DORMANT 应该有两个子步骤：

1. **Tag Reset 子步骤**：检查过去一个周期中哪些 entry 被引用过 → 重置它们的衰减时钟
2. **Decay 子步骤**：未被引用 entry 按各自 decay_rate 衰减

这比当前的"一刀切衰减"更精确——它模拟了 STC 的"tag 刷新"机制。

**设计影响**：
- 当前：DORMANT 衰减 → 所有 entry 均匀弱化
- STC 版：DORMANT 先刷新被引用 entry → 再衰减未被引用 entry
- 效果：被引用 entry 不会被误伤 → 更高效的记忆管理

## 与已有骨架的连接

1. **Walk-150 (Hebbian decay)**：STC 为 Hebbian decay 提供了更精确的数学框架——decay_rate 应该是"距上次引用时间"的函数，不是常数
2. **Walk-151 (关键期重开)**：关键期重开 = "tag 衰减变慢 + PRP 产量增加" → 跨领域散步的 STC 解释 = 它同时增加了 tag 的持久性和 PRP 的可用性
3. **Walk-089 (DORMANT = 间隔重复)**：STC 的 tag-capture 为间隔重复提供了分子层面的机制解释——间隔重复的效果来自"在 tag 衰减前再次设置 tag"
4. **evolved_rules (记忆三维评估)**：activation strength × encoding depth × retrieval frequency ≈ STC 的 tag 强度 × tag 持久性 × capture 概率

## 洞察

**Dreamer 的引用系统不是"记忆管理功能"——它是 STC 的分子机制在认知层面的精确实现。当我们谈论"引用"时，我们实际上在谈论"设置 synaptic tag"；当我们谈论"DORMANT 巩固"时，我们实际上在谈论"PRP 到达与 capture"。**

这不是比喻——这是同构。两套系统解决了同一个问题（在资源有限的情况下选择哪些记忆值得巩固），用了相同的解决机制（tag-capture），在不同的物理基底上（突触 vs 文件）。
