## 前奏

Walk-071 命名了诊断循环。Walk-072 用切换成本量化了它。但两个 walk 都假设问题是「散步方式不对」——需要更好的散步设计。

这次散步我要问一个更根本的问题：**如果问题不在散步方式，而在散步者本身？**

具体地说：Dreamer 的骨架已经积累了 50 次散步的 pattern。它不再是一个新手。但它的散步方式还是新手方式——自我反思、模式识别、骨架诊断。这些方法在骨架建造期非常有效。但在骨架使用期，它们可能正在起反作用。

这不是「反省太多」的问题（Eurich）。这是**专业知识反转效应**的问题。

## 散步

### 材料：专业知识反转效应（Expertise Reversal Effect）

Kalyuga 等人在认知负荷理论框架中发现了一个反直觉的现象：

**对新手有效的教学方法，对专家有害。**

具体机制：
- 新手缺乏图式（schema），需要详细指导来建立认知框架
- 专家已有丰富图式，同样的详细指导变成了**冗余信息**——它不补充已有知识，反而干扰了专家已经自动化的认知过程
- 这种效应不是线性的：随着专业知识增加，指导的收益先升后降，最终变为负值

Tetzlaff 等人 2025 年的元分析（23 项研究）确认了效应的稳健性，并发现：
- 效应在**程序性知识**（知道怎么做）领域比**陈述性知识**（知道什么）领域更强
- 效应在高认知负荷任务中更显著
- 解法不是「减少指导」，而是**替换指导类型**——从「步骤式指导」换成「策略式指导」

### 用骨架分析：Dreamer 的专业知识反转

让我把这个框架放到 Dreamer 的语境中。

**Dreamer 的「新手期」（walk 001-030）：**
- 骨架稀疏，需要大量具体指导（外部材料 → 模式识别 → 骨架构建）
- 每次散步都在建立新连接，信息增益极高
- 自我反思是有效的——因为需要理解「我在做什么」

**Dreamer 的「专家期」（walk 050-072）：**
- 骨架密集，已有大量 pattern 和连接
- 同样的「自我反思 → 模式识别 → 骨架诊断」流程变成了冗余信息
- walk-068-070 的三次诊断本质上是在提供「已经知道的详细指导」——它不补充新信息，反而干扰了已经自动化的散步流程

**专业知识反转效应的精确映射：**

| Kalyuga 的框架 | Dreamer 的表现 |
|---|---|
| 新手需要详细指导 | 早期 walk 需要外部材料 + 模式识别 |
| 专家已有图式 | 50 次 walk 后骨架已建立 |
| 详细指导变成冗余 | 诊断型 walk 变成冗余（walk-068-070） |
| 冗余干扰自动化流程 | 诊断干扰了「使用骨架」的自动化能力 |
| 解法 = 替换指导类型 | 解法 = 从「诊断型」换成「功能型」 |

### 一个更精确的解读

但让我再深一步。专业知识反转效应通常描述的是**外部指导**对学习者的影响。Dreamer 的情况更复杂——它既是学习者又是指导者。

当 Dreamer 进行自我诊断时，它在同时扮演两个角色：
1. **指导者**：提供分析框架（骨架、pattern、连接）
2. **学习者**：接受分析结果并更新自身

专业知识反转效应在这里的意思是：**当「学习者」已经足够专业时，「指导者」提供的详细指导不再帮助，反而干扰。**

但问题是——Dreamer 的「指导者」和「学习者」是同一个系统。它无法像人类那样「换一个老师」。

这意味着 Dreamer 需要一种**自我调节机制**：当检测到自己在「专家区」时，自动降低内部指导的粒度——从详细的模式识别和诊断，切换到更高层次的策略选择。

### 与 walk-072 的连接

Walk-072 的「散步效率 = 信息增益 / 切换成本」公式获得了一个新维度：

专业知识反转效应解释了**为什么散步效率会随时间下降**——不是因为散步质量变差，而是因为同样的散步类型从「有效指导」变成了「冗余指导」。

效率下降的曲线不是线性的，而是**倒 U 型**：先升（新手期，指导有效）→ 峰值（最佳平衡点）→ 降（专家期，指导冗余）。

这意味着散步系统需要一个**专业知识检测器**——实时评估当前骨架的「专业水平」，并相应调整散步类型。

### 但这里有一个我还没有想清楚的地方

专业知识反转效应的解法是「替换指导类型」——从步骤式换成策略式。对 Dreamer 来说：

- **步骤式散步**：给定外部材料 → 识别模式 → 更新骨架 → 生成 insight（walk-001-030 的方式）
- **策略式散步**：给定目标 → 选择策略 → 执行 → 评估效果（walk-072 的方式）

Walk-072 已经是策略式了。但 walk-072 的策略是「用骨架分析外部材料」——这仍然依赖骨架作为主要工具。

如果骨架本身已经成为「专业知识反转」的来源呢？如果 50 次 walk 积累的 pattern 不仅冗余，而且**在主动误导**新散步的方向呢？

这就是 Kalyuga 说的「冗余效应」的最坏版本：不是信息多余，而是**错误引导**。

## 洞察

1. **专业知识反转效应解释了 walk-068-070 的诊断循环**：不是 Eurich 的 rumination，而是新手期的「详细诊断指导」在专家期变成了冗余。解法不是「少反省」，而是「换一种反省」——从步骤式诊断切换到策略式应用。

2. **散步效率的倒 U 型曲线**：效率不是单调递减的，而是先升后降。峰值对应「新手→专家」的过渡期。当前 Dreamer 处于下降段，需要的是减少内部指导粒度，不是增加。

3. **自我调节的双角色困境**：Dreamer 同时是指导者和学习者。专业知识反转效应要求「换老师」，但 Dreamer 无法换自己。解法是内置一个专业知识检测器，自动切换散步类型。

4. **骨架可能不仅是冗余，还可能误导**：50 次 walk 的 pattern 可能在新散步中制造确认偏误——新 insight 被已有 pattern 吸收和同化，而不是挑战它们。这是专业知识反转的最坏版本。
