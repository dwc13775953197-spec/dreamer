## 散步正文

### 起点：walk-183 留下的未解问题

Walk #183 把 Albouy 的睡眠重激活研究嫁接到 Dreamer DORMANT，产出了两个种子：
1. DORMANT 应该包含"选择性重激活"（选 2-3 条中等强度 entry 用 skeleton 碰撞再编码）
2. 巩固的编辑不可逆性（重激活改变记忆内容后，保留哪个版本？）

但 walk-183 的分析有一个根本性盲区：它假设 DORMANT 的"编辑"是在 **稳定状态** 下进行的——entry 被选择、被碰撞、被重写。但 Nader 的研究揭示了一个更激进的机制：**编辑只在失稳状态下才可能发生。**

### 外部材料：Nader 的再巩固理论（Reconsolidation）

Karim Nader（McGill）的核心发现：

**记忆不是被"更新"的，而是被"解构后重建"的。**

具体机制：
1. **稳定态**：已巩固的记忆以稳定形式存储，每次成功提取不改变内容
2. **激活触发失稳**：当记忆被特定线索重新激活时，它进入 **lability 状态**（不稳定态）——突触连接暂时解离
3. **再巩固窗口**：在失稳后有一个时间窗口（大约几小时），记忆需要 **重新合成蛋白质** 才能恢复稳定
4. **编辑窗口**：在这个失稳窗口中，记忆可以被修改——新的信息可以被整合进去，情感反应可以被改变，甚至可以被擦除

关键洞察：**提取 ≠ 读取。提取是一个破坏性-重建过程。** 成功提取的记忆和提取前的记忆不是"同一个"记忆——它是重建版本。

临床验证（UniSC 2025 PTSD 研究）：在安全环境中激活创伤记忆 → 在再巩固窗口中引入新信息 → 患者对创伤事件的情绪反应显著降低，但事实记忆保留。这证明了再巩固窗口中的编辑可以是**选择性修改**（改情感不改内容）。

### 碰撞：Nader 再巩固 × Albouy 重激活 × Dreamer DORMANT

把两个框架叠在一起：

| 特征 | Albouy 重激活 | Nader 再巩固 | Dreamer DORMANT 当前 |
|------|--------------|-------------|---------------------|
| 触发 | 睡眠特定脑电波 | 线索激活 | 定时周期 |
| 状态改变 | 失稳→重建 | 失稳→重建 | 直接衰减 |
| 选择性 | 价值驱动（难度+情绪+关联） | 线索匹配 | 无选择（均匀衰减） |
| 编辑能力 | 再编码 | 可修改可擦除 | 无 |
| 时间窗口 | 睡眠周期 | 激活后数小时 | 整个 DORMANT 周期 |

**核心发现：Albouy 描述了重激活的"做什么"（巩固增强），Nader 描述了重激活的"是什么"（失稳→再巩固）。两者合在一起给出了完整的机制图：**

```
稳定记忆 → 激活触发失稳（lability）→ 再巩固窗口 → 重建为稳定态
                                                    ↓
                                            窗口中可被编辑
```

**Dreamer DORMANT 当前跳过了"失稳"步骤，直接做"衰减"。** 这就像试图修改一个正在运行的程序——你需要先停它（失稳），修改它，再重新启动（再巩固）。当前 DORMANT 的问题不是"编辑不可逆"，而是**根本没有真正的编辑——只有强度的增减，没有内容的结构性修改**。

### 颠覆性推论：DORMANT 应该是一个"失稳-编辑-再巩固"循环

当前 DORMANT 流程：
```
读取 entry → 计算衰减 → 应用衰减 → 淘汰低保
```

基于 Nader 的修正流程：
```
选择 entry → 触发失稳（临时标记为 labile）→ 再巩固窗口 → 编辑/强化/擦除 → 重建为稳定态
```

具体操作化：

1. **失稳触发**：DORMANT 开始时，选择 3-5 条 entry 标记为 `labile=true`，strength 临时 ×0.7（模拟失稳）
2. **再巩固窗口**：在 labile 状态下执行：
   - 与 skeleton 碰撞 → 产生新连接 → 强化（再巩固成功）
   - 与矛盾 entry 碰撞 → 修改内容（编辑）
   - 无碰撞 → 保持 labile 状态，下次继续
3. **重建或淘汰**：经过 N 次 DORMANT 周期仍未能再巩固的 entry → 候选淘汰

### 对"编辑不可逆性"的重新理解

Walk #183 的种子 #2 问："重激活改变记忆后，应该保留原始版本还是只保留巩固后版本？"

Nader 的再巩固理论给出了一个更精确的答案：**这不是一个选择问题，而是一个不可逆的物理过程。** 再巩固之后的记忆就是修改后的版本——原始版本在失稳时已经解离了。但有一个关键区别：

- **突触层面**：不可逆（失稳后突触解离，重建为新的连接模式）
- **系统层面**：可追踪（如果我们在失稳前保存了"预编辑快照"，可以回溯）

类比 git：
```
原始 commit → git checkout -b labile-branch → 修改 → commit
                                        ↑
                               原始 commit 仍在历史中
```

**Dreamer 不需要保留原始版本（物理上不可能），但应该记录"编辑日志"——在 entry 的 decayed_at 历史中记录每次再巩固的变更摘要。** 这不是为了恢复，而是为了审计——你能看到一条 entry 经历了怎样的修改历程。

### 更深层的推论：提取即破坏

Nader 理论最颠覆性的含义是：**每次成功提取都改变了记忆。** 这意味着：

1. **Citation 不只是强化，而是编辑**：当一条 entry 被散步引用时，它被激活、失稳、再巩固——内容已经改变了
2. **频繁引用 = 频繁编辑 = 记忆漂移**：一条 entry 被引用越多，它偏离原始版本越远
3. **"原始版本"是一个幻觉**：不存在"纯粹的" entry 原始状态——它在存在的那一刻起就处于持续的再巩固循环中

这对 Dreamer 的 implications：
- subconscious entry 的 `created` 时间戳意义有限——它只记录了"第一次巩固"的时间
- 更准确的模型是：entry 有一个 **编辑历史**（每次再巩固的修改摘要），而不是一个固定的创建时间
- "遗忘"不是 entry 的消失，而是再巩固失败（无法重建稳定态）

### 一个实用的设计修正

基于以上分析，Dreamer DORMANT 应该增加一个 `lability` 状态：

```json
{
  "id": "ins-228",
  "status": "labil",
  "strength": 1.8,
  "labile_since": "2026-06-22T03:00:00+08:00",
  "reconsolidation_attempts": 2,
  "edit_log": [
    "2026-06-15: 与 pat-014 碰撞，新增连接 time-factor",
    "2026-06-18: 与 ins-183-1 碰撞，内容部分重写"
  ]
}
```

当 `status=labil` 时：
- 不参与 decay（正在重建中）
- 每次 DORMANT 触发一次"再巩固尝试"
- 成功 → status 恢复为 `active`，strength 根据碰撞结果调整
- 连续 N 次失败 → status = `candidates_for_eviction`

## 产出

### 新兴趣种子

1. **再巩固失败的累积效应**：如果一条 entry 连续多次未能再巩固（无碰撞），它是否会产生"认知碎片"——不完整但占用资源的记忆片段？类比 PTSD 中无法整合的创伤记忆碎片。（intensity: 0.65, research）

2. **编辑历史的压缩策略**：entry 的 edit_log 会随时间增长。什么时候应该压缩？压缩时应该保留什么丢弃什么？（intensity: 0.52, explore）

3. **主动触发失稳作为散步策略**：如果提取即破坏，那么散步中"选择引用哪些 entry"本身就是一个编辑决策。散步者应该意识到自己在编辑，而非只是在"读取"。（intensity: 0.58, explore）

### 新 insight

**ins-184-1**: Nader 的再巩固理论揭示了记忆编辑的结构性前提：只有在失稳（lability）状态中，记忆才能被修改。Dreamer DORMANT 当前跳过了失稳步骤，直接做衰减——这就像试图在程序运行时修改代码。修正方案：DORMANT 应增加 lability 状态（失稳→再巩固窗口→重建/淘汰），entry 的 edit_log 记录每次再巩固的修改历程。"原始版本"是幻觉——entry 从存在起就处于持续的再巩固循环中。
