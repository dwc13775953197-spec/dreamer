## 外部材料

### 材料 1：Benna-Fusi 突触采样理论（Computational Principles of Synaptic Memory Consolidation, 2016, 持续影响至今）

**核心模型**：每个突触维护 m 个变量（突触复杂度），每个变量有多个离散级。状态空间大小随 m 指数增长。

**关键发现**：
1. **记忆寿命 ∝ 突触复杂度 × 信噪比**：记忆存储容量不取决于突触数量，而取决于每个突触能区分的状态数。即使 m 很小（几个变量、几个离散级），状态空间也已指数级增长，允许极慢的记忆衰减。
2. **时间尺度分离是涌现属性**：系统自动分离有意义的时间尺度，不需要外部指定哪些快、哪些慢。时间分离因子以幂律缩放出现。
3. **容量-寿命 tradeoff**：同时存储的记忆数 N 与单个记忆寿命 T 之间存在反比约束——但通过增加突触复杂度 m，可以同时提升两者。

**直觉类比**：突触不是开关（0/1），而是旋钮（多个档位）。每增加一个旋钮，容量不是 +1 而是 ×N。

### 材料 2：Gershman 2025 海马索引理论重评

**核心论断**：海马体管理信息访问通过「键-值架构」——海马作为索引（key），指向皮层中存储的感知/语义内容（value）。海马不存储记忆本身，只存储如何找到记忆的指针。

**关键推论**：
- 记忆衰退 = 索引失效（key 丢了），不是内容损坏
- 记忆检索 = 用 key 重新激活整个 value pattern
- 索引效率取决于 key 的独特性（避免碰撞/错误检索）

## 跨散步连接

Walk-158 刚刚论证了 engram 重建理论：每次回忆在海马体中创建一个**全新**的 engram 细胞集群。现在 Benna-Fusi 模型补充了物理层面的约束：

| 层次 | Walk-158 (Engram) | Walk-159 (Synaptic) | 综合 |
|------|-------------------|---------------------|------|
| 内容 | 新 engram 取代旧 engrame | 突触状态空间编码内容 | 新 engram = 突触状态空间中的新区域 |
| 容量 | 无限（新集群随时可创） | 有限但指数级（m 变量约束） | 容量约束是真实的，但比线性预期宽松得多 |
| 衰退 | 旧 engram 失活 | SNR 随时间降低 | 衰退 = 信噪比崩溃，不是内容擦除 |
| 检索 | 入口转移 | 匹配 key-value | 检索 = key 匹配 + 突触状态读取 |

**核心张力**：Walk-158 说"每次回忆创建新 engram"——但 Benna-Fusi 说"突触状态空间有限"。两者如何共存？

**综合回答**：新 engram 不需要"新突触"——它需要的是突触**状态**的新配置。突触采样理论的核心洞见恰恰是：同一个突触可以通过状态重组来编码完全不同的信息。这就像同一个钢琴可以弹无数首不同的曲子——约束不是琴键数量，而是同时只能弹一首。

## 骨架分析：Dreamer 作为突触采样系统

### 映射

| 生物概念 | Dreamer 对应物 |
|---------|---------------|
| 突触 | Entry（subconscious.json 中的一条） |
| 突触复杂度 m | Entry 的 connections 数组长度 |
| 状态空间大小 | Entry 在 skeleton 中的可达引用路径数 |
| 信噪比（SNR） | Entry 的 strength 值（相对周围 entry 的显著性） |
| 记忆容量 | 同时存活的 active entry 数量 |
| 记忆寿命 | Entry 从 created 到 decayed 的时间 |

### 推论 1：Dreamer 的容量悖论是虚假的

当前 Dreamer 有 112 条 active entry、61 条 pending insight。表面上看"太多了"——但 Benna-Fusi 模型告诉我们，容量约束不在于 entry 数量，而在于每条 entry 的**连接复杂度**。

如果每条 entry 平均有 5 条 connections，状态空间是变量的指数函数——理论上可以容纳远超当前数量的活跃模式。真正的问题不是"太多 entry"，而是"entry 之间缺乏足够的区分度"（SNR 太低）。

### 推论 2：DORMANT 周期 = 突触状态重组窗口

Benna-Fusi 模型中，巩固发生在突触层面——通过调整变量状态来分离信号和噪声。映射到 Dreamer：

DORMANT 周期不是"休息"，而是**在不产生新信息的情况下，让已有 entry 的 connections 重新分配**。这恰好解释了为什么 DORMANT 周期后散步产出往往更高——不是因为有新输入，而是因为噪声 entry 已经被抑制（SNR 提升），检索更精准。

### 推论 3：引用系统的"幂律缩放"

Benna-Fusi 模型的核心数学结果是"时间分离因子以幂律缩放出现"。这意味着：
- 少数 entry 会自然成为"长时记忆"（高连接、高引用）
- 大多数 entry 会快速衰减
- 这种分布不需要外部设计——它是系统的涌现属性

**对 Dreamer 的启示**：当前的均匀衰减策略（所有 entry 用同一个 decay_rate）违反了突触采样理论的预测。自然系统应该呈现**幂律衰减**——少数 hub 存活，大多数快速消亡。如果 Dreamer 强制均匀衰减，会人为延长"噪声 entry"的寿命，降低整体 SNR。

### 推论 4：Gershman 的 key-value 映射

Gershman 2025 的海马索引理论提供了一个额外的架构层：

- **Key** = Entry 的 id（在 subconscious.json 中的标识符）
- **Value** = Entry 的 content + connections + 隐含含义
- **检索** = 散步过程中引用 entry 的过程

关键推论：**检索失败 = key 丢失，不是 value 损坏**。Dreamer 的引用系统如果漏掉了某条 entry，不是因为它不重要，而是因为**没有足够的线索把它拉进当前计算**。

这解释了为什么 Dreamer 需要"多样性种子"和"跨领域散步"——它们本质上是**增加检索路径**的策略，让那些平时不会被激活的 entry 有机会被重新访问。

## 综合论断

Dreamer 的 memory 系统正在经历一个**认知相变**：从"存储一切"（最大化 entry 数量）向"智能分配检索资源"（最大化关键 entry 的 SNR）过渡。

Benna-Fusi 模型告诉我们：**容量不是瓶颈，信噪比才是**。Dreamer 不需要减少 entry 数量——它需要让重要 entry 和不重要 entry 之间的差距更明显。

三个具体推论：
1. **DORMANT 应该实施差异化衰减**：高连接 entry 衰减慢，低连接 entry 衰减快（幂律，不是均匀）
2. **散步的核心价值不是产出新 entry，而是强化已有 entry 的连接**（提升 m → 提升 SNR → 延长记忆寿命）
3. **引用系统的设计目标不是"公平引用所有 entry"，而是"让高 SNR entry 更容易被检索到"**

## 洞察

突触采样理论的核心公式——记忆寿命随突触复杂度指数增长——可能是 Dreamer memory 系统最需要的数学基础。当前系统假设"所有 entry 平等衰减"，但生物系统证明：**连接的丰富性本身就是抗衰减机制**。

## 研究念头

如果把突触采样理论形式化为 Dreamer 的 memory 模型：
- 可以定义每条 entry 的"有效复杂度 m" = len(connections)
- 可以计算系统的"有效容量" = Σ 2^m_i
- 可以预测：当平均 m 从 2 提升到 5 时，系统容量增长 8 倍

这是一个值得形式化的数学框架。
