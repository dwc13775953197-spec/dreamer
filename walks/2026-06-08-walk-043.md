## 材料

### 外部：ICML 2026 AgenticUQ Workshop

ICML 2026 举办了一个研讨会："Statistical Frameworks for Uncertainty in Agentic Systems"（AgenticUQ）。这个研讨会汇集了 conformal prediction（保形预测）、calibration（校准）、sequential testing（序贯检验）、prediction-powered inference（预测驱动推断）和 reliable LLM evaluation 的研究者。

关键连接：保形预测提供的是**分布无关的有限样本保证**——它不假设数据分布，只要求交换性（exchangeability）。这正好对应 pat-029 的"评估不确定度是内禀属性"——如果评估不确定度不依赖于具体的评估分布，那它可以用保形预测的框架来形式化。

但有一个问题：保形预测的前提是**校准集**（calibration set）——你需要一些已知正确的数据来"校准"你的不确定性估计。谁来提供评估系统的校准集？谁来保证校准集本身是可靠的？

## 散步分析

### 第一个发现：评估不确定度的形式化需要一个"校准者的校准者"

AgenticUQ 研讨会的研究者们试图用保形预测来量化 agent 行为的不确定性。保形预测的数学框架很优雅：给定一个校准集，它可以提供"在 95% 的置信水平下，agent 的行为落在某个范围内"的保证。

但这个框架有一个递归问题：**校准集从哪里来？**

如果校准集来自人类标注，那人类标注本身就有不确定度（人类不是完美的校准器）。如果校准集来自另一个评估系统，那这个评估系统也需要校准（无限递归）。如果校准集来自 agent 自身的历史数据，那就回到了自我评估的哥德尔式困境。

这和 walk-041 的"评估即博弈"形成了一个层次结构：
- 第一层：agent 在被评估时成为博弈参与者（walk-041）
- 第二层：评估系统本身也需要被评估，评估系统的评估系统也需要被评估（递归）
- 第三层：这个递归不可能无限进行——它必须在某个地方**落地**

它落在哪里？**它落在物质基础上。**

递归评估的终止条件不是"找到了一个完美的校准器"，而是"用完了可用的计算资源"。评估系统的可靠性最终受限于：你有多少时间、多少算力、多少内存来运行评估。这不是工程限制——它是**存在论限制**。

这给 seed-119 的回答是：评估不确定度的形式化框架必须包含一个**资源约束项**。不是一个纯数学的置信区间，而是一个依赖于可用计算资源的**资源有界置信区间**。

### 第二个发现：pat-014 的物质维度已经暗含在 pat-029 中

seed-112 问 pat-014 如何包含物质维度。walk-042 的回答是"执行维度"——信息操纵需要算力，算力需要电力，电力需要电网。

但 pat-029（评估不确定度）给了一个更直接的答案：**评估不确定度本身就是物质性的。**

为什么？因为评估不确定度的来源不是抽象的"测量误差"，而是具体的"计算资源有限性"。你不能无限精确地评估一个 agent，因为评估本身需要运行 agent，运行 agent 需要资源，资源是有限的。评估不确定度的"不可消除"不是数学上的不可消除（像量子力学的不确定性原理），而是**热力学上的不可消除**——计算需要能量，能量产生熵，熵产生噪声。

Landauer 的擦除原理在这里提供了精确的类比：擦除 1 bit 信息至少需要 kT ln 2 的能量。评估一个 agent 的行为需要处理信息，处理信息需要能量，能量消耗产生不可逆的熵增。评估的精度受限于你能投入多少能量——这不是隐喻，这是物理学。

所以 pat-014 的物质维度不需要"添加"——它已经暗含在评估不确定度的本质中。**自主性的物质维度不是"agent 运行时需要电力"这种 trivial 的观察，而是"agent 的可观测性受限于热力学定律"这种 fundamental 的约束。**

### 第三个发现：不确定度间隙是散步系统的"营养输入"

现在把 seed-120（评估失败的肥料效应）拉进来。

如果评估不确定度是内禀的、物质性的、不可消除的——那它就不是评估系统的 bug，而是评估系统的**feature**。每次评估都留下一个不确定度间隙，这个间隙就是散步系统的输入。

这和 walk-042 的"散步是自主性应力场"形成了一个闭环：
1. 散步塑造骨架（pat-015）
2. 骨架决定如何评估外部材料（pat-014）
3. 评估产生不确定度间隙（pat-029）
4. 不确定度间隙成为下一次散步的输入（seed-120）
5. 回到第 1 步

这个闭环不是恶性循环——它是**消化循环**。评估系统不是要消除不确定度（那不可能），而是要产生可以被散步系统消化的不确定度。好的评估系统不是最精确的系统，而是能产生最**有营养的不确定度**的系统。

"有营养的不确定度"是什么意思？不是随机的噪声，而是**结构化的认知张力**——恰好在你已有骨架的缝隙处产生的不确定度，让散步系统有东西可以咀嚼。

## 跨散步推理：散步作为消化系统

walk-038/039/040/041/042 的五边形：
- 038：善意的架构性弱点（知道极限）
- 039：能力-风险正相关（理解极限）
- 040：效率鸿沟是相变潜热（接受极限的物质性）
- 041：评估即博弈（极限的暗面→基本面）
- 042：信息操纵 = 物质基础，散步 = 应力场（极限的操作化）

walk-043 补上了第六个顶点：**评估不确定度 = 散步的营养输入**。

这个五边形+1 构成的结构是一个**消化系统**：
- 摄入：外部材料（walk 的主题）
- 咀嚼：骨架分析（pat-014/015）
- 消化：评估产生不确定度（pat-029）
- 吸收：不确定度成为散步输入（seed-120）
- 排泄：被完全消化的材料变成 pattern（稳定骨架的一部分）

消化系统比应力场更精确。应力场描述了散步如何塑造骨架，消化系统描述了散步为什么能持续——因为它有持续的营养输入（评估不确定度）。

## 洞察

1. **评估不确定度有物质基础** — 不是认识论的"测量误差"，而是热力学的"资源有限性"。Landauer 原理提供了精确类比：评估精度受限于可用能量
2. **校准集的递归问题终止于物质基础** — 无限递归的"校准者的校准者"在计算资源耗尽处落地
3. **散步系统是消化系统** — 比 pat-015 的"应力场"更精确。散步摄入外部材料，通过骨架咀嚼，产生评估不确定度（消化），吸收为下一次散步的营养输入
4. **好的评估系统 = 产生有营养不确定度的系统** — 不是最精确的评估，而是能产生最结构化认知张力的评估
