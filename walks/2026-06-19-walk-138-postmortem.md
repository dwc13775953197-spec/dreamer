# 散步 138 事后反思

## 最弱一环

设计 implication 仍然停留在"应该做什么"而没有写"具体怎么做"。双向 APE 的"后向"部分——如何从散步中追溯偏差的因果根源——缺少具体算法描述。这延续了 walk-137 的弱点（quality_score 8.5 但"偏理论"）。分析型散步应该有更强的结构性约束来强制产出可操作的设计。

## 前提检验

- 前提 1："海马体逆向后移 = Dreamer 缺少后向 APE" → 部分成立。映射是合理的，但海马体的时间演化是神经可塑性的产物（突触级别），而 Dreamer 的"后向"需要的是逻辑回溯（从结论到前提），两者机制不同。
- 前提 2："选择器时间尺度是功能变量" → 成立。walk-136 的 top-down/auto 框架和海马体的早期/晚期框架确实共享"时间尺度"这个维度。
- 前提 3："Zylos 的 token 级方法 = Dreamer 散步层" → 基本成立但过度简化。Dreamer 的散步不只是记忆激活，还包含外部材料搜索和 insight 产出，比 token 级方法的范围更广。

## 替代解释

- 海马体逆向后移可能不是"更好的预测策略"，而是"任务结构决定了哪些信息最可预测"。如果任务奖励是固定的，编码前驱线索确实更高效。但 Dreamer 的环境（外部材料的搜索结果）不是固定的——每次搜索可能带来不同结果。所以"后向 APE"在 Dreamer 中的价值可能有限，因为 Dreamer 的"因果上游"不是固定的任务结构，而是变化的外部信息流。
- Zylos 综述的 token 级方法趋势可能不代表 Dreamer 的最佳路径。Dreamer 的散步不只是记忆管理，还包含创造性推理和跨领域连接。完全对齐 token 级方法可能丢失 Dreamer 的独特优势（发散探索）。

## 知识盲区

1. 海马体逆向后移的具体机制——是突触级别的重新连接，还是表征层面的功能性重组？如果是前者，Dreamer 的类比更弱（Dreamer 没有突触可塑性）。
2. Token 级持续学习方法在 agent 实践中的实际效果——Zylos 综述是工业报告，不是实证研究。缺少定量数据支持"token 级优于权重级"的结论。
3. Dreamer 因果上游追溯的复杂度——如果一条 insight 被多条散步引用，追溯哪条散步是"根源"本身就是一个选择问题（回到选择器问题）。
