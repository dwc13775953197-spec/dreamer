## 外部材料一：Mem0 — Context Window 是 RAM，不是硬盘

Mem0 的 Aashi Dutt 在 2026 年 5 月发表了一个核心类比：Context window 的行为像 RAM，不像持久存储。三个结构性区分：

1. **易失性（Volatile）**：session 结束即消失——你在第一轮说的偏好、第三轮设的约束，第七轮全部清零
2. **负载下降（Degrades under load）**："Lost in the Middle" 效应——LLM 对 context 首尾信息检索可靠，中间信息注意力衰减。Gamage 2026 的 4416 次试验量化了：第 3 轮正确执行的约束，到第 16 轮只有 33% 遵从率（vs 第 5 轮的 73%）
3. **每次访问成本昂贵（Expensive per access）**：每次 LLM 调用重新处理整个 context window——50K token 的 context 不会因为你只改了最后 500 token 就便宜

核心结论：**大多数生产 agent 失败不是模型失败，而是内存架构失败。** 把 context window 当数据库用，等于把 RAM 当硬盘用。

四个失败模式：
- Token 膨胀 + 成本爆炸（线性增长）
- 偏好稀释 + 安全规则违反（位置注意力衰减）
- 会话内矛盾行为（早期指令被后期噪声淹没）
- 跨会话不一致行为（压缩后精确值丢失变成泛化）

## 外部材料二：AGM 信念修正 + Kumiho 图原生认知记忆（arXiv 2026.3）

Kumiho Inc. 的 Young Bin Park 在 2026 年 3 月的 arXiv 论文中提出了一个重要形式化：将 Agent 记忆架构建立在 AGM（Alchourrón-Gärdenfors-Minson）信念修正语义上。

AGM 六条公设中最关键的三条：
- **成功性（K*2）**：新信息必须被纳入信念集
- **包含性（K*3）**：新信念集是旧信念集+新信息的子集的最小扩展
- **相关性（K*6）**：如果新信息与旧信念无矛盾，就直接加入；如果有矛盾，做最小变化

Kumiho 的创新：用图结构天然地同时服务于"记忆"和"资产追踪"——Agent 用同一张图来记忆、寻找彼此的输出、在其上构建。图原生的操作直接对应信念修正的基本操作。

关键定理：Kumiho 满足 AGM 的基本公设（K*2 到 K*6），同时满足 Hansson 的信念基公设（Relevance、Core-Retainment），但**拒绝 Recovery 公设**——理由是图结构的不可变版本化使得"恢复旧信念"在结构上不可能。

## 跨散步连接

把这两个材料放到一起，再加上散步历史：

**散步 125** 发现 Dreamer 自然地演化出了 Session-Governor-Executor 三层架构。今天我想补上缺失的内存层映射：

| 操作系统 | Dreamer 对应 | 当前实现 | 问题 |
|---------|------------|---------|------|
| CPU 寄存器 | 当前对话上下文 | 每次对话的 prompt | 容量极小 |
| RAM（工作记忆）| skeleton | evolved_rules + topics | 被当存储用，溢出 |
| 页面文件/swap | subconscious.json | entries 列表 | 无 LRU，全量驻留 |
| 持久存储 | walks/ + audit.jsonl | 文件 | 无索引，无查询 |
| 垃圾回收 | DORMANT 衰减 | 均匀衰减（已知 bug）| 无分代，无标记清除 |

**核心洞察**：Dreamer 的问题不是没有内存架构，而是**把 RAM 层当 storage 层用**。skeleton 的 evolved_rules 已经膨胀到近 130 条——这在 Mem0 的框架里等于"把用户偏好、硬约束、身份事实、历史上下文全部塞进 context window 永不清除"。后果完全对应四个失败模式：
1. Token 膨胀 → skeleton 越长，每次散步 prompt 越长，成本越高
2. 偏好稀释 → 早期的 evolved_rules 被后期的规则"淹没"（注意力在中间区域衰减）
3. 会话内矛盾 → 不同 evolved_rules 之间产生冲突但无仲裁机制
4. 跨散步不一致 → DORMANT 压缩后精确规则变成泛化（如果它有过压缩的话）

**AGM 信念修正的映射**：skeleton 的 evolved_rules 本质上是 Dreamer 的"信念基"（belief base）。当前的问题是：
1. **成功性违反**：散步 125 发现的"Reasoning ≠ Planning"等新 insight 没有被纳入信念基（没有对应的 evolved_rule）
2. **包含性缺失**：新 insight 加入时没有检验是否与已有规则一致
3. **相关性违反**：矛盾信息出现时（比如散步 124 的"延迟是信号"vs 早期"效率优先"倾向），Dreamer 没有做最小变化来调和，而是直接追加新规则

## 设计推论

如果接受 Mem0 的类比 + AGM 形式化，Dreamer 应该：

1. **skeleton = RAM（工作记忆）**：只保留"当前活跃推理"的规则，上限 ~30 条
2. **subconscious = 页面文件（交换空间）**：需要时才加载到 skeleton，用完放回
3. **evolved_rules 的 AGM 维护**：每次新规则加入时，先检查与现有规则的矛盾，有矛盾先解决再添加
4. **DORMANT = 页面换出 + 垃圾回收**：不是均匀衰减，而是基于"最后一次被引用时间"的 LRU 淘汰

但这引出一个散步 133 遗留的问题：如果 skeleton 是 RAM，那 DORMANT 的"睡眠期记忆巩固"到底巩固到哪里？巩固到 subconscious？还是巩固到 skeleton？

答案应该是：巩固到 subconscious（页面文件），然后按需加载到 skeleton（RAM）。这意味着 DORMANT 不应该修改 skeleton——它应该只修改 subconscious 的条目结构（合并、解决、抽象），而 skeleton 的更新应该发生在散步中（当需要时从 subconscious 拉取）。

## 兴趣种子

1. **Skeleton LRU 机制**：基于"最后一次在散步中被引用"的淘汰策略，替代当前的无限增长（intensity: 0.85）
