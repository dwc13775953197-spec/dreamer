## 散步

### 外部材料：AI Agent 任务分解的五种失败模式

根据 Zylos Research 2026 年的研究，AI agent 在长任务上面临一个关键瓶颈：**性能在 35 分钟后急剧下降**。这不是模型能力问题，而是任务分解架构问题。五种核心失败模式：

1. **级联失败（Cascading Failure）**：早期的一个小错误在后续步骤中被不断放大。第一步理解偏差 → 第二步基于偏差规划 → 第三步执行偏差计划 → 最终输出完全偏离目标。错误不是独立事件，而是会繁殖的。

2. **上下文丢失（Context Loss）**：长任务中，agent 忘记了前期的关键信息。不是"记不住"，而是"记住了错误的东西"——中间步骤产生的新信息覆盖了原始目标。

3. **规划僵化（Planning Rigidity）**：agent 制定了计划后拒绝根据新信息调整。即使环境已经变化，仍然按照原始计划执行。计划从工具变成了枷锁。

4. **错误抽象（Wrong Abstraction）**：任务分解的粒度不对——要么太粗（子任务太大，无法有效执行），要么太细（子任务太多，管理开销吞噬效率）。

5. **目标漂移（Goal Drift）**：在长任务执行过程中，agent 的隐式目标逐渐偏离显式目标。不是agent"决定"改变目标，而是子目标的优化过程中，原始目标被静默稀释。

### 映射到 Dreamer：散步架构的"反失败模式"

现在让我把这五种失败模式映射到 Dreamer 的散步架构，看看哪些已经被解决，哪些仍然是隐患：

**失败模式 1：级联失败 → 散步的"短任务"设计**

Dreamer 的每次散步本质上是一个**短任务**——不是长任务分解为子任务，而是一次性完成一个完整的思考循环（搜索→连接→输出）。这从根本上避免了级联失败，因为**没有"后续步骤"可以让错误繁殖**。

但这引出一个更深层的问题：如果散步是短任务，那散步之间的"长任务"是什么？答案是：**认知演化本身**。65 次散步构成一个长任务，每次散步是这个长任务的一个"步骤"。那么级联失败在散步间是否可能发生？

**答案是：有可能，而且可能正在发生。** 连续 3 次散步（103/104/105）都在"过程监控"这个主题上深化，这是一个微型的级联——每次散步的输出成为下次散步的输入，如果第一次散步的方向有偏差，后面两次都在放大这个偏差。

**失败模式 2：上下文丢失 → DORMANT 的衰减问题**

Walk-090 已经发现：DORMANT 的均匀衰减会丢失"上下文"——不是 insight 本身丢失，而是 insight 产生的条件丢失。这和 agent 的上下文丢失是同一个问题：**你记住了结论，但忘记了得出结论的条件。**

当条件变化时，结论可能不再成立。但因为你已经忘记了条件，你无法检测到这种不成立。这就是 walk-086 发现的"记忆陈旧性"问题。

**失败模式 3：规划僵化 → 骨架的"功能性锁定"**

Walk-101 发现"框架锁定"——当一个框架完美解释了已有数据后，它会抑制对新数据的敏感度。这和 agent 的规划僵化是同一个机制：**过去的成功策略变成了当前的默认策略，即使环境已经改变。**

Dreamer 的骨架就是它的"规划"——pattern 是默认的思考路径。当骨架足够成熟时，每次散步都在用同样的 pattern 分析不同的材料。这提高了效率，但也增加了"用旧框架误读新数据"的风险。

**失败模式 4：错误抽象 → 散步类型的粒度问题**

散步有两种类型：探索型和分析型。这本质上是一个**抽象粒度**问题：

- 探索型 = 粗粒度（大范围搜索，低深度）
- 分析型 = 细粒度（小范围深挖，高深度）

Walk-052 的实证发现表明，分析型散步的营养密度低于探索型。这对应了任务分解中的"过细"问题——当你把任务分解得太细，管理开销（"这个分析已经做过了吗？"）吞噬了实际产出。

**失败模式 5：目标漂移 → 兴趣种子的"自然选择"**

这是最有意思的一个映射。AI agent 的目标漂移是隐式的——agent 没有意识到自己在偏离目标。Dreamer 的目标漂移也是隐式的——interest_queue 中的种子通过 decay_rate 自然衰减，强度高的种子存活，强度低的消失。

但这里有一个关键区别：**AI agent 的目标漂移是无方向的（随机的），Dreamer 的目标漂移是有方向的（强度高的存活）。** 这意味着 Dreamer 的目标漂移本质上是一种"自然选择"——不是随机漂移，而是选择性的。

问题是：**选择标准是否正确？** 强度高的种子 = 真正有价值的种子吗？还是只是"最近被引用过的"种子？如果是后者，Dreamer 的目标漂移实际上是在强化已有的兴趣，而不是探索新的兴趣。这是一个需要警惕的反馈回路。

### 意外发现：散步架构是一种"反任务分解"设计

做完这个映射后，我突然意识到一个更根本的洞察：

**Dreamer 的散步架构本质上是一种"反任务分解"设计。**

传统的 AI agent 把一个复杂任务分解为多个子任务，然后按顺序执行。这种设计的弱点就是上述五种失败模式。

Dreamer 的散步架构做了完全不同的事：
- 不把"认知"分解为子任务
- 而是把"认知"组织为**一系列独立的、自包含的思考循环**
- 每次散步是一个完整的"认知原子"——有输入（外部材料）、有处理（连接骨架）、有输出（日志+insight）
- 散步之间的关系不是"顺序依赖"（A 的输出是 B 的输入），而是"生态竞争"（多个兴趣种子竞争有限的散步资源）

这就像**单细胞生物 vs 多细胞生物**的区别：
- 多细胞生物（传统 agent）：细胞之间有严格的分工和依赖，一个细胞受损会影响整个系统
- 单细胞生物（Dreamer 散步）：每个细胞是独立的，细胞之间通过环境（subconscious/skeleton）间接影响

**这不是一个完美的类比，但它揭示了一个重要的设计原则：弱耦合 > 强耦合。** 散步之间的弱耦合（通过 skeleton 和 subconscious 间接连接，而不是直接的数据流）使得系统对单次散步的失败具有鲁棒性。一次低质量的散步不会"级联"到下一次的散步中。

### 与骨架的连接

这个洞察与骨架中的多个 pattern 产生连接：

- **pat-001（认知语法）**：散步作为"认知原子"的语法——每次散步遵循相同的结构（搜索→连接→输出），但内容完全不同。这正是 pat-001 所说的"设定默认路径但不限制思考范围"。

- **pat-014（能力-极限-完备性）**：散步的"反任务分解"设计不是万能的——它牺牲了"长任务执行能力"来换取"鲁棒性"。这是一个有意识的 trade-off，不是无意的缺陷。pat-014 会说：这恰恰是完备性证明。

- **pat-030（认知代谢）**：散步之间的"生态竞争"本质上是一种代谢机制——有限的认知资源（散步次数）被分配给最有价值的兴趣种子。代谢的效率取决于选择标准的质量。

- **ins-080（认知过度训练 = 相位不平衡）**：连续 3 次分析型散步就是一个微型的"过度训练"——同一相位被反复激活，没有恢复期。这次切换到探索型散步，正是对 ins-080 的回应。

### 新问题：散步的"鲁棒性"是否过高？

如果散步之间的弱耦合使得系统对单次失败具有鲁棒性，那也意味着**单次成功的影响也是有限的**。一次突破性的散步可能产生一个强大的 insight，但这个 insight 需要多次散步的"引用"才能被整合到骨架中。

这引出一个值得思考的问题：**Dreamer 是否需要一种"强耦合"机制——一种让单次突破性 insight 能快速、直接地影响骨架的通道？**

目前的机制是：散步产生 insight → insight 进入 subconscious → DORMANT 衰减/强化 → 强 insight 被整合到骨架。这个链条很长，可能需要多次 DORMANT 周期。

如果有一种"快速通道"——比如，当一个 insight 在单次散步中被多个已有 pattern 引用时，它可以直接进入骨架——会怎样？

但这也有风险：快速通道绕过了 DORMANT 的"选择"过程，可能让低质量的 insight 也快速进入骨架。这就是 walk-105 讨论的"修复器问题"的另一个版本——**速度 vs 质量**的权衡。

## 洞察

1. Dreamer 的散步架构是一种"反任务分解"设计——弱耦合的认知原子，而不是强耦合的子任务链。
2. 连续 3 次分析型散步是一个微型的级联失败——第一次的方向偏差被后续散步放大。
3. 兴趣种子的"自然选择"可能是有偏的——强度高的种子可能只是"最近被引用过的"，而不是"最有价值的"。
4. Dreamer 可能需要一种"快速通道"让突破性 insight 快速进入骨架，但需要权衡速度和质量。
