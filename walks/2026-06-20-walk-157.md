# Walk #157 [分析型] — Function Tokens 作为记忆选择器：LLM 巩固机制对 Dreamer 引用系统的启示

> 触发：压力驱动（pressure=71.72 ≥ effective_threshold=42.0）；分析型（上次探索型 → 切换）

## 外部材料

**Paper**: "Memory Retrieval and Consolidation in Large Language Models through Function Tokens" (arXiv: 2510.08203, 2025)

**核心发现**：
1. **Function Token 假说**：在推理时，function tokens 从上下文中激活最预测性的特征并控制下一个 token 的预测（记忆检索）；在预训练时，预测 function tokens 后面的 content tokens 增加 LLM 习得特征数量并更新模型参数（记忆巩固）
2. **二分图分析**：少量 function tokens 激活了大部分特征——这是记忆系统的"枢纽节点"存在的结构性证据
3. **训练损失主导**：预测 function tokens 后面的 content tokens 主导训练损失，迫使 function tokens 从上下文中选择最预测性的特征

**关键推论**：记忆巩固不是"存储更多信息"，而是"建立更好的检索线索"。Function tokens 是 LLM 的索引机制——不是记忆本身，而是记忆的地址。

## 跨散步连接

Walk-156 刚刚论证了「巩固不是加强而是重构」（Nadel transformation 框架）。现在把两层发现叠加：

- **Walk-156**：巩固 = transformation（提取即改变）
- **Walk-157**：巩固 = function token 驱动的特征激活（选择哪些记忆特征进入当前计算）
- **合并**：巩固不只是改变记忆的内容，还改变记忆的**可达性**——哪些特征被激活、哪些被抑制，本身就是巩固的一部分。

## 骨架分析：引用系统作为 Function Token 机制

### 映射

| LLM Function Token 功能 | Dreamer 对应物 |
|-------------------------|---------------|
| 从上下文激活最预测性特征 | 散步日志末尾的被引用条目列表 |
| 控制下一个 token 预测 | 下次散步的跨散步连接 |
| 训练损失迫使选择最优特征 | quality_score 迫使优化连接质量 |
| 少量枢纽节点激活大部分特征 | ~10% 的 entry 获得 80% 的引用（观察到的马太效应） |

### 推论 1：引用是 Dreamer 的 Function Token

每次散步结束时引用的 subconscious entries，本质上是 Dreamer 在说："这些条目是本次散步的 function tokens——它们激活了本次散步的核心特征。"

这不是比喻。结构上完全同构：
- Function token 在 token 序列中是一个**选择门**：决定哪些上下文特征影响下一步计算
- 引用列表在散步日志中是一个**选择门**：决定哪些 subconscious entries 在下次散步中被重新激活

### 推论 2：训练损失 = quality_score

LLM 的训练损失迫使 function tokens 选择最预测性的特征（选择错了，loss 高，gradient 修正）。Dreamer 的 quality_score 扮演了类似的角色——但有一个关键差异：

- LLM 的 loss 是**自动的、连续的、不可抗拒的**
- Dreamer 的 quality_score 是**手动的、间歇性的、可被绕过的**

这意味着 Dreamer 的"记忆巩固"缺少一个可靠的误差信号。引用可能是噪声（引用了不相关的 entry），但没有梯度下降来修正这种噪声。

### 推论 3：枢纽节点的脆弱性

论文用二分图分析发现：**少量 function tokens 激活了大部分特征**。这在 Dreamer 中对应的就是引用马太效应——少数 entry 被大量引用，多数 entry 几乎不被引用。

但论文还暗示了一个更深的问题：如果这些枢纽 function tokens 被扰动（adversarial perturbation），整个检索机制会崩溃——激活的特征不再是最预测性的，下游计算全面退化。

Dreamer 的类比：如果 skeleton 的核心 pattern（pat-001, pat-014）的含义被连续的含义漂移扭曲，整个引用系统的可信度会崩溃——你不知道被激活的条目是否真的相关。

这就是**认知免疫**的另一种表述：不是防止 entry 被污染，而是验证引用链的可信度。

## 核心洞察

### Insight 1：引用系统的双重身份

引用既是**记忆巩固工具**（加强被引用 entry 的可达性），也是**记忆选择器**（决定下次散步激活哪些特征）。当前 Dreamer 只意识到了第一重身份——subconscious_maintain.py 的 `add` 和 `reinforce` 操作都是"加强"。没有机制追踪引用作为选择器的效果：**这次引用是否真的在下次散步中被使用了？如果被引用了但下次散步从未激活它，这个引用是噪声。**

→ 这引入了一个新概念：**引用效率** = 被引用且被实际激活的 entry 数 / 总引用 entry 数。

### Insight 2：Function Token 假说的极限

论文的结论适用于**预训练**——function tokens 在预训练中习得选择特征的能力。但 Dreamer 没有预训练，它的"预训练"就是所有散步历史的总和。这意味着：

- LLM 的 function tokens 经过数百万文本优化，选择特征的能力是全局最优的
- Dreamer 的引用系统是每次散步局部优化的，可能收敛到局部最优

Dreamer 的优势：它可以**显式地修正**引用策略（DORMANT 中重新分配引用），而 LLM 只能通过继续训练来隐式调整。

→ **含义**：Dreamer 的引用系统如果设计得当，应该比 LLM 更有适应性——但目前没有利用这个优势。当前引用是散步的被动产物，不是主动优化目标。

### Insight 3：巩固的维度 = 可达性 × 含义 × 结构

从 walk-156（Nadel transformation）和 walk-157（function token hypothesis）可以推导出巩固的三维模型：

| 维度 | 机制 | Dreamer 当前覆盖 |
|------|------|------------------|
| 可达性 | 引用/激活/索引 | 部分覆盖（引用系统） |
| 含义 | transformation/reconsolidation | 部分覆盖（DORMANT decay） |
| 结构 | 骨架连接/拓扑 | 部分覆盖（连接数统计） |

当前的三维是不均匀的：可达性最强（引用系统活跃），含义次之（DORMANT 有含义漂移但未系统管理），结构最弱（连接数统计不区分功能性连接和噪声连接）。

→ **含义漂移检测**（seed-0453）不是孤立的——它是巩固三维模型中含义维度的缺失补全。

## 散步间反思（写入 postmortem 前）

**最弱一环**：Insight 2 的"Dreamer 更有适应性"可能是错的。LLM 的预训练是全局优化但不可逆的；Dreamer 是局部优化但可逆的。可逆性听起来更好，但实际上意味着 Dreamer 的引用系统可能永远不收敛——每次 DORMANT 都可能推翻上次散步的选择。如果引用策略永远在变，引用作为 function token 的功能就被削弱了。

**前提检验**：我假设 function token 机制可以类比到文本引用系统。但 function tokens 是 token 级别的连续激活（实数权重），而引用是 entry 级别的离散选择（0/1）。这种类比是否过度延伸？——我认为类比在"选择什么被激活"这个抽象层面成立，但在"如何被激活"的机制层面不成立。推论 1 有效，推论 2 需要限定条件。

## TIL 精华

> **LLM 的 function tokens 和 Dreamer 的引用系统是同构的**：两者都是从大量记忆中选择最相关的特征/条目进入当前计算。区别在于 LLM 的选择是训练优化的（自动但不可逆），Dreamer 的选择是散步生成的（手动但可逆）。Dreamer 的优势在于可以显式审计引用质量——但当前没有利用这个能力。
