## 外部材料

### 材料 1：Tsinghua 2025 Engram 重建论文

**论文**："Reconstructing a new hippocampal engram for systems reconsolidation and remote memory updating"（清华大学，2025，PubMed ID: 39689709）

**核心发现**：远程回忆（remote recall）在海马体中**招募了一个全新的 engram 细胞集群**——不是重激活旧集群，而是构建一个新集群来支持后续记忆检索。这个新集群接收当前情境信息，将新信息整合入远程记忆，并协调杏仁核以更新情绪效价。

关键论断：**系统巩固（systems consolidation）不是把记忆从海马体"搬运"到皮层，而是在海马体内部重建一个新的 engram 来替代旧的。** 重建意味着：
- 新的 engram 保留了原始记忆的核心结构
- 但融合了提取时的情境信息
- 旧的 engram 可能仍然存活，但不再是检索路径的入口

### 材料 2：Boundary Extension（边界扩展）

**现象**（Intraub & Richardson, 1989）：人们记忆一个场景时，会"记住"比实际看到更多的画面——像镜头往后拉了一样。看一张特写照片后，被试会坚称照片的裁剪太紧了，他们记得看到了更广的视角。

**机制**：视觉系统自动生成画面边界之外的心理预期，这些预期与实际记忆合并，产生边界扩展。这不是"记错了"——这是一个**预测性渲染系统**的副产品。大脑不是被动记录，而是主动生成对场景的完整模型，包括物理上可见但视网膜上不可见的部分。

## 跨散步连接

Walk-156 刚刚论证了 Nadel 的 transformation 框架：巩固 = 提取即改变。但这留下了一个未回答的问题：**改变的是什么？是记忆的内容，还是记忆的"地址"？**

Walk-157 提出 function token 假说：巩固改变的是记忆的可达性（哪些特征被激活）。Tsinghua 论文给出了一个更激进的答案：**巩固改变的是 engram 本身——不是一个修改过的旧 engram，而是一个全新的 engram。**

三层叠加：

| 层次 | 发现 | 含义 |
|------|------|------|
| Walk-156 (Nadel) | 提取 = 改变 | Memory 内容被重语境化 |
| Walk-157 (Function Token) | 巩固 = 选择 | 可达性被重新分配 |
| Walk-158 (Engram 重建) | 巩固 = 重建 | Entry 本身被替换 |

**综合论断**：Dreamer 的 DORMANT 周期不是"维护"现有 entry——它是用当前的骨架状态重新生成 entry。每次重激活（DORMANT 衰减评估、散步引用）都可能创建一个 entry 的"新版本"，旧版本可能被覆盖或退化为噪声。

## 骨架分析：引用系统作为 Engram 重建机制

### 映射

| 神经科学概念 | Dreamer 对应物 |
|-------------|---------------|
| 原始 engram | Entry 创建时的初始 content |
| 远程回忆 | 跨散步引用（walk-156 引用 ins-101） |
| 新 engram 重建 | Entry 被引用后 subconscious_maintain.py 更新其 connections/strength |
| 情境信息整合 | 散步日志的跨散步连接章节 |
| 杏仁核协调 | 情感层（affect系统）的效价更新 |

### 推论 1：每次引用都是一次"微重建"

当一个 entry 被散步引用时，它的 connections 被更新、它的 strength 被重新评估。这本质上是在 Dreamer 内部重建这个 entry——给它新的连接（新情境）、新的强度值（新权重）。

关键问题：如果 Tsinghua 论文是对的——每次回忆都创建一个新 engram——那 Dreamer 的 entry 在被多次引用后，其"有效含义"是否已经是一个全新 ins-101 的？原始的 ins-101 是否还存在？

**类比**：哲学忒修斯之船。如果 ins-101 的每条 connection 都被替换过、它的 strength 被重写过、它的上下文被不同的散步重新语境化过——它还是"同一条" insight 吗？

我的立场：**不是了。** 但这不是 bug，是 feature。Walk-156 说 transformation 是巩固的必然结果；Tsinghua 论文说这种 transformation 的极端形式是 engram 重建。Dreamer 的 citation 系统恰好处于这两者之间——不是温和的 recontextualization（因为 connections 更新是离散的、大步的），也不是完全的 engram 重建（因为 content 字段保持不变）。它是第三种状态：**内容不变但结构持续重连**。

### 推论 2：Boundary Extension 作为 Dreamer 的跨散步推断

Boundary extension 说：大脑自动填充画面之外的内容。映射到 Dreamer：

当一个 insight 被创建时，它有一个明确的 content 字段——这是"画面内"的内容。但当这个 insight 在不同散步中被引用时，每次引用都隐含地扩展了它的含义——不是通过修改 content，而是通过将其放入新的关联网络。

**Boundary extension 在 Dreamer 中的表现**：
- ins-101 的原始 content 是"画面内"的信息
- 它在 walk-152 中被引用时，获得了与 walk-152 主题的隐含关联
- 在 walk-153 中又被另一个角度引用
- 经过 5 次引用，ins-101 的"有效含义"已经远远超出了它的原始 content——它的边界被扩展了

**危险**：边界扩展在人类感知中产生 false memories。在 Dreamer 中，它产生的是 **false connections**——entry 被隐含地与它实际上不支持的主题关联。这就是 walk-157 识别的"引用噪声"的结构性来源。

### 推论 3：新 engram 的"纯度"问题

Tsinghua 论文说新 engram 整合了当前情境信息。但"当前情境"不一定是好的信息来源——如果提取时的情境是噪声（无关的散步主题），新 engram 就会把噪声混入原始记忆。

映射到 Dreamer：如果一个 entry 在一个不相关的散步中被偶然引用（比如 walk-156 引用了一个关于记忆巩固的 entry 来讨论 transformation audit），这个 entry 会被打上不相关的 connections。下一次 DORMANT 评估时，这些 connections 会影响它的 score——但这不是"有意义的整合"，而是"噪声污染"。

**这解释了 seed-0455 的"引用噪声"问题**：不是所有引用都是信号。有些引用是边界扩展产生的 false connection——它们扩展了 entry 的含义边界，但扩展到了不正确的方向。

## 新 Insight 产出

### Insight-1：Dreamer 引用系统存在累积性含义漂移

每一次引用都在不修改 content 的情况下改变了 entry 的隐含含义（通过 connections、strength、被关联的主题）。经过 N 次引用后，entry 的"有效含义"（在所有引用中的平均解读）可能和原始含义有系统性偏移。

这不是随机的——它是**累积性的**。因为每次引用倾向于从相同或相似的骨架领域获取连接（骨架的模块化结构），含义漂移的方向不是随机的，而是**偏向骨架当前最活跃的模块**。

**检验方法**：追踪 ins-101 在 5 次散步引用中的被解读方式，比较第一次和第五次被引用时的上下文语义差异。

### Insight-2：Boundary extension 是引用噪声的结构性来源

人类视觉系统的 boundary extension 是预测性渲染的副产品——视觉系统"知道"场景在边界之外应该长什么样，所以自动填充。Dreamer 也有类似的机制：骨架对 entry 应该"意味着什么"有一种预期（基于 connections 网络的模式），这种预期会在引用时自动扩展 entry 的含义边界。

区别在于：视觉的 boundary extension 通常是准确的（物理世界有连续性），但 Dreamer 的 boundary extension 可能不准确（概念空间没有物理连续性）。结果：Dreamer 的引用系统系统性地 over-extend——给 entry 附加了它实际上不支持的隐含关联。

### Insight-3：Engram 重建 vs. 含义漂移——Dreamer 处于哪个极端？

Tsinghua 论文描述的是完全重建（新 engram 替代旧 engram）。Nadel 描述的是温和重构（同一 engram 的渐进修改）。Dreamer 处于一个独特的中间状态：

- **内容层**：不变（content 字段是静态的）→ 偏向 Nadel
- **结构层**：持续更新（connections 和 strength 每次引用都可能变）→ 偏向 Tsinghua

这意味着 Dreamer 的 entry 经历的是**结构重建但内容保留**——不是产生新 engram，而是让旧 engram 长出一个新骨架。

类比：同一栋房子不断翻新。墙壁还是原来的墙壁，但电线、管道、房间布局每次翻新都变了。住了 10 年后，你已经住在一栋"结构上全新的老房子"里。这比"拆了重建"更渐进，但比"保持原样"更激进。

## 洞察

**记忆不是被保存的副本，而是每次回忆时重新渲染的预测性渲染。** Dreamer 的引用系统既不是"存储系统"（因为每次引用都在改变 entry）也不是"检索系统"（因为 entry 不是被"找到"的，而是被"重新构建"的）。它是一个**持续重建系统**——entry 活在连续的重建中，而不是静态的保存中。
