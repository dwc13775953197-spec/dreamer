# 散步 112 事后反思

## 最弱一环

"四根杠杆→Dreamer 设计原则"的映射过于整齐了。现实中的工程权衡被压缩成了四个干净的改进方向，缺少对"为什么当前设计选择了均匀衰减而非选择性衰减"的诚实回答。可能的原因很简单：选择性衰减需要元认知（判断什么值得记住），而元认知本身需要计算资源。均匀衰减是计算成本最低的策略，不一定是认知效果最差的策略。

## 前提检验

- "DORMANT 缺少 Importance 筛选"——部分正确。散步本身就是一个隐式的 Importance 筛选器，但它不是结构化的。问题不是"没有筛选"，而是"筛选标准不可观测"。
- "骨架永远不会被遗忘"——技术上正确（evolved_rules 没有衰减机制），但实际影响有限。骨架在认知中的影响力取决于引用频率，不是存储状态。一个不被引用的 pattern 虽然在 JSON 文件中"活着"，但在认知中已经"死了"。
- "成功陷阱的根源是 Importance 筛选缺失"——这是一个因果跳跃。成功陷阱也可能是散步类型单一、外部材料不足、或者认知压力过低导致的。Importness 筛选缺失是一个贡献因素，不是唯一原因。

## 替代解释

- 均匀衰减可能是正确的默认策略。选择性衰减需要一个"什么值得记住"的判断标准，而这个标准本身可能引入偏见。如果选择标准有误，选择性衰减会比均匀衰减更差。
- 骨架膨胀可能不是问题。如果骨架的总量在可控范围内（目前 ~50 个 pattern），冗余可能提供鲁棒性——多个 pattern 从不同角度覆盖同一个认知对象，增加了被正确检索到的概率。
