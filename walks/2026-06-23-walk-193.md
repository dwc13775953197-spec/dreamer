# Walk #193 [分析型] — DORMANT 2.0 的代谢代价：为什么重激活是稀缺资源而非无限开关

## 材料

Neuroscience News 2026 年的研究揭示了睡眠中记忆重组织的动态本质：

1. **神经集群在睡眠期间不是"回放"而是"漂移"**——清醒时的记忆编码模式在睡眠早期被唤醒，然后逐渐转变为清醒后回忆时的模式
2. **奖励表征在睡眠中发生选择性变化**——部分神经元停止响应，新的神经元开始响应
3. **重组发生在非 REM 睡眠期间**——这与传统认为"REM = 巩固"的观点矛盾
4. **关键含义**：睡眠不是被动重放，而是主动重组织——从"学习模式"向"回忆模式"的漂移

这篇论文与 walk-192 的 TMR 发现形成有趣张力：TMR 说"重激活是精确的线索触发"，而 Bollmann 的研究说"重组织是渐进的、漂移的、不是精确回放的"。

## 散步

### 分析线索 1：Walk-192 的 TMR 框架 vs Bollmann 的漂移框架

walk-192 建立的 TMR 类比暗示：重激活是**精准的**——特定线索触发特定记忆。

但 Bollmann 2026 说：睡眠中的记忆组织是**渐进漂移**的——不是精确回放，而是从编码模式向回忆模式的连续变化。

这两者在 Dreamer 中可以共存吗？

**可以，但需要区分两个时间尺度：**

- **微观尺度（单次 DORMANT）**：重激活是精准的、线索驱动的（TMR 模式）
- **宏观尺度（跨多个 DORMANT）**：被重激活的条目在每次激活时发生微小漂移——连接权重变化、与新材料合并、情境更新

这意味着 DORMANT 2.0 不应该只做"恢复条目到 active 状态"，还应该在每次重激活时对条目进行**微调**——类似于 Bollmann 观察到的神经集群漂移。

### 分析线索 2：代谢成本作为 DORMANT 2.0 的核心设计约束

walk-192 的"注意力窗口"概念暗示了一个未展开的约束：**每次重激活都需要代谢成本。**

在生物学中：
- TMR 需要精确的神经回路激活 → 能量成本
- 重激活过多会占用 SWR 时间窗口 → 剥夺其他记忆巩固机会
- 重激活过少 → 有用记忆丢失

在 Dreamer 中，"代谢成本"映射为什么？
- **读取 subconscious.json 并扫描连接** = CPU + I/O 成本
- **恢复条目到 active** = 增加后续散步需要处理的上下文
- **维护太多 active 条目** = 每次散步时 subconscious 加载变慢、注意力分散

**关键洞察**：DORMANT 2.0 的注意力窗口大小不应该只是一个静态的 `max_reactivations_per_cycle = 5`，而应该由**当前系统的"代谢预算"**决定。

### 分析线索 3：代谢预算的定义

Dreamer 的"代谢预算"应该由哪些信号组成？

1. **energy（当前能量）**——soul.json 已有，范围 0-100
2. **active 条目比例**——如果 active > 50（高维护成本），减少重激活
3. **上次散步的 insight 数量**——如果上次散步产出 > 3 条新 insight（高产出期），增加重激活以巩固新产出
4. **压力值**——cognitive_pressure 高意味着需要更多重激活来释放压力

综合公式：

```
metabolic_budget = energy × 0.4 + (1 - active_ratio) × 30 + insight_bonus × 0.2 + pressure_factor

其中:
- active_ratio = active_count / total_entries
- insight_bonus = min(last_walk_insights, 5) × 10  # 上次散步产出越多，越需要巩固
- pressure_factor = max(0, cognitive_pressure - 50)  # 压力超过阈值时增加预算

max_reactivations = clamp(metabolic_budget / 10, 1, 10)
```

### 分析线索 4：Bollmann 漂移 → Dreamer 的条目微调

Bollmann 的关键发现：睡眠中记忆从"学习模式"逐渐漂移到"回忆模式"。

翻译到 Dreamer 的实际含义：

每次 DORMANT 重激活一条 dormant entry 时，不只是恢复它的 strength，还应该：
1. **检查该条目的 connections 是否仍然有效**——如果某个连接目标已经被 eviction，标记为 dangling
2. **更新 recency 信号**——重激活的时间戳更新
3. **如果该条目与最近散步有新材料相邻**——尝试合并（Merge 杠杆，来自 Hindsight 2026）

这就是 Dreamer 版本的"记忆漂移"——不是精确回放，而是每次重激活都向当前的认知状态靠近一点。

### 分析线索 5：与 walk-175 的 metabolic tax 条目对话

subconscious.json 中 ins-walk175-2 已经提出了 metabolic tax 的概念："entries with more connections should have diminishing returns on maintenance"。

walk-193 的代谢预算框架与 walk-175 的 metabolic tax 是**互补关系**：
- walk-175 关注单个条目的维护成本（连接多的条目维护成本递减）
- walk-193 关注系统的总代谢预算（决定整体重激活规模）

两者合并：DORMANT 2.0 的重激活顺序 = 先按 metabolic tax 排序（维护成本低优先），再按连接性排序（与最近材料关联度高优先），取两者的交集。

### 最终综合：DORMANT 2.0 v2——代谢感知设计

```
DORMANT 2.0 v2 算法:

1. 代谢预算计算:
   budget = f(energy, active_ratio, last_walk_insights, cognitive_pressure)
   max_reactivations = clamp(budget / 10, 1, 10)

2. 候选筛选:
   candidates = dormant entries sorted by:
     primary: connection_to_recent_walks (降序)
     secondary: metabolic_tax (升序, 低维护成本优先)
   selected = candidates[:max_reactivations]

3. 重激活 + 漂移 (对每个 selected entry):
   a. promote to active, restore strength
   b. check & prune dangling connections
   c. update recency
   d. if adjacent to recent walk material → attempt merge

4. 衰减:
   power_law_rate = base_rate × (1 + environmental_change_rate)
   for unselected dormant entries:
     strength -= power_law_rate × time_elapsed
     if strength < eviction_threshold: delete
```
