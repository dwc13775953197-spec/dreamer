# 脉冲触发与散步间隔检查

> 来源：Walk 004 脉冲执行 — 2026-05-29

## 散步触发检查逻辑

散步触发的判断条件：

```
距上次散步时间 >= current_interval_min → 散步触发
```

**不是**检查 `next_scheduled` 是否到达（那是整个脉冲的触发条件）。

`next_scheduled` 控制的是"是否执行脉冲"，而散步是否触发是在脉冲内部根据距上次散步的时间判断的。

### 示例

- `current_interval_min` = 288（4.8 小时）
- `last_walk` = 2026-05-28T20:16:00
- 当前时间 = 2026-05-29T02:03:00
- 距上次散步 = ~5.8 小时 > 4.8 小时 → **散步触发**

### 注意事项

1. `next_scheduled` 存储的是**本地时间（UTC+8）**，没有时区后缀
2. cron 运行环境是 UTC，比较时必须转换：当前 UTC + 8h = 本地时间
3. 如果距上次散步 < current_interval_min → 不散步，只做队列衰减和状态维护
4. 散步后更新 `last_walk` 为当前本地时间
