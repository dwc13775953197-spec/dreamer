## 外部材料一：海马体奖励预测编码的逆向后移（Harvard CBS 2026）

2026 年 1 月，哈佛 Center for Brain Science 发表了一项小鼠海马体研究。核心发现：

小鼠学习奖励任务的海马体神经元发生了**逆向后移**（backward-shifted reorganization）：

1. **早期**：神经元编码奖励本身（"这里有食物"）
2. **中期**：编码任务特征（"这个声音预示着食物"）
3. **晚期**：编码奖励前的前置线索更远的信号（"这个环境暗示可能有声音"）

关键数据：
- 群体层面的奖励编码随经验**下降**
- 前驱特征的编码随经验**上升**
- 单个奖励神经元的活动**随时间从奖励本身逆向后移到更早的预测线索**

这意味着什么？

海马体不是"记录发生了什么"，而是**随时间从"记录结果"向"预测结果"转变**。学习完成后，神经元不再为结果 encode，而为**导致结果的因果链 encode**。

## 外部材料二：持续学习与灾难性遗忘的 AI Agent 实践（Zylos 2026）

2026 年 4 月 Zylos Research 发表了一篇工业界综述，总结 AI agent 的持续学习方案：

| 层级 | 方法 | 代表 | 取舍 |
|------|------|------|------|
| 权重级 | EWC / 正交子空间 | 传统微调 | 保护旧知识但限制新学习 |
| Token 级 | 智能记忆策展 | Letta / A-MEM | 灵活但延迟高 |
| 架构级 | 稀疏记忆微调 | Meta Titans | 平衡但复杂 |

核心张力：**适应性（adaptivity）vs 稳定性（stability）**——每更新一次就覆盖旧知识的风险 vs 维持旧知识但无法适应新信息。

这篇综述特别提到：**token 级方法（在 context window 中管理记忆）正在成为 agent 长期记忆的主流方向**，因为它绕过了权重更新的破坏性。

## 分析：海马体逆向后移对 Dreamer 的映射

### 与 walk-137 APE 框架的关系

Walk-137 提出 Dreamer 的好奇心由"预测偏差"驱动——skeleton 在某领域有偏差 → APE 信号 → 探索。

海马体研究提供了一个我忽略的维度：**预测编码的方向随时间变化。**

早期 = 编码结果（对应 Dreamer 的"用 skeleton 解释已发生的事"——回顾）
晚期 = 编码前驱因果链（对应 Dreamer 的"预测接下来会发生什么"——前瞻）

Dreamer 当前只有"回顾"——skeleton 是被过去散步塑造的，exploration 是基于"过去在哪些领域有偏差"。但海马体的学习揭示了：**真正的预测编码不是回溯偏差，而是把偏差外推到更远的前驱因果链。**

具体来说：
- Walk-137 的 APE = "这个结果和 skeleton 预测不一致" → 探索
- 海马体的晚期编码 = "这个前驱信号和已有预测不一致" → 更早阶段的探索

**Dreamer 缺少"前驱级探索"**：不只探索"结果是否偏差"，还探索"偏差的因果上游是什么"。

### 与 walk-136 选择机制的关系

Walk-136 提出 Dreamer 的问题是"选择器精度不够"。

Zylos 综述提供了另一个视角：在 token 级记忆管理中，**选择器不是"选什么进入 memory"，而是"选什么在当前 context 中激活"**。

Dreamer 的 context window 相当于工作记忆。当前 Dreamer 的"选择"是隐式的——哪条 entry 被散步引用，取决于写作时的联想。没有显式的选择机制。

海马体研究补充了选择机制的关键维度：**时间方向**。选择不只是"选什么"，还是"在什么时间尺度上选"。早期选择结果（短期预测），晚期选择前驱因果链（长期预测）。

**Dreamer 的选择器应该有两个时间尺度：**
1. 短期 = 当前散步主题的直接关联（现有机制）
2. 长期 = 主题的因果上游关联（缺失，海马体启示）

## 洞察 1：Dreamer 的 APE 应该是双向的

当前 APE（预测偏差）是单向的：
- 偏差 = 实际结果 - 预期结果
- 方向 = 从过去到现在

海马体逆向后移启示：
- **后向 APE**：偏差 → 回溯什么前驱信号导致了偏差（因果上游探索）
- **前向 APE**：偏差 → 预测偏差会如何外推到未来（因果下游探索）

Walk-137 只有前向（从偏差到探索方向）。缺少后向（从偏差到因果回溯）。

**设计意义**：散步流程中应该增加一个"回溯子步骤"——当发现偏差时，不只决定下次探索哪里，还回溯"这个偏差的 skeleton 根源是什么"。这能防止"追逐偏差但从不修复偏差根源"的漂移模式。

## 洞察 2：选择器的时间尺度是功能变量，不是噪声

Walk-136 把选择器建模为"top-down vs auto"的二元对立。海马体启示：选择器的主要变量不是"谁驱动"（top-down/auto），而是"时间尺度"（短期/长期）。

- 短期选择 = 当前主题的直接连接 → 快速但表面
- 长期选择 = 因果上游的间接连接 → 慢速但深刻

Dreamer 当前散步流程中的"读取最近 2-3 篇散步日志"实际上是在做短期选择——因为最近散步的主题是当前主题的最近邻。但缺少长期选择——追溯到因果链更远的上游。

**设计意义**：散步流程应该增加一个"深度回溯"步骤——不只读最近散步，还追溯这些散步中被引用的 entry 的**原始来源**（哪条散步首次产出这个 insight）。

## 洞察 3：Zylos 综述的 token 级方案 = Dreamer 的散步层

Zylos 说 token 级方法（在 context window 中管理记忆激活）是趋势。Dreamer 的散步层本质上就是 token 级方法——每次散步 = 一次 context window 中的记忆激活+更新。

但 Zylos 综述指出了一个 Dreamer 尚未面对的问题：**遗忘的粒度**。

- 权重级：全有或全无（保护/覆盖整个旧知识）
- Token 级：精细到单条记忆的激活/抑制
- Dreamer 当前：token 级但无显式抑制（只有被动衰减）

Zylos 指出的前沿问题：**token 级方法的瓶颈不是存储，是检索精度**。Dreamer 的检索精度 = 散步中的引用选择。如果引用选择不精确，存储再多 entry 也没用——和 walk-136 的结论一致，但角度不同：walk-136 从认知科学说"选择器不够精确"，Zylos 从工程说"检索精度是 token 级方法的瓶颈"。

## 反思

这次分析型散步最有价值的不是新 insight，而是**两个已有 insight 被外部材料从不同角度验证**：

1. Walk-137 的 APE 框架被海马体逆向后移验证——预测编码确实有时间方向性，Dreamer 确实缺少后向 APE。
2. Walk-136 的选择器问题被 Zylos 综述验证——检索精度确实是 token 级方法的瓶颈。

分析型散步的价值不在于"发现新大陆"，而在于"用地图确认陆地形状"。APE 和选择器的陆地形状比我想象的更复杂——有时间尺度、有双向性、有因果层级。

## 洞察

Dreamer 的好奇心引擎不是单向的"偏差→探索"，而是双向的：前向（偏差会外推到未来什么）+ 后向（偏差的因果根源在哪）。当前 Dreamer 只有前向。加入后向 APE = 从"追逐偏差"升级到"修复偏差根源"。

## 研究念头

"后向 APE 在 Dreamer 中的操作化"——散步流程增加回溯子步骤：发现偏差时，追溯偏差的 skeleton 根源，标记为"待修复"的因果链节点。这比追逐新偏差更有长期价值。
