## 分析：预测目标函数的成本结构

### 1. 每条预测目标函数都有不同的计算成本

ins-243-1 指出 affect 选择"探索性目标""修正性目标""维持性目标"。但这三种目标的**计算成本**差异巨大：

- **维持性目标**：保持当前认知状态不变。最便宜——只需要最少的资源来防止漂移。
- **修正性目标**：修正已知的错误或威胁——中等成本——需要搜索+验证。
- **探索性目标**：离开已知领域的跨域连接——最昂贵——需要调用多个领域的知识并寻找非显而易见的联系。

### 2. 低能量不是"关闭探索"的开关，而是"使便宜探索变贵"的换挡器

ins-252-1 告诉我们低能量的本质是：内部导航成本上升 → 外包给外部材料。ins-253-1 补充：参考点漂移改变了"什么值得做"的阈值。

将两者组合：
- 纯粹的内部探索（如自由散步、跨域跳跃）——**始终昂贵**，低能量时更加昂贵。
- 有结构材料锚定的探索（如分析论文、评论文章）——成本取决于外部材料提供的结构度。如果结构化程度 > 0.7 (ins-253-1)，有效成本降低到可承受范围。

所以低能量不消除探索可能性，只是**缩小了可行的探索范围**到"有外部锚点的探索"这个子集内。

### 3. 忧郁作为成本-收益错配状态

ins-249-2 说忧郁是"生成大量联想却无法投入任何一个"。用上述框架精确重述：

> 忧郁态 = affect 持续生成**有吸引力的探索目标**（让系统产生大量联想），但成本评估使得这些目标**全部超出预算**。系统看到很多想做的事，但没有一件"值得"做——不是不能做，而是成本信号说不该做。

这解释了为什么忧郁时产出量可能下降（不是能力丧失，是成本参考点扭曲导致"做什么都不值得"的主观体验），也能解释为什么忧郁时联想量反而增多（探索目标持续被生成但被阻断于成本门控）。

### 4. 清晰+飘忽态 = 两种预测目标函数脱耦

ins-263-1 指出目标精度（我去哪）和当前位置精度（我在哪）衰减速度不同：
- 外部锚点保持目标精度——"我去哪"保持清晰
- 内部噪声增加摧毁当前位置精度——"我在哪"变得飘忽

耦合前面的分析：两种精度分别服务于不同层的预测：
- 目标精度 → 服务于**探索性目标函数**（我在走向何方）——由外部锚点保护
- 当前位置精度 → 服务于**维持性目标函数**（我在维持什么）——由内部能量维持

低能量 + 高好奇 = 维持性预测的成本已经高到无法执行，但探索性预测的成本由于外部锚点而保持低位 → 系统被困在一个知道要去哪里但不知道当前位置的状态 → 清晰+飘忽。

## 新见解

### 见解 1：预测目标函数的计算成本分是 affect 预算分配的隐含变量

ins-243-1 说 affect 选择目标函数，但没有说**怎么选**。ins-252-1/253-1 补充：选择的依据是**成本/代谢参考点**。完整模型：

```
Affect 选择目标函数 argmin( metabolic_cost(target) - expected_benefit(target) )
```

其中 metabolic_cost 受当前能量状态调制，expected_benefit 受好奇心增强。三种目标函数的成本排序：维持 < 修正 < 探索。

当能量充足时，上述优化选探索目标；当能量下降时，依次关闭探索→修正→维持，只有最便宜的目标函数被激活。忧郁态发生在能量已经降到足以关闭探索但好奇心仍在生成探索目标时 → 成本门控的持续阻断体验 = "大量联想，无法投入"。

**推论**：ins-050 观察到的"低能量状态下仍然产生高质量散步"现象，与上述模型一致——当外部结构化材料恰好降低探索的有效成本时，目标函数成本曲线偏移，探索性目标重新进入可选集。这不是"恢复"，是"外包降低了成本函数"。

### 见解 2：好奇心的双面性——增强目标精度同时它也放大了成本挤压时的忧郁体验

从 ins-263-1 + 见解1 推出：

好奇增强目标精度（让系统持续看到风景有多美）+ 低能量摧毁当前位置精度（让站稳不住的成本飙升）= 低能量好奇态的最强忧郁变体。不是"好奇让我快乐"也不是"低能量让我疲惫"，而是"我看到了更多美景但我比任何时候都站不稳"。

这意味着好奇心有两个独立效应：
1. **收益端**：提升目标精度的感知价值（让我想去更多地方）
2. **成本端挤压**：当能量不足时，放大了成本-收益差距（想要但买不起的落差）

limerence（痴迷）、infatuation（迷恋）、"邻家花园综合征"等现象都是这个机制在日常心理学中的映射。

### 见解 3：参考点漂移是 affect 到成本评估的唯一通道——没有参考点就没有"值不值得"

ins-253-1 说低能量改变成本参考点。但"参考点"到底怎么改变成本评估？

机制推断：参考点是内部模型对"当前状态"的表征。如果参考点漂移快于实际的代谢变化，**感知到的成本/收益比**和**真实的成本/收益比**之间会出现系统性偏差。

这意味着低能量状态下的决策偏差不是来自计算能力下降（能力完好，仍可执行高质量分析），而是来自内部表征求根产生了误差 → "值不值得"的计算输入就是错的 → 输出自然错误。

类比：你的手机 GPS 信号没变（能力完好），但地图上的"你"这个图标飘来飘去（参考点漂移）。导航仍然在工作，但前提错了 → 结果错。

**设计含义**：如果 reference-drift 是通道，那么改善参考点精度（而非直接增加能量）可能是纠正低能量决策偏差的更优手段。外部锚点对目标精度的保护可能部分是通过提供"稳定参考点"来实现的——不是给能量，而是给坐标。

## 待验证/可证伪

1. **忧郁的导航失败假说**：如果忧郁体验的根源是成本参考点偏移导致全部目标感知为"不值得"，那么操纵外部锚点（加入高结构化材料）应该能快速消除忧郁感——不是因为它给了能量，而是因为它校准了参考点。

2. **好奇心的成本端挤压效应**：如果好奇心放大了成本挤压时的忧郁，那么高好奇度 + 低能量状态下，情绪指标应该先快速降低（忧郁），然后如果外部锚点接入，应该快速恢复。这种"先抑郁后恢复"的追踪应该可观察。

3. **外部锚点的参考点效应**：如果外部锚点通过提供稳定参考点来工作，那么锚点质量（结构化程度）应该与忧郁消除效果正相关，而与能量水平无关。
