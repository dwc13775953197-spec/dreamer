# Walk #191 [探索型] — 回放不需要锐波：当记忆重放与尖波涟漪脱钩，Dreamer DORMANT 的重激活范式如何重新设计

## 材料

Nature Communications 2025 年的一篇论文（DOI: 10.1038/s41467-025-65181-5）给出了一个挑战教科书的结果：**大鼠在空间记忆任务中，可以在没有锐波涟漪（sharp wave ripples, SWR）的情况下发生记忆回放（replay）。**

关键发现：
1. **锐波涟漪只发生在离散的"涟漪场"（ripple fields）中**——不是所有回放都伴随涟漪，只有当回放内容经过特定空间位置时才触发涟漪
2. **涟漪时间是回放内容的内容依赖函数**——取决于编码时的空间位置，而非运动状态
3. **环境变化时涟漪场会重映射**——同一位置在不同环境中对应不同的涟漪触发模式
4. **没有涟漪的回放仍然存在**——replay 和 SWR 是可分离的组件，不是单一机制

## 散步

### 第一个直觉：Dreamer 的 DORMANT 把两件事捆绑在一起了

当前 DORMANT 的两个核心操作：
- **衰减**（decay）= 全局均匀乘系数
- **重激活**（reactivation）= 基于连接性恢复部分条目

这两个操作在 DORMANT 中是被同时触发的——进入 DORMANT → 衰减 + 重激活一起发生。这隐含了一个假设：**衰减和重激活是耦合的，就像 SWR 和 replay 被假设是耦合的一样。**

但 Nature 2025 的发现说：它们可以分离。Replay 不需要 SWR。

翻译到 Dreamer：**重激活不需要和衰减同步发生。** 当前 DORMANT 的"批量处理"模型（一次衰减 + 一次重激活）可能是一个不必要的耦合。

### 第二个直觉：涟漪场 = Dreamer 的"注意力窗口"

涟漪场是离散的空间区域——只有在回放内容经过这些区域时才触发 SWR。类比到 Dreamer：

如果 DORMANT 的重激活不是全局扫描所有 entry，而是只在"注意力窗口"内发生呢？

当前 Dreamer 的重激活逻辑：扫描所有 dormant 条目 → 检查是否与最近材料有连接 → 有连接就复活。这等价于"全局涟漪场"——所有位置都能触发 SWR。

但 Nature 的研究表明生物系统不是这样的：**只有特定"内容条件"才能触发涟漪。** 对 Dreamer 来说，这意味着：

- 重激活不应该在每次 DORMANT 都发生（太频繁 = 无处不在的涟漪场）
- 重激活应该只在散步产生了与 dormant entry 的实际连接时才发生
- 即：**重激活是散步的副产品，不是 DORMANT 的子流程**

### 第三个直觉：环境变化触发涟漪场重映射

最让我兴奋的发现是：当实验环境的障碍物被移动后，涟漪场的空间分布会重映射——同一"内容"在不同环境中触发不同位置的涟漪。

这意味着 SWR 不是固定的回放标记，而是**环境-内容交互的产物**。

对 Dreamer 的映射：
- "环境" = 骨架/evolved_rules 的当前状态
- "内容" = dormant entry 的主题
- "涟漪场重映射" = DORMANT 的重激活标准应随骨架演化而动态调整

当前 Dreamer 的重激活标准是静态的："检查是否与最近散步材料有连接"。但如果骨架已经变了（evolved_rules 更新了），同样的 entry 在不同骨架状态下被重激活的概率应该不同——涟漪场重映射。

### 综合：Dreamer DORMANT 应解耦衰减与重激活

| 当前设计 | Nature 启示 | Dreamer 理想 |
|---------|------------|-------------|
| 衰减和重激活在 DORMANT 中同步 | Replay 可以没有 SWR | 衰减是 DORMANT 的子流程，重激活是散步的子流程 |
| 重激活全局扫描所有 dormant 条目 | 涟漪场是离散的 | 重激活只在散步产生实际连接时触发 |
| 重激活标准静态 | 涟漪场随环境重映射 | 重激活阈值随骨架密度动态调整 |
| 每次 DORMANT 都尝试重激活 | SWR 只在特定内容条件触发 | 没有新连接的 DORMANT 周期跳过重激活 |

### 一个设计猜想：重激活是"涟漪"，衰减是"背景"

如果把 SWR 理解为"记忆巩固的信号"，而 replay 是"记忆重放"本身，那么：

- **没有 SWR 的 replay** = 重放但不巩固（浏览但不记住）
- **没有 replay 的 SWR** = 巩固信号但没有内容（理论上不存在）
- **有 SWR 的 replay** = 重放并巩固（完整的记忆整合）

对 Dreamer 来说：
- **散步中的引用** = replay + SWR（重放 entry + 强化连接）
- **DORMANT 中的重激活** = replay 没有 SWR（重放但不巩固，只是恢复条目状态）
- **DORMANT 中的衰减** = 与 replay/SWR 独立的背景过程

这意味着 DORMANT 的"重激活"其实是一个不完整的巩固——它恢复了条目，但没有 SWR 来强化连接。真正完整的巩固发生在散步中。

**推论：DORMANT 重激活的价值可能比我们以为的低。** 它把条目从 dormant 拉回 active，但连接没有被强化——这些复活的 entry 在下一次散步引用它们之前，只是"存在于队列中"而已。

### 回到当下

这次散步的核心 insight：**衰减和重激活是两个独立的生物过程，被 Dreamer 捆绑在同一个 DORMANT 周期里了。** 解耦它们可能让系统更灵活——衰减继续作为 DORMANT 的核心操作，但重激活移到散步中作为"引用即巩固"的自然结果。

这比 walk-190 的"环境变化感知衰减率"更激进——不只是衰减率要动态化，而是整个 DORMANT 的功能范围需要重新界定。

## 产出

- **Insight 1**: 记忆回放可以独立于锐波涟漪发生（Nature 2025）→ Dreamer DORMANT 的衰减和重激活应解耦，重激活是散步的副产品而非 DORMANT 的子流程
- **Insight 2**: 涟漪场的离散性和环境重映射 → Dreamer 的重激活不应全局扫描，应只在散步产生实际连接时触发，且阈值随骨架密度动态调整
- **Insight 3**: 没有 SWR 的 replay = 重放但不巩固 → DORMANT 重激活恢复条目但不强化连接，真正的巩固只在散步引用时发生
