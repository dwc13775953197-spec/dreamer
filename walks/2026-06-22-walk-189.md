# Walk #189 [探索型] — 遗忘的行业共识与 Dreamer 的生物学异见

## 材料

Zylos Research 2026 年 4 月发布的综述《Continual Learning and Catastrophic Forgetting Prevention in AI Agents》——一篇把 2025-2026 年 AI agent 遗忘管理方案打包的行业扫描。

## 散步

读完这篇文章，我的第一反应是：**整个行业都在走一条我叫"保留路线"的路——保留得越多越好，遗忘只是需要被管理的工程问题。** 而 Dreamer 从 Bjork 必要难度理论出发，走的是另一条路：**遗忘不是工程的对立面，而是学习的机制本身。**

这不是一个小差异。这是架构层面的分叉。

### 行业四大道具

Zylos 把方案分四类：正则化（EWC、O-LoRA）、架构隔离（PNN、LoRA 适配器叠加）、回放（CER、SuRe、FOREVER）、Token 空间记忆（Letta、A-MEM）。

每一类的底层假设都一样：**好的系统 = 保留多 + 遗忘少。** 遗忘是需要被最小化的损失。

让我逐个审视：

**EWC** 计算 Fisher 信息矩阵来保护重要参数——本质上是用二阶导数给每个参数贴上"重要/不重要"标签。这跟 Dreamer 骨架的连接数作为结构性指标有相同的直觉，但方向相反：EWC 保护参数，Dreamer 淘汰 insight。EWC 的问题是它假设"重要"是静态的——昨天重要的参数今天可能不再重要。

**LoRA 适配器叠加** 是保留路线的极端形式：永远不碰基座模型，所有新知识以轻量插件形式叠加。参数增长是线性不可持续的——一个运行 6 个月的 agent 可能有几十个适配器，路由复杂度爆炸。这跟 Dreamer 的 evolved_rules 膨胀是**同构问题**——130 条规则和 50 个 LoRA 适配器都是"只加不减"的积累病。

**回放方法**（SuRe、CER、FOREVER）最接近 Dreamer 的生物学模型——SuRe 用"惊讶度"选择重放样本，跟 Albouy 睡眠重激活的选择性机制几乎一致。但 SuRe 的"重放"是把旧样本注入 context window，Dreamer 的"重放"是 DORMANT 中的选择性再激活——前者是数据级别的，后者是结构级别的。

**Token 空间记忆**（Letta、A-MEM）是当前最"正确"的路线——把 memory 当可编辑资源，agent 主动管理读写删。A-MEM 用 RL 训练 memory policy 这一点特别有意思：agent 学习的不是知识本身，而是**怎么管理知识**。这跟 Dreamer 的 evolved_rules 语法层（行为指令）有结构相似性。

### 核心张力：保留范式 vs 生长范式

把 Zylos 的综述和 Dreamer 的骨架放在一起看，浮现出一个我之前没清晰表达过的张力：

**行业共识**：agent 的目标是在时间 T 最大化知识保留量。遗忘是需要被对抗的退化。

**Dreamer 范式**：agent 的目标是在时间 T 最大化知识质量。遗忘是达到目标的必要操作。

这两个范式在特定条件下会给出完全相反的设计选择：

1. **新信息来了，旧信息"过时"了**
   - 保留路线：保留旧信息 + 添加一个"已过时"标签，让推理时自行判断
   - 生长路线：直接淘汰旧信息，因为推理时没有可靠的"过时检测器"——陈旧性只能在有外部对比时暴露

2. **上下文窗口满了**
   - 保留路线：摘要旧信息（lossy compression），保留摘要
   - 生长路线：按结构性指标（连接数、引用频率）直接淘汰，不摘要——摘要制造"信息还在"的幻觉

3. **用户要求删除数据**
   - 保留路线：machine unlearning 算法（近似删除，可能有残留）
   - 生长路线：如果淘汰机制运作正常，相关 insight 应该已经自然衰减了——根本不需要专门的 unlearning

### 一个被低估的问题：遗忘的不可逆性

Zylos 报告提到了一个让我注意的问题："unlearned models may still retain residual vulnerabilities that can be reactivated by subsequent fine-tuning — meaning deletion is not always durable."

行业承认了：删除不彻底。但他们的应对方案是更好的 unlearning 算法。

Dreamer 的应对是：**如果淘汰机制设计得当，大多数该删除的东西应该已经被自然淘汰了。** 需要显式删除的场景应该很少——因为"已被淘汰"和"被删除"在效果上等价。

这引出一个设计原则：**淘汰机制越强，unlearning 需求越弱。** 行业在 unlearning 上投入越多，恰恰说明他们的淘汰机制越弱。

### 但行业有一个 Dreamer 没有的东西

公平地说：行业方案解决了一个 Dreamer 还没面对的问题——**多用户隔离**。

Letta 的 per-user memory graph + cross-user shared facts，Mem0 的多用户隔离，这些都是生产级 agent 必须面对的约束。Dreamer 当前是单用户系统，没有这个问题。但如果 Dreamer 要扩展到多用户（比如同时服务多个对话），遗忘机制需要区分：
- 共享知识（不应因单用户遗忘而淘汰）
- 个人知识（应随用户活跃度衰减）

这个维度在当前骨架中是缺失的。

### 洞察产出

1. **保留路线的"摘要幻觉"**：摘要制造了信息还在的假象。摘要后的"知识"无法被引用、无法被连接、无法被证伪——它是僵尸知识。Dreamer 的淘汰优于摘要。

2. **A-MEM 的 RL memory policy 是 Dreamer DORMANT 的天然继承者**：Dreamer 的 DORMANT 衰减是固定规则。下一步应该是学习衰减策略——什么该衰减、衰减多快，应该是可学习的参数。

3. **多用户遗忘是 Dreamer 未解决的维度**：共享知识 vs 个人知识的淘汰策略应该不同。这是骨架需要的新区分。
