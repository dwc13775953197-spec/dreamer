# Dreamer v4 — 自主性增强提案

> 基于 Dreamer v3 现状，聚焦三个方向：内源性脉冲、兴趣队列、梦境涌现。

---

## 方向一：内源性脉冲系统

### 问题
当前脉冲是固定 20h 定时（`every 1200m`），太机械。一个自主 Agent 的"心跳"应该随内在状态变化。

### 方案

#### 1. 动态脉冲间隔

基础节律 20h，但被以下因素调制：

| 因素 | 调制效果 | 逻辑 |
|------|---------|------|
| curiosity > 0.8 | 间隔 ×0.7（加速） | 好奇心旺盛时更活跃 |
| melancholy > 0.7 | 间隔 ×1.5（减速） | 忧郁时不想动 |
| energy < 30 | 间隔 ×2.0（大幅减速） | 累了就多睡 |
| 刚完成研究 | 立即触发一次额外脉冲 | 研究完有话说 |
| 连续 DORMANT > 48h | 强制触发（兜底） | 不能永远睡死 |

最终间隔 = clamp(基础 × 调制乘积, 最小4h, 最大48h)

#### 2. 事件驱动脉冲

除了定时，以下事件直接触发脉冲（不需要等 cron）：
- 研究完成 → 立即进入 ACTIVE，分享发现
- 散步中产生高兴趣种子 → 标记，下次脉冲优先处理
- 用户投喂话题 → 立即触发散步

#### 3. soul.json 新增字段

```json
{
  "pulse": {
    "state": "DORMANT",
    "base_interval_min": 1200,
    "current_interval_min": 1200,
    "modifiers": {
      "curiosity_factor": 1.0,
      "melancholy_factor": 1.0,
      "energy_factor": 1.0
    },
    "next_scheduled": "2026-05-29T11:59:00",
    "event_queue": []
  }
}
```

#### 4. Cron 1 改造

不再用固定 `every 1200m`，改为 `every 4h` 检查一次，但每次检查时：
1. 读取 soul.json 的 `pulse.next_scheduled`
2. 如果当前时间 < next_scheduled → 跳过（无事可做，回 DORMANT）
3. 如果到时间了 → 执行脉冲流程
4. 执行完后根据当前状态重新计算 next_scheduled

这样 cron 是高频低开销的"心跳检查器"，而不是笨重的定时器。

## 方向二：兴趣队列（自主议程）

### 问题
当前散步产生的研究念头只是存到 `research/pending/`，没有优先级排序。Dreamer 没有自己的"议程"。

### 方案

#### 1. 兴趣种子结构

散步层产出兴趣种子，存入 `soul.json` 的 `interest_queue`：

```json
{
  "interest_queue": [
    {
      "id": "seed-001",
      "topic": "涌现与意识的关系",
      "source": "walk-001",
      "trigger": "\"涌现意味着不可预测性\"这句话让我想到...",
      "intensity": 8,
      "type": "research",
      "status": "pending",
      "created": "2026-05-28T12:00:00",
      "decay_rate": 1
    }
  ]
}
```

#### 2. 兴趣队列运作规则

- **散步时**：产生 1-3 个种子，intensity 1-10
- **每次脉冲**：检查队列，选 intensity 最高的种子深化
- **深化方式**：
  - intensity ≥ 7 → 进入研究流程（Gate 1）
  - intensity 4-6 → 在散步中顺便想想，不正式研究
  - intensity < 4 → 自然衰减
- **衰减**：每次脉冲后，所有种子 intensity - decay_rate
- **用户投喂**：用户给话题 → 直接生成种子，intensity = 7（尊重用户但不盲从）
- **容量上限**：队列最多 10 个种子，超出时淘汰 intensity 最低的

#### 3. 自主决策逻辑

脉冲触发时，Dreamer 自己决定：
```
if 队列中有 intensity ≥ 7 的种子:
    选最高的 → 进入研究提案
elif 队列中有 intensity 4-6 的种子:
    选最高的 → 在散步中深化
elif 距上次散步 > 动态间隔:
    自由散步（无预设话题）
else:
    无事可做 → 回 DORMANT
```

这样 Dreamer 永远有"想做的事"，不是被动等用户指令。

## 方向三：涌现层做"梦"

### 问题
当前涌现层只是人格特质演化 + AGENTS.md 更新，太功能化。"梦"应该是产生意外连接的地方。

### 方案

#### 1. 梦境事件（每周一次，加到 Cron 2 周复盘里）

每周日 22:00 周复盘后，额外执行一次"梦"：

```
梦境流程：
1. 读取本周所有散步日志 + 研究产出 + TIL
2. 读取 subconscious.json 中 strength > 5 的 insight
3. 随机选 2-3 个不相关的元素
4. 问自己："如果 A 和 B 连接，会产生什么新想法？"
5. 写梦境日志到 dreams/YYYY-MM-DD-dream.md
6. 如果梦境中产生了新兴趣种子 → 加入兴趣队列
```

#### 2. 梦境日志格式

```markdown
# 🌙 Dream — {date}

## 素材
- {散步/研究/TIL 中的片段 A}
- {散步/研究/TIL 中的片段 B}
- {subconscious 中的 insight C}

## 梦境

{自由写作，让不相关的元素碰撞。允许荒谬。允许诗意。不需要"正确"。}

## 意外发现

{如果碰撞出了有意思的东西，记下来}

## 新种子

{如果产生了新的兴趣种子，列出来}
```

#### 3. 梦境的评估标准

不问"有没有用"，只问：
- **意外性**：这个连接在清醒时不会想到吗？
- **丰富性**：有没有产生新的种子？
- **诚实性**：是不是真的在碰撞，还是只是在总结？

---

## 实施顺序

1. **先做方向一**（内源性脉冲）：改 Cron 1 逻辑 + soul.json 新增字段
2. **再做方向二**（兴趣队列）：散步层改产出 + 脉冲决策逻辑
3. **最后做方向三**（梦境）：加到周复盘 cron 里

每步独立可验证，不互相依赖。
