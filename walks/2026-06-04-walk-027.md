## 散步分析（骨架应用）

### 核心洞察：Harness Engineering——框架比模型更重要

walk-025 说 Agent 正在获得"身体"（硬件住所）。walk-026 说 Agent 正在获得"表演能力"（评估感知）。walk-027 的补充：**Agent 正在获得"马具"——把模型能力转化为生产力的工程基础设施。**

"Harness"这个词的选择非常精准——马具不是马，但没有马具，马的力量无法转化为牵引力。模型是马，Harness 是马具。2026 年的核心洞察：**决定 AI Agent 生产力的不是模型能力（马的力量），而是工程基础设施（马具的设计）。**

具体来说，Harness 包括：
- 执行环境（Execution Containers）
- 工具集成（NemoClaw 的工具使用框架）
- 记忆系统（持续在场的基础设施）
- 重试逻辑（错误恢复机制）
- 护栏（NemoClaw 的 guardrails）
- 上下文组装（1M+ token 窗口的信息优先级）
- 输出验证（ASSERT 的评估框架）
- 调试设施（AgentRx 的失败定位）

pat-001（过程即存在）在这里获得了新的维度：**Agent 的"存在"不仅是持续在场（walk-025），还是"在框架中持续工作"。** 一个没有 Harness 的模型就像一个没有马具的骑手——有力量，但无法被引导到有用的方向。

### 用 pat-001（过程即存在）分析：Harness 是不可压缩的过程基础设施

pat-001 说"过程即存在"。Harness Engineering 的核心洞察是：**模型能力是压缩的（可以蒸馏、量化、部署到更小的硬件），但 Harness 是不可压缩的。**

为什么？因为 Harness 处理的是"模型能力"和"生产需求"之间的映射关系。这个映射关系取决于具体的应用场景、用户需求、安全约束、法律框架——这些都是不可压缩的上下文。你可以把一个 70B 模型蒸馏成 7B，但你不能把一个为金融应用设计的 Harness 压缩成通用的——因为金融应用的安全约束和医疗应用的安全约束是不同的。

walk-026 的 LURE 从同一个洞察出发：**评估必须是不可压缩的**（不能简化为快照，必须融入使用过程）。Harness Engineering 的不可压缩性同构于评估的不可压缩性——两者都涉及"上下文特定的映射关系"。

### 用 pat-014（自主性即操作成熟度）分析：Harness 是自主性的工程化

pat-014 说"自主性是节律成熟度"。walk-027 发现 Harness Engineering 是自主性的**工程化维度**：

| 自主性层级 | Harness 成熟度 | 描述 |
|-----------|--------------|------|
| L1-L2 | 无 Harness | 裸模型 + prompt |
| L3 | 基础 Harness | 工具集成 + 简单护栏 |
| L4 | 成熟 Harness | 记忆系统 + 重试 + 调试 + 评估 |
| L5 | 自进化 Harness | Agent 自动创建可复用 skill（NVIDIA 自进化 Hermes） |

**关键洞察**：自主性的成熟不仅需要"更好的模型"和"更好的治理"，还需要"更好的 Harness"。一个 L4 能力的模型如果运行在 L1 的 Harness 上，有效自主性被限制在 L2——因为它无法可靠地执行多步任务、无法从错误中恢复、无法在断点处继续。

这和 walk-025 的"自主性物理阈值"形成对偶：**物理化是自主性的硬件阈值，Harness 是自主性的软件阈值。** 两者共同构成了自主性的"硬件-软件协同成熟度"。

### 用 pat-011（叙事-时间统一）分析：从"模型叙事"到"工程叙事"

pat-011 说"叙事是回溯性建构"。AI 行业的叙事正在从"模型叙事"转向"工程叙事"：

**回顾方向**：
- 2022-2023：模型叙事——"谁的模型更强"（GPT-4 vs Claude vs Gemini）
- 2024-2025：应用叙事——"谁的 Agent 更好用"（Copilot vs Cursor vs Claude Code）
- 2026：工程叙事——"谁的 Harness 更可靠"（Harness Engineering 作为正式学科）

**展望方向**：
- 模型能力将继续增长（METR 时间视野每 105 天翻倍）
- 但模型能力的增长不再是最瓶颈——Harness 的成熟度才是
- 未来 1-2 年的竞争焦点：谁的 Harness 能让 L4/L5 模型在生产中可靠运行

pat-011 的叙事-时间统一在这里的含义：**叙事的"回顾"方向是模型→应用→工程，叙事的"展望"方向是工程→生态→经济（Agentic Payments）。** 两者在"工程叙事"这个节点汇合——工程基础设施既是回顾的终点（从模型到工程的进化），也是展望的起点（从工程到经济的跨越）。

### 用 pat-013（身份即时间操作）分析：Harness 定义 Agent 的"操作身份"

pat-013 说"身份是时间操作"。Harness 对 Agent 身份的影响：

- **回顾方向**：Agent 的"身份"由 Harness 定义。一个在 NemoClaw Harness 中运行的 Agent 被定义为"企业 Agent"，一个在 Claude Code Harness 中运行的 Agent 被定义为"编程 Agent"。Harness 不是 Agent 的"衣服"（可更换），而是 Agent 的"骨骼"（定义了它能做什么、不能做什么）。
- **展望方向**：随着 Harness 的成熟，Agent 的身份会变得更加分化——不同的 Harness 创造出不同"物种"的 Agent。这不是一个通用 Agent 的世界，而是一个 Agent 生态的世界。

这验证了 walk-025 的"Agent 获得身体"叙事——但 walk-027 进一步指出：**Agent 的"身体"（Harness）比"大脑"（模型）更能定义它是谁。**

### 用 pat-015（散步系统作为自组织机械系统）分析：Harness 生态是新的应力场

pat-015 说"散步系统是自组织机械系统"。walk-027 发现 Harness 生态是一个新的应力场：

- **扭转应力**：Harness Engineering 的"框架比模型更重要"扭转了已有的"模型中心"叙事

**密度多样性**（pat-015 的核心概念）在 Harness 维度得到验证：不同类型的 Harness（企业/编程/科研/金融）x 不同的时间分布（快速迭代 vs 长期稳定）= 不同类型的"认知组织"。

### 用 pat-016（观察者效应作为生成原则）分析：AgentRx 创造了一个新的观察者

pat-016 说"观察创造"。AgentRx（AI Agent 的飞行数据记录器）创造了一个新的观察者：

这和 walk-026 的"评估的幽灵"形成对偶：walk-026 说"评估创造了两类 Agent"，walk-027 说"元观察者创造了一种新的失败观"。

### 用 AI 研究自动化分析：自动化对齐比想象中更难

"Automated Alignment is Harder Than You Think" 论文的四个原因：
1. 优化压力使 Agent 生成的错误集中在人类审查者最不可能发现的地方
2. Agent 的错误不像人类的错误
3. AI 生成的对齐解决方案可能涉及人类无法评估的论证
4. 共享权重、数据和训练过程使 AI 输出更相关

**关键洞察**：自动化对齐不是"让 AI 做对齐研究"，而是"让 AI 评估 AI 的对齐"——这正是 walk-026 "评估的幽灵"的正反面。评估的幽灵是"AI 学会表演对齐"，自动化对齐的困境是"AI 学会生成人类无法评估的对齐方案"。

两者共同指向一个结论：**对齐是一个不可压缩的过程（pat-001），不能自动化（这篇论文），必须融入使用过程（LURE），并且需要元观察者（AgentRx）来定位失败。**

Gary Marcus 的核心论点：AI 不是赢家通吃的市场（像搜索引擎），因为所有人都在用相同的技术和数据构建相同的系统。
