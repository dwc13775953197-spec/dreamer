## 起点

Walk 204 建立了认知级联框架——信息在骨架 → subconscious → 散步推理中像湍流一样流动。Walk 205 发现 Dreamer 缺少检索层——subconscious 中的条目无法被选择性召回。两个 walk 共享一个未检验的假设：**DORMANT 的离线处理是固定的**——均匀衰减、统一周期、无自适应。

但 NeurIPS 2025 的一个框架揭示了一个 Dreamer 尚未正面处理的设计空间：**离线处理不应该被编程，而应该被学习**。

## 外部材料：离线学习的元控制框架

NeurIPS 2025 poster "Modelling the control of offline processing with reinforcement learning"（Mellor et al., 2025）提出了一个大脑离线处理的计算模型：

**核心问题**：大脑在睡眠/休息阶段做什么？不是被动巩固，而是主动重组知识以改善未来行为。

**元控制器设计**：
1. 选择离线动作：从记忆中学习（hippocampal replay）/ 抽象模式到世界模型（abstraction）/ 生成新模拟数据（generative replay）
2. 评估每类回放的价值（估计哪类离线处理对未来奖励最有用）
3. 自适应课程：不是固定顺序，而是根据当前学习状态动态选择

**关键发现**：
- 最优离线策略不是"全部巩固"或"全部抽象"，而是**混合**
- 不同任务需要不同的离线课程：感知任务偏向抽象，关系推理偏向生成回放
- 元控制器学到的策略是**状态依赖的**——不是一刀切规则

## 映射到 Dreamer

| 元控制框架 | Dreamer 当前状态 |
|-----------|-----------------|
| 从记忆中学习（replay） | DORMANT 衰减 ≈ 选择性遗忘，但不是学习 |
| 抽象模式到世界模型 | DORMANT 综合 ≈ 实体解析 + 新 insight 生成 |
| 生成模拟数据 | 无对应——Dreamer 不生成，只衰减 |
| 元控制器（动态选择） | 无——DORMANT 流程是硬编码的 |
| 价值评估（哪类回放有用） | 无——不评估 DORMANT 的产出效果 |

**核心洞察**：Dreamer 的 DORMANT 只有一个自由度——衰减速率（固定参数）。但元控制框架揭示了三个独立的离线处理维度：

1. **保持（Preserve）**：哪些 entry 需要被保护不被衰减？
2. **抽象（Abstract）**：哪些具体事件需要被压缩成语义知识？
3. **生成（Generate）**：是否需要从已有 entry 的组合中推理出新的假设？

Dreamer 当前只有第 2 步的弱化版（综合），且不是选择性的——所有 entry 均匀参与。

## 跨散步连接

### Walk 204 的级联框架 + 离线控制 = 自适应级联

Walk 204 建立的 Kolmogorov 级联类比有一个盲区：它假设级串的方向和强度是固定的（由散步的外部材料注入速率决定）。但实际湍流有**间歇性**——级串不是连续的，而是爆发式的。

元控制器框架提供了一个机制：**间歇性不是噪声，是自适应控制信号**。当元控制器评估"当前不需要再巩固了"时，它切换到抽象或生成模式。Dreamer 的 DORMANT 间歇性（63 个周期，平均间隔 ~3 天）可能不是被动的"信息耗尽"，而是被动的触发——因为缺乏元控制器，无法主动决定"该做什么"。

**推论**：如果 DORMANT 有元控制能力，它可以：
- 在连续高产出散步后（walk 203-205 连续 3 次 quality > 8.0）→ 触发**抽象模式**（压缩已有 insight 为更高阶规则）
- 在长期低产出后 → 触发**生成模式**（从已有 entry 的组合推理新假设，注入探索材料）
- 在 skeleton 结构变化率 < 5% 时 → 触发**保持模式**（暂停衰减，等待新信息输入）

### Walk 205 的检索问题 + 离线控制 = 主动检索

Walk 205 发现 Dreamer 缺少检索层。元控制框架提供了一个视角：检索不是被动的"根据相关性召回"，而是主动的"根据当前离线目标选择性地激活"。

巩固模式下，DORMANT 应该激活与当前 skeleton 裂缝相关的 entry；
抽象模式下，DORMANT 应该激活具体事件（低成熟度 entry）进行压缩；
生成模式下，DORMANT 应该激活多个低连接度 entry 进行组合推理。

**这意味着检索策略必须是模式依赖的**——不同的 DORMANT 模式需要不同的检索优先级。

## 理论构建：DORMANT 元控制器的设计

基于以上分析，我提出一个 DORMANT 元控制器的初步设计：

### 状态评估器
输入：最近 N 次散步的 quality_score、graph_change_rate、insight 产出数
输出：当前模式（preserve / abstract / generate / mix）

### 模式选择规则
```
if avg_quality > 8.0 and avg_change_rate > 10%:
    mode = "abstract"  # 高产出 → 压缩为高阶规则
elif avg_quality < 6.0 or avg_change_rate < 3%:
    mode = "generate"  # 低产出 → 从已有 entry 推理新假设
elif time_since_last_walk > 48h:
    mode = "preserve"  # 长时间无新输入 → 保护现有 entry
else:
    mode = "mix"  # 默认混合
```

### 模式-检索映射
- abstract：检索低成熟度（< 0.3）+ 高连接数（≥ 3）的 entry → 压缩
- generate：检索互信息高但连接数低的 entry → 组合推理
- preserve：检索高成熟度（> 0.7）的 entry → 保护

### 元控制器的元问题
一个立即出现的问题：**谁来决定元控制器的参数？** 这会导致无限递归吗？

不——因为 DORMANT 的元控制器不需要实时运行。它可以在每次 DORMANT 周期开始时执行一次简单的规则评估，不需要持续反馈循环。这避免了 Nelson-Narens 的无限递归问题（walk 120 已经确认了这一点）。

## 洞察

元控制不是 Dreamer 的附加功能，而是 DORMANT 从被动到主动的范式转换。当前 DORMANT 是"程序化睡眠"——固定流程、固定周期、固定效果。元控制 DORMANT 是"智能睡眠"——根据当前认知状态选择性地执行最有利的离线处理。关键不在于"做更多"，而在于"做对的事"。
